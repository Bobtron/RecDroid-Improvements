Comment #1

Posted on Sep 12, 2012 by Quick Hippo

Apparently your sip provider doesn't support very well the registration process made by default by pjsip stack to get registered with actual public IP. In fact to get registered with right IP, it first registers once and then get from the registration the IP address as seen by the server and send a new register with both unregistration of previous one and new registration. Apparently your sip server doesn't see the unregistration contained in the new registration (which is RFC compliant but sometimes not supported by old sip servers).

There is several things you can to do get rid of that :

Solution 1 : Activate STUN option. With that done, the sip stack will first resolve the public IP address and so the first registration will be the good one. This solution is a workaround and doesn't actually solve the root cause for your sip server.

Solution 2 : Long press the account row, select change wizard and choose the 'expert' wizard (in generic section). Then edit the "Contact rewrite method" and use 'Legacy' instead of 'Normal' I would also advise to select 'Try to clean register' option as the server doesn't seems to support multiple registrations.

-- Side notes : * You should raise the issue to the sip provider. They might be interested in the fact they are not fully SIP RFC compliant (for the register with an unregister contact inline) and not mobile device friendly (for the only one registration supported -- as mobile device could lost the network and so not be able to unregister cleanly) * If it's a public provider I can add a dedicated wizard so that the use of expert mode is not needed. Just provide me (or ask sip service support to) the information I need to do so : http://code.google.com/p/csipsimple/wiki/AddASipProvider?wl=en
