Posted on Aug 23, 2011 by Quick Hippo

There is several levels of integration. I don't know what is the current status of skype integration, but previously they did really really crappy thing (by intercepting an intent from the private android api).

So in order to really understand what is happening to you I need to know :

When you launch a call from the stock dialer, is there a popup with the choice between "dialer" and "skype" and a checkbox with "use this action by default". If so, NEVER use skype. This choice lead to go in the way skype did things and that is not correct regarding android OS. If there is not such a popup you have probably already choose one of these two choice and selected the "use by default" option. To cancel that, just go in android settings > application > application management > skype > remove default actions.

Then, you'll have csipsimple chooser (which can silently ignore a call if current network connection is not valid for sip or if you added a rule in csipsimple to ignore this kind of number). CSipSimple version of the market should support by default skype (if they didn't change their application). So you can still select skype as outgoing option.

If you are using a nightly build, this is now available as a plugin.

Nightly builds : http://nightlies.csipsimple.com/trunk/ Skype plugin for outgoing calls : http://nightlies.csipsimple.com/plugins/

The point, is really to never use the way skype did the integration. What they do break all other applications that may handle outgoing calls intents.
