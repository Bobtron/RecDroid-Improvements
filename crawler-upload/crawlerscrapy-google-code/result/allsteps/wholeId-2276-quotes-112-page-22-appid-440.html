Comment #4

Posted on Nov 18, 2013 by Quick Hippo

If you don't want csipsimple to auto-detect the 'real' contact you have to disable the "allow contact rewrite" feature (use expert account wizard to do so : see http://code.google.com/p/csipsimple/wiki/ExpertSettingMode?wl=en#Account_settings to switch to expert wizard)

The "allow contact rewrite" is a feature, not a bug. When you register a sip server the contact should be reachable by server even if socket is closed (in case of reliable transport). That's why a contact with local IP is virtually wrong and should be rewritten by the app. The allow contact rewrite feature, use a way that is described in a RFC to detect what should be the real contact based on the headers returned by your sip server after the first register round. This way, the contact header registered on server is the real one. Actually, some sip server does that automatically and rewrite internally the remote ip/port as they know it from socket that is currently registering. XLite has not such a feature, and that's why you don't see the rewrite of contact header done.
