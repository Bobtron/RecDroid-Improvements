csipsimple - issue #1496

One way audio after ringing more then 4 sec

Posted on Jan 14, 2012 by Quick Giraffe

What steps will reproduce the problem? 1. make a new call from PhonerLite (PL) 2. wait for couple of rings, then pick up CSS phone 3. there is no audio from PL to CSS, audio from CSS to PL is OK

What is the expected output? What do you see instead? Expected - audio is OK in both direction. But when I wait for ringing more then several seconds and then pick up CSS phone - there is on audio from PL to CSS...

What version of the product are you using? On what operating system? r1196, Android 2.1

Please provide any additional information below. The new bug was introduced after r1158 build (the latest build without that problem). If I use CSS r1158 there is no any problem, even if I wait for many rings and then pickup the phone - audio is just fine in both direction. But when I install r1196 (r1193, r1177 exhibited similar problem) in the same environment and run the test again - the problem start occurring and it's 100% reproducible. I switched builds forth and back many times and made the same call with only one difference - wait time for pickup the ringing CSS: 1. almost immediately (in less then 4 sec) 2. after 4-5 sec of ringing In the first case audio is always just fine. In 2nd case it's always one way (no audio from PL to CSS).

I looked at detailed logs, produced by FreeSWITCH, and there is absolutely no difference between two sessions (case #1 and case #2).

Current solution is: install old version r1158 or, if I use a latest build, pick up CSS phone very promptly...

There is no such problem, when I make calls in opposite direction - from CSS to PL though.

PhonerLite (SIP client on Windows) is here: www.phonerlite.de/index_en.htm
