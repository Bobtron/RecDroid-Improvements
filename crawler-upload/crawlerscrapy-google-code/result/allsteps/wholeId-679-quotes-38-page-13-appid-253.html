Comment #6

Posted on Dec 21, 2009 by Happy Hippo

The issue, is that Cyanogen's build seemed to handle background applications differently, effectively killing QM while the browser was present. This cause the request token to be lost, so when the user returned QM was restrted, and looked for A RT that was no longer there.

I have introduced code that allows QM to reinstantiate itself after a kill from the activioty manager
