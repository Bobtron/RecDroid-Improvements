csipsimple - issue #471

"Fail to create transport" error

Posted on Dec 8, 2010 by Grumpy Lion

What steps will reproduce the problem? A "Fail to create transport <non-printable chars>" message will show on the screen occasionally. There's no specific event or action triggering it, as far as I can tell. I only have one account configured, to pbxes.org.

What is the expected output? No error. What do you see instead? Message repeats every few seconds until the application is force closed from a task manager. Once restarted, the application behaves normally.

What version of the product are you using? Latest Market version (00-16 I believe) On what operating system? Android 2.2, Sense HD ROM (http://forum.xda-developers.com/showthread.php?t=861690), HTC Desire GSM.

Please provide any additional information below. Log files already submitted via mail.

Comment #1

Posted on Dec 9, 2010 by Quick Hippo

You have probably another SIP application running on your phone that hold TCP & UDP port and prevent CSipSimple to bind ports locally.

Either ensure that there is no other SIP apps running (even in background at the same time). Or change TCP / UDP port of csipsimple in expert settings (Settings > menu switch to expert mode and Network > TCP & UDP port.)

If it doesn't help can you try to send me logs (see HowToCollectLogs)
