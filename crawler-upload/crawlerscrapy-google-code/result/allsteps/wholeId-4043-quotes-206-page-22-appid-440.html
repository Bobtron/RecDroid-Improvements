Comment #13

Posted on Mar 17, 2011 by Quick Hippo

I get some feedback from Motorola guys. They are very reactive that's great :)

Regarding their feedback the atrix already has a echo canceller that could conflict with a software echo-canceller. As consequence they advise to disable it in the app if frequency is set to 8kHz.

So : In csipsimple media settings > Disable echo cancellation and disable VAD

Try 8kHz and 16kHz.

Also I'd be interested by a pure loopback audio test (with logs). To do so : * Turn on log recording : (in csipsimple dialer - press menu > help > Record logs..) * Then go in csipsimple settings, press menu and select "Expert mode" * Then press again menu button and press "Test audio". It will open a little popup and you'll have a loopback audio test. (It directly send what is recieved from mic to the earpiece). * Press back (it will quit the test mode), press back again (you should be on the main dialer), press menu > help > Stop log recording and send it. Comment #49

Posted on May 10, 2011 by Quick Hippo

Not sure : older version supports less codecs. Pbxes.org is known to behaves a weird way (non RFC compliant) with TCP. Depending on the way you create the account and the size of UDP packets (latest versions produce bigger SDP packets cause supports more codecs), CSipSimple can try to switch to TCP which will lead to this behavior (not able to end the call). Try the "Use compact mode", it can help in this case. (Or you can try to disable a couple of codecs in codecs list, for example only keep PCMU/A).

Comment #52

Posted on Jun 12, 2011 by Massive Lion

Hi. Just giving an update since I figured out the problem I was having on the Atrix. After taking a look at the info logs, I noticed that on outgoing calls, which were working fine, it was using PCMU 8khz. On incoming calls, which were not working (as described in my last post), it was using speex 8khz. Another detail I noticed was that on ending the call on my Atrix, the other phone would hear a high pitched tone.

Anyway, I disabled the speex 8khz codec, and now it uses PCMU 8khz for both outgoing and incoming and works fine. Now if my understanding is correct, these codecs are only used to talk to my sip provider, which translates and sends a different stream to the other phone. So my guess is that pbxes.org has poor support for the speex codec? If that's right, seems like an easy fix to csipsimple. In fact, it may already be implemented because I'm using the generic (advanced) sip account and not the premade pbxes.org one.
