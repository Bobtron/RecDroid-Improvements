csipsimple - issue #248

Issues while using in call dialpad

Posted on Sep 28, 2010 by Quick Bear

What steps will reproduce the problem? 1. Call VM access number 2. enter mailbox number 3. enter password

What is the expected output? What do you see instead? 2 problems. 1. Screen goes black and lights up only after I connect so I can't enter any digits because I miss the first voice prompt. Connect screen should come up faster.

  1. Then I open the dialpad and enter the mailbox number and get to the passcode and the screnn locks in the middle of entering digits. This should not happen. The timeout should be interrupted while you enter digits.

What version of the product are you using? On what operating system? Dev 12-35, on HTC Hero with VillainRom 12

Please provide any additional information below.

Comment #1

Posted on Sep 28, 2010 by Quick Hippo

For first one it's an UI optimization task. For now, I'll postpone this. In fact audio has priority over everything, and I think that it's the better approach. Besides implementing some predictive thing will add a lot of complexity to the code. So for now, it's not a good idea at all. Code must be kept simple while not fully stable and compatible with all devices.

For your second issue, you're right, I missed that, timer was reset if you click on call control buttons but not for dialpad buttons. I've fixed it on my branch and will be included in next release.
