Comment #1

Posted on May 12, 2011 by Quick Hippo

You should try to revert to the old (before android 2.3 api) for the communication audio routing.

Read this part of the FAQ : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting

And test (once ExpertSettingMode is activated to : * Set "micro source" to DEFAULT (instead of VOICE_COMMUNICATION) * Set "mode for sip calls" to NORMAL (instead of IN_COMMUNICATION)

The new API should work better on android 2.3. However some ROM actually do not take that correctly into account (even official ROMs).

Also, you are maybe affected by the PSP behavior. Normally this one two should has been fixed in android 2.3 (it's an android behavior, not a csipsimple behavior). Read this FAQ entry for this point : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#When_screen_goes_off_sound_quality_is_bad

Let me know how it goes and which of the two point helped. Maybe I could change new default settings for the Desire HD. (What is strange is that I have a desire HD and with the official ROM the new android API works well).
