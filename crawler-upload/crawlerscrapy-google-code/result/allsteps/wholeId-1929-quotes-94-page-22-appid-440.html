csipsimple - issue #2745

Registration is not governed by settings

Posted on Jul 21, 2014 by Swift Giraffe

What steps will reproduce the problem? 1. Use the default registration value (900) 2. Connect to FreeSWITCH 3. Look at registrations 4. CSipSimple always registers for 90 seconds or less 5. CSipSimple reregisters in 25 seconds or less 6. Don't use any sane fail2ban rule, because you will get banned

What is the expected output? What do you see instead? Registration Expire time should be 900 with the reregistration somewhere around 300 seconds according to the code in start_timer (https://code.google.com/p/csipsimple/source/browse/trunk/pjsip_android/apps/pjsip/project/pjsip/src/pjsip-ua/sip_timer.c?r=835)

Also, the minimum expire is not honored.

What version of the product are you using? On what device / operating system? Latest in play store, Samsung S3, Android

Please provide any additional information below. I am trying to dig through the code (first real attempt at digging into an Android project). I will try to figure this out and post a patch.

Comment #1

Posted on Jul 21, 2014 by Quick Hippo

Hi, CSipSimple entierly rely on the rock robust pjsip sip stack. It's not used only in pjsip but in various other opensource and commercial projects. Re-registration interval should work properly. There is obviously settings that would allow to override default settings, but by default it should not behaves the way you observe. So there is maybe a bug in csipsimple glue inserting bad/unexpected settings or something with the server vs app configuration that leads to this situation. So, let's try to debug that together. I don't think it's a general bug as I don't reproduce that on all sip providers I test with.

To debug the problem : 1- install the latest nightly build (always better as contains latest code : http://nightlies.csipsimple.com/trunk/) 2- follow steps indicated here to collect and share logs : http://code.google.com/p/csipsimple/wiki/HowToCollectLogs?wl=en 3- in the email produced refer this issue number and indicate the way you configured your account (expert mode? basic?)

In your case, during reproduction step, left the app idle for at least 15minutes.
