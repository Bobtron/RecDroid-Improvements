csipsimple - issue #2617

CSipSimple Delay in Detecting Incoming Calls

Posted on Jan 9, 2014 by Helpful Rabbit

What steps will reproduce the problem? 1. Using Google Voice to forward calls to CallCentric registered in CSipSimple 2. Be sure CallCentric is registered properly (green status) 3. Place Call to Google Voice number

What is the expected output? What do you see instead? The expected response is for CSipSimple to almost immediately begin ringing and allowing me to answer to the call. Instead, CSipSimple doesn't detect the calls at all and my phone never rings OR I get one ring and then the call is missed. The calling line will have experienced 7 or 8 rings prior to this time.

Please note - when I forward my GV calls to a LAN line or a real mobile number, the delay is almost 0. I don't understand what is taking CSipSimple so long to pick up the call. (CallCentric shows that the calls are registered on-time)

What version of the product are you using? On what device / operating system? CSipSimple 1.01.00 r2351 and r2352 (I've tried both recently) Galaxy Note II, Android 4.2

Please provide any additional information below.

I tried forwarding the call from CallCentric to a landline and the connection still was almost instant. I know the issue is not CallCentric receiving the call, rather I'm confident that it's just CSipSimple not connecting to the server often enough to pick up the call nearly instantly. I don't care if it drains my battery, I need to get these calls!

Comment #1

Posted on Jan 10, 2014 by Quick Hippo

Actually csipsimple has a persistent connection with your sip provider (SIP does not work a way things are frequently checked to ensure there is no incoming call, when there is an incoming call, the sip server is supposed to send something to the SIP client and the client is supposed to ring).

The possible reason why the app could not ring immediately is that the phone is in deep sleep mode (in this mode network layer of the phone is sleeping and the application might possibly not be awaken if there is a problem in the ROM of the device that does not wakup apps when there is some network activity).

That's why my first question is : when the problem occurs is the phone screen on? If yes, the problem is probably that the sip packet sent by callcentric server simply arrives late on the sip client. Maybe a problem on callcentric config (I don't know if they have a way to delay calls and try on various things before trying sip device). To be sure, I would be interested by logs (http://code.google.com/p/csipsimple/wiki/HowToCollectLogs?wl=en) that could help to see if there is a big delay between packet received from server and ringing action of the phone.
