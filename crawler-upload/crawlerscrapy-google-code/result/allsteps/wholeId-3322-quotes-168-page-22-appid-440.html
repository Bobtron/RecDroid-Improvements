Comment #1

Posted on Jan 9, 2012 by Quick Hippo

That's more likely to be an issue for the sip server side. Basically if server tell that sip:620@my.dyndns.org username is "not found", it probably means that it's not configured to recognize this domain name but rather it's own IP as the domain name for its users.

The client side just send info you configure to server, and if server reply that with the dyndns name as domain it can't find user in db, I really don't think that's a problem on client side, but rather on the server side configuration and how it recognize its users.

To try to give you more details : the account ID is something send as "username" to the sip server. Registrar and proxy are somehow the uri of sip server. So indeed, every dns name that resolve to your box IP will works for registrar and proxy. However, for account id, since this is never resolved as an IP, it entierly depend on how sip server understand that.

To try to do a comparison. If you want to access your gmail account. You'll first try to access gmail on your browser. To do so, you'll type in url bar "http://gmail.com". It will also works if you type http://www.gmail.com or http://googlemail.com or http://mail.google.com. It goes the same for registrar and proxy uri. Sip client, as web browser do, will first resolve the dns entry and try to talk with the IP behind this name. As for the account id, it's a little bit as your username when you'll try to login on gmail. If you type user@mail.google.com in login box, gmail will cry. If you type user@gmail.com it will works. It's the same with sip id, if the sip server is not configured to recognize that gmail.com is the same that mail.google.com as username domain, it will not consider you as a valid user.

I'd love to be able to help you with fritzbox configuration, but unfortunately I have and never had any account on fritzbox. Hopefully, if some kind user of fritzbox read this and is aware of how to configure the domain name on it, he will help you here. But as an issue of csipsimple, I mark the issue as wrongconfig one.

Posted on Jun 12, 2012 by Happy Rabbit

Hi, thanks for sharing this!

It also works on my fritz.box and samsung galaxy s advance (I9070p) using the csipsimple client

I believe the trick is to : * set the accountid to * force the registrar uri to sip:fritz.box * set the proxy to your dyndns hostname. (replace 620 by your fritz boxes internal telephony device number)

For anyone interested in the complete settings:

accountname: yourchoice accountid: registration URI: sip:fritz.box

realm: * username: 620 data(password): changeme data type: plain password scheme: digest

transport: auto publish enabled: unchecked (default) register timeout: 900 (default) register delay before refresh: -1 (default) force contact: blank (default) allow contact rewrite: checked (default) contact rewrite method: normal (default) try to clean registers: unchecked (default)

proxy uri: sip:yourhost.dyndns.org (your dyndns or similar hostname) srtp mode: disabled (default)

voice mail: empty (default) Registrar: sip:fritz.box

Don't forget to activate telnet and enable the reg_from_outside = yes; for this extension first ! to enable use an internal handset (yes a handset) and dial #96*7* (and #96*8* to turn of afterwards). it will be reset after a router restart.
