Comment #5

Posted on Nov 14, 2010 by Quick Hippo

With 0.00-15-12 you can now choose which method you want to use for DTMF.

I guess that your problem is that your SIP server (media proxy) misbehaves regarding DTMF. Csipsimple support 3 modes, and that the default is to try the method 1 and fallback to method 2 if server announce that method 1 fails... I think that's what is happening with your server : server say that it fails with method 1 while it actually works, and then I fallback to method 2.

So what you should try now is to use 0.00-15-12 and in settings > media > DTMF mode choose "In band generation" (the second choice). It will force only the mode 2 (which simply simulate the tone in the flow).
