Ok, I think I was not clear enough in last reply.

When I say "the plugin from java source code", it's because there is a java project (separated from main core csipsimple java project) that is dedicated to the plugin.

So, you finally need 2 apps to be installed : CSipSimple (BTW, you can use the one from playstore or one you build by yourself it does not matter to get g729 working!) AND (what I was talking about): CSipSimpleCodecG729 which is another app (see the link of the comment 1, but see it carefully, it point to a different folder!). It's a plugin. You need to install it as an application. As soon as this application is installed, CSipSimple application (or your built version of csipsimple) will discover the plugin application and propose you to enable g729 in list of codecs.

But again, there is several applications. Do not confuse between the core csipsimple (that you aready built successfully) and the plugin(s) applications.

That's what the sentence "Pick at least SipHome? and ActionBarSherlock?. Add CSipSimpleXXX-projects if needed. " (in http://code.google.com/p/csipsimple/wiki/HowToBuild#Setup/import_application_in_your_IDE_%28Optional%29) means.

CSipSimpleXXX-projects are plugin projects that have their own installation. It's not library projects, it's plugins projects. So you need to install these projects separatly from core app.

Just as side note, there is other plugins for csipsimple : one that bring the alpha video feature, one that bring some legacy codecs and some not yet finalized codecs (opus, g726, codec2 etc), and several others.
