csipsimple - issue #1905

CSIP does not switch back to standard volume control after call

Posted on Aug 24, 2012 by Happy Dog

What steps will reproduce the problem? 1. Install latest version on Samsung Galaxy W I8150 2. Receive Phone Call

What is the expected output? What do you see instead? During the phone call received with CSIP all other audio settings are set to nothing and you regulate the call volume (as normal). After the call the system does not switch back to the standard volume setting and remains in volume state during phone call (no sound and regulation of call volume).

What version of the product are you using? On what operating system? CSIP 4-00 r1788 Android 2.3.6. Kernel 2.6.35.7

Comment #1

Posted on Aug 24, 2012 by Quick Hippo

I'm not sure to get the point.

There is a lot of volume settings in android and there is also inside the app (software volume). So not sure which one you are talking about.

Here's how it should work : - When the app enter call state it saves the volume of the stream type it will use to play audio. "standard volume" doesn't exists. There is the volume for audio calls, the volume for music, the volume for notifications, volume for ringer etc. Depending on the configuration of the application it will use one of these volumes/stream. By default it uses the one of calls. - Then it changes this volume to the one set in settings (there is settings depending on bluetooth or not use). - When you press volume up/down, it changes the csipsimple settings so that next call will be able to get again the same volume level. - If you go in media settings while in call you can also change software volume setting. It also save that as a setting (that you can also modify when not in call). - When you finish the call. The application will restore the previous volume for the audio stream it used. So for example if was call volume it's restored to previous value and if you place a GSM call it will use the previous volume it used for GSM calls. - It doesn't restore / modify the application settings (the initial stream volume that you could have changed during call and the software volume amplication that you could also have changed during the call. - If you place a new SIP call, same process will be done and it will so restore the same volume than the one you had at the end of the previous call.

  • When you are not in call, the volume you control using the volume buttons is the ringer volume. Unless the app is configured to use ringer stream (which is normally never the case unless you volutarily go in expert setting mode and changed that), the application never changes this volume.

In your case : If what is described doesn't match what you observe : which is the failing point and how do you test it fails? If it's what you observe, I guess that it's annoying for you? If so which point is annoying and what would you expect instead? I could introduce a new expert setting to improve your user experience.
