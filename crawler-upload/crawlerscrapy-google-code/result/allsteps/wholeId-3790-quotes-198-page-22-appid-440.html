Are you running an official gingerbread rom? Normally gingerbread introduces a way to lock wifi mode so that this problem shouldn't appear. By default if gingerbread is detected, csipsimple desativate the PSP hack. If it does not work it is cause the rom doesn't support it. Usually htc do things cleanly, but possible they postponed this feature of the 2.3 if they also do not include a stock sip app. If you are running a custom rom you should alert makers of the rom that they don't support correctly the high perf WiFi lock in their rom.

If they need more details I'm very active on the subject on android issues tracker and can give some pointers ;-).

Let me know how it goes with the psp hack activated.

Comment #6

Posted on Jun 12, 2011 by Quick Hippo

@casou : instead of the "invert proximity sensor" option, you should try the "keep awake while in call" option. While invert just invert the way the application interprete the proximity sensor, keep awake while in call change the way the app behaves :

Instead of using the proximity sensor to turn off the screen, it lock screen in dim mode and add an overlay that prevent from unwanted action. So this setting is the good one to choose if your device have the PSP behavior.

I didn't know Defy was also impacted by this problem, I'll add it to list of device with this problem so that it will be automatic :).
