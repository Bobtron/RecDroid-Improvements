csipsimple - issue #1102

Connecting To Televantage Sever

Posted on Jun 24, 2011 by Swift Hippo

What steps will reproduce the problem? 1.Input Settings In Phone account id: <sip:user@xxx.xxx.xxx.xxx:5060> Reg URI: sip:xxx.xxx.xxx.xxx:5060 Proxy: sip:xxx.xxx.xxx.xxx:5060 2.Obain Binding from Server 3.Try to initiate call

What is the expected result? I would expect that the binding would be on port for the contact URI to be 5060. What do you see instead? The resut that I get is sip:user@xxx.xxx.xxx.xxx:37706;ob

What version of the product are you using? On what operating system? Android r944

Please provide any additional information below.

With Televantage Server when you connect and register your binding you must have a contact uri with port 5060. If you dont have port 5060 and you initiate a call the android will ring but once you connect the call the call is dropped. We have had the same issue with Aastra SIP phone and we fixed by forcing the port 5060. With csipsimple even though we specify the port in the settings we are still not getting the binding with the port 5060. Always in the port range from 33000-52000.

So to recap. We have no problem getting a registration with the server, we have a problem with incomming calls dropping when the call is answered.

Posted on Jun 24, 2011 by Quick Hippo

1 - switch to expert setting mode (see the wiki about this point : ExpertSettingMode) 2 - go in settings > network change Local binding port from 0 (random available port) to 5060.

It may not be enough. In fact the contact port should be the actual port where to contact the sip client. If you are behind a NATed network the port may change from the one that is actually locally bound (on the phone) from the one that is open by the router outside of the nated network for this client.

By default csipSimple (actually the underlying sip stack pjsip) use a contact rewrite method to find out the actual public port (and IP) in order to announce the correct one to the server (in case this one use it ;) ). So if you are in this case and that the server does not accept to have the correct actual IP and port (because it solves NAT on it side), you have to disable this feature in the corresponding account. To do so :

Switch your account into an expert account (see the same wiki page than for the first point, but about expert account this time), and disable the feature "allow contact rewrite".

If it helps, let me know, and gives me infos about this server so that I can add a Televantage wizard in order to solve the problem for all other users in the same case ;)

Comment #2

Posted on Jun 24, 2011 by Quick Hippo

Oh and I forgot, don't confuse between remote port (sip:yourserver:5060) and the locally bound port in CSipSimple that's two really different things. One is configured in the account, and the other, the local port is obviously a general setting not linked to the account configuration at all. So when I talk about local bound port it is in gloabal settings of csipsimple in network (there is settings for TCP, UDP and RTP local port). And keep in mind the second point, registering like that (without stun and without the pjsip rewrite contact feature) will mean announce a wrong IP/port in case you are behind a NATed network ! Cause obviously the client does not decide how the router map the public port. (But I guess you are inside a private network that is the same than the one on which your server is, isn't it?)
