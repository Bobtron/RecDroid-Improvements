csipsimple - issue #2653

Extremely slow "Pickup SIP contact" when attempting to transfer call internally

Posted on Feb 24, 2014 by Swift Bear

What steps will reproduce the problem? 1. Select "Transfer call to Number" when in a call 2. Attempt to type in extension number (210) and click OK

What is the expected output?

Software should allow extension number to be typed in directly without any delay, and it should be possible to transfer the call within a few seconds. It should ideally be possible to just use the phone keypad rather than the Pickup SIP Contact screen, and not attempt to select a contact. If contact selection is mandatory, then it should either not interrupt typing, or it should at least wait until no keys have been pressed for a second or so before attempting to filter contacts.

What do you see instead?

20-30 sec delay between each key press (I think device is trying (inefficiently) to scan phonebook for number entered - takes nearly 2 minutes to be able to transfer call (by which time the other party has often hung up)

What version of the product are you using? On what device / operating system?

CSipSimple 1.02.00 r2330 on Samsung Galaxy S4 running Android 4.3 (Kernel 3.4.0-2082040) - this behavior has been consistent across all versions of android 4.x installed on this device.

Please provide any additional information below.

I have 4 google accounts linked to this device with ~300 contacts total
