csipsimple - issue #563

Task switching with open call - no way to end call

Posted on Jan 3, 2011 by Swift Giraffe

What steps will reproduce the problem? 1. Commence call from within csipsimple (over voip). 2. Press the 'home' button to return to the Android home screen. Green 'in call' phone icon shows on status line and the line is open (both parties can still hear each other) 3. Pressing the csipsimple icon from the home screen opens a new instance of the program - a dialler screen. Original call is still active. 4. Use the 'disconnect' option from within csipsimple. Call is still active 5. Check active processes. csipsimple is not listed as an active task, but call is still active.

Essentially, if you return to the home screen during a call the call still remains as a phantom task, with no way of being able to reenter the task to end the call.

What is the expected output? What do you see instead? Pressing the csipsimple icon from the home screen should re-enter the current session at the call screen.

What version of the product are you using? On what operating system? Huawei Ideos U8150 running Android 2.2 csipsimple v0.00-16

Please provide any additional information below.

Comment #1

Posted on Jan 4, 2011 by Quick Hippo

You should have still a notification icon in the top notification bar. It allow you to enter again the call.

So what is expected is that this notification icon sill appear after disconnecting. What you expect is not a good behavior.it is not the behavior of the stock android telephony app. On the android phone app in call screen is also a different process and is not disconnected when you quit the dialer or contact app.

So the bug is that notification icon disappear not that you can quit csipsimple nor that opening csipsimple open the dialer (think about the future conference feature and think about somebody that just want to consult cal l logs.)

The correct way to end the call, just like for gsm calls (at least on a majority of android rom and on the standard one) is to open the notification bar and clic on the active call row. You'll then retrieve the in call screen and be able to end the call.

Revision 559 should fix the problem with the disappearing icon in notification bar. However to access ongoing call, you still need to read my comment 1 to understand that to the way to access the ongoing call screen is to press the notification and not launching the app.

What I fixed is that if you quit CSipSimple using CSipSimple dialer > menu > Exit the ongoing call icon in notification bar will remain which is what is expected. Quiting the app does not mean terminating calls. Just like pressing back key in Android stock dialer/contact app.

This icon in notification is THE way to access an ongoing communication when you are multitasking. Launching CSipSimple will launch CSipSimple dialer as it should do.
