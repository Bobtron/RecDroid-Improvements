csipsimple - issue #1202

Call will not complete- enters loop??

Posted on Aug 11, 2011 by Massive Rhino

What steps will reproduce the problem? 1. Install Google Voice and set it as the handler for all calls (not certain this is necessary for the problem to recreate- probably isn't based on the behavior I am seeing).

  1. Create a rewrite filter for 10 digit numbers similar to the one below (001 is to make a US call via Voipcheap, 10 digit number for my Google Voice access number, * and pin number to access Google Voice options, 2 to place a Google Voice call followed by pound):

Prefix: 0018595551234;*,1234,2, Suffix: #

  1. Initiate any 10 digit call using the native dialer.

What is the expected output? What do you see instead?

Expect the dialer choices to come up, choose "Use Voipcheap" (voipcheap being the name of the SIP phone), and then for the call to complete as normal using the csipsimple dialer.

Instead after selecting "Use: Voipcheap" in the Dialer app, the red-gradient screen comes up for a second (I think this is the csipsimple splash screen?) but then quickly disappears to the homescreen. Concurrently a message pops up in the notification bar with the green phone icon: "call in progress".

After this, if you try to bring up the csipsimple app again using the recent apps shortcut, it will go back to the dialer selection dialogue. If you select csipsimple from the regular app drawer, it will go to the csip dialer. Also, the "dialer" app (I think this app is part of the csip package..?) shows up and stays in the recent apps drawer even afer csip crashes (don't think this is normal).

If you select the dialer app or the csip app from the recent app drawer, the dialer selection box (populated with the number you tried to dial before) comes up. If you select the desired SIP dialer (in this case, Voipcheap), csip crashes just as before. Then if you select Dialer or csip from the recent app drawer again, you get the dialer selection box (populated with the original number you tried to dial) once again. This will happen over and over again in a loop.

What version of the product are you using? On what operating system?

0.02-03 r944 on Android 2.2

Please provide any additional information below.

Shorter filters using Voipcheap seem to work just fine. Replacing the wait symbol (";") with a pause symbol does not solve the problem.

Comment #1

Posted on Aug 12, 2011 by Quick Hippo

Duplicate of issue 138.

, and ; are not yet supported for SIP calls. CSipSimple basically try to call

sip:0018595551234;*,1234,2,XXXXXXXXXXXX#@voipcheap.com

What you'd like to have is : sip:0018595551234@voipcheap.com and then the app to send dtmf codes (* 1234 2 XXXXXXXXXXXX#)

Please also note that if you want to enrich csipsimple chooser with other access number (but for GSM calls), there is existing CSipSimple plugins for that point. But it's not about making sip calls. Usually users are more in this case than in the case to try to use an access number through SIP (which usually make no sense cause SIP already allow same (or better) call rate than rates from an access number) -- usually those that put an access number on PSTN network also have a SIP service.
