Comment #4

Posted on Jun 2, 2012 by Quick Hippo

When you have the problem only on one network, it usually means that this network has firewall that blocks outgoing sip packets. Unfortunately the sip client, in this case, can't do nothing to solve the issue. The sip server on its side can try to use other port, or to support encryption (sips). Another solution is to configure the wifi access point so that it let SIP protocol go out.
