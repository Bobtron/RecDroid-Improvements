csipsimple - issue #2611

More options for local account

Posted on Jan 5, 2014 by Swift Wombat

What steps will reproduce the problem? 1. Create an account with the 'Local' wizard. 2. Long-press on the account, 'Choose wizard', select 'Expert'. 3. Select the account.

What is the expected output? What do you see instead? Expect to be able to save the account again. (Actually I'm looking just to tweak the DSCP value, leaving everything else intact.)

Instead, I see 'Account id' empty and orange (required but missing). 'Save' button is disabled.

What version of the product are you using? On what device / operating system? CSipSimple 1.02.00 r2330 on Android 4.4 on a Moto X (ghost).

Comment #1

Posted on Jan 10, 2014 by Quick Hippo

Indeed, local wizard is a little bit special. Actually it just show some hidden account that must always be there : the local binding account of the transport. As consequence, as soon as you change the wizard to something else than "local" it does not become anymore local but become a real account that does not benefit anymore of automatic ip/port contact discovery. Currently there is few settings possible for local account. More are planned (and might be necessary for various use cases); but currently the only way you have to get the other settings are to manually do what is done automatically by "local" account. To do so, you simply have to enter as account id your "contact". This means if you want it to be valid for a remote (which is not mandatory depending on your use cases); that you have to enter your ip address in domain; (for example if 192.168.1.1 is the ip address of your device).

I keep the issue open to track progress on the task to support more settings on local account.
