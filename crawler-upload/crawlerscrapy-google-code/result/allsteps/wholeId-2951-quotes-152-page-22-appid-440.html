Comment #1

Posted on Jul 26, 2012 by Quick Hippo

Hi,

Usually the required thing is not in To but in From. Are you sure it's in To?

If it's in the from there is already a way to add that. If it's the to...that's some option I could add, but it's very weird behavior of the server. The Display info is optional in SIP and when you call someone you don't know his display name before calling him... it's a little bit like if a email server would refuse an incoming email if the email is not in the form "Nice Display Name" ... Normally the only thing a server should take into account to route is user@domain.com and not the display name that could change from a client to another.

In the from it's different it could be expected by the server to be used as your caller id (BTW, that's the name of the field in the advanced wizard of CSipSimple).

(Side note, latest version are available from nightly build repository : http://nightlies.csipsimple.com/trunk/ , the market one is a little bit outdated).

So, are you sure it's what is required by your sip server? And what is this server implementation? And what are existing sip clients that comply that?

I can try to make a private build for you with that hard coded just to see it's really the root problem.
