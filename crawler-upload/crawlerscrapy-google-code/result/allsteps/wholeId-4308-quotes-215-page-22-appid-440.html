Comment #3

Posted on Dec 23, 2010 by Quick Hippo

Well, that's true there is this option too. But that's a little bit odd, and not well understood. I think that I'll change this one to keep GSM option in outgoing call chooser.

What I was thinking about is filter options (see http://code.google.com/p/csipsimple/wiki/FAQ#When_integrated_to_android,_I_don%27t_want_some_number_to_be ) What you want to do is just the inverse, but can also be managed the same way ;)

You have just to add a rule "Force call" "All matches" (all matches is maybe only available in nightly builds only, but else you can do a regexp)

It will force SIP call with this account.

However if your current data connection is not valid for a SIP call it will fallback to normal GSM call.

What I'd propose is to have a rule for the virtual account "Mobile" (that is now available in nightly builds), that allow you to "Prevent calls" with GSM.

Would be a clean and extensible way to do what you want. (cause it will for example allow you to do that only for numbers that match another rule).
