Ahah tu es français aussi :). Désolé j'aurai du le remarquer avant avec les entêtes des réponses :)

Pour le CALL_PRIVILEDGED en réalité, il n’apparaît pas dans la liste sur le téléphone car il n'est pas autorisé pour les applications tierces non signées avec la signature de la ROM. En fait elle ne sert uniquement si CSipSimple est produit dans une ROM qui le signe comme une application système. Sinon c'est ignoré et la permission n'est pas donnée à l'application. La permission qui permet à l'application d'appeler via le GSM c'est CALL_PHONE. Je vais essayer d'être un peu plus clair sur ce point dans le wiki. Il faut aussi que je vérifie dans le code si ce que j'avais fait pour être intégré à une ROM est toujours valable ou si ce n'est plus nécessaire.

Pour le WRITE_CALL_LOG ça a été introduit dans l'api 16 (qui correspond à android 4.1). Donc si ton téléphone utilise un android avant Jelly Bean, la permission n’apparaît pas car elle n'existait pas (et n'était pas nécessaire). Elle n'est donc pas donné à l'application.

Pour le RAISE_THREAD_PRIORITY, bien vu ! J'avais complètement oublié que je l'avais commenté dans le code :).

A propos de Chrome, en effet, je parlais bien de Chromium :). En général quand je parle de Chromium les utilisateurs lambda ne vois pas de quoi je parle donc.... Mais sinon en effet je n'utilise pas Chrome, et même Chromium je ne l'utilise pas trop. Je préfère de loin l'esprit de Mozilla. D'ailleurs si tu as un peu joué avec le html5/css3/svg tu t'es sûrement aussi rendu compte que Chrom(e/ium) est un peu développé à l'emporte-pièce (ils prennent souvent des raccourcis dans l'implémentation des nouvelles fonctionnalités). Ils vont vite mais ils ont plein de bugs alors que chez mozilla quand une feature arrive elle est vraiment valable de bout en bout. Par exemple, 'ai notamment vu des trucs très frappant sur les transitions css + le @font-face sur chromium quand ça avait débuté... complètement buggé. Alors du coup c'est sûr qu'ils scorent plus sur les trucs de tests bateau et sur les benchmarks... mais comme c'est fait juste en surface...
