csipsimple - issue #1348

Duplicate incoming calls from sip2sip.info

Posted on Oct 27, 2011 by Massive Elephant

What steps will reproduce the problem? 1. Register your sip2sip.info account with CSIP 2. Use a service like ipkall to forward calls to sip2sip.info 3. Call the ipkall number

What is the expected output? What do you see instead? We expect to see one incoming call. Instead, I see two.

What version of the product are you using? On what operating system? CSIP version 0.02-03 r944 on HTC Hero Android version 2.1

Please provide any additional information below. When I check the sip2sip.info logs, in the "devices" section, multiple CSIP devices are registered. I think there should be just one.

Comment #1

Posted on Oct 28, 2011 by Quick Hippo

There is two possible reason that can lead to this.

The first one is about the way pjsip registers server by default. It register once with private IP and then it unregisters the first registration and register with public ip obtained. Normally with server that are correct with sip rfc it should go fine, if not there is expert settings to workaround that (see the wiki FAQ). I think that sip2sip.info is compliant and should be fine regarding this point.

So it's probably about the second point which is much more tricky. When you lost a connection on a network, csipsimple can't obviously unregister (cause there is not the network available anymore so no way to communicate anymore). As in order to save your battery life, in this case it stop the sip stack behind. When it restarts, it justs connect again. If the sip server is not clever enough to detect it is actually the same client (most servers does not detect that), and if your IP address is the same, you will be registered twice with the same IP and may receive your calls twice. If it's a different IP, it has usually no impact on the way it will work. For this reason, there is a setting in csipsimple to avoid multiple calls. By default, it is set to true for this reason, in order to avoid to have two incoming calls at a time. Usually mainstream users does not really care about this feature.

If you enable this feature, you must be aware about what I've just explained, and that it is a possible backside of enabling this option, having a sip server not clever enough, and behing on a mobile connection that can stop and restart at anytime.

In newer release, there is some start to workaround parts of this problem. There is at least an option that allow to reset all registration if you run out of registration slot on the server (some sip servers only allow a limited number of registration and then just refuse newer registrations). The problem with that is that it will reset even non csipsimple clients registration (sip protocol does not allow to match per user agent, but only to wildcard registrations or to specify one exact registration). There is however, one theorical possible fix, however, this is very very complicated to do due the fact pjsip sip stack is done in native layer while android things are in java layer. This point is already opened in another issue and is already taken into the todo list ;).

Comment #2

Posted on Oct 28, 2011 by Massive Elephant

Thanks for the response. I managed to fix the multiple devices issue. I used the expert wizard configs as you said, and saw to my surprise that "contact rewrite method" was "legacy" by default! So I switched it to "Normal" and the extra devices disappeared on sip2sip.info :)

Now I just get one call.
