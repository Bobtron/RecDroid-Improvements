csipsimple - issue #603

"Hang Up" issue persists if the person placing the call hangs up before the other person answers

Posted on Jan 20, 2011 by Happy Wombat

What steps will reproduce the problem? 1. Make a call to another phone (or yourself) 2. Hang up before other person answers 3.

What is the expected output? What do you see instead?

The expected output would be that the call was terminated at both the originating and receiving phones simultaneously. What happens instead is that the call gets terminated at the receiving phone and the system acts as expected. After pressing "End" on the originating phone, the phone continues to display "Ringing" for another 31-32 seconds, though the ringing sound stops immediately. The app works perfect if the receiving phone terminates the call, whether or not they answer the call first.

What version of the product are you using? On what operating system?

I've replicated this error on both v0.00-16r410 and 0.00-12 (? the August 2010 release, I believe). We're using Axon Virtual PBX server on a Win7x64 PC. Both test phones are HTC Desire w/ Android 2.2.

Please provide any additional information below.

Comment #6

Posted on Jan 20, 2011 by Happy Wombat

I thought that was the "nightly-build phone". We're testing a system with four identical phones (htc desire), so sometimes I get confused as to which software version is on which phone.

There are four cases possible in a scenario where Phone A calls Phone B:

  1. Phone A hangs up before Phone B answers.
  2. Phone A hangs up after Phone B answers.
  3. Phone B ignores the call (hangs up without answering)
  4. Phone B hangs up after answering.

In case 1, we get the "hang up error" as I've described. In cases 2-4, both phones act as expected where the call is terminated as usual. The error we experience in case 1 can be replicated if either phone calls itself (Phone A calls Phone A or Phone B calls Phone B). This has been the case with v0.00-12, v0.00-16 (the current marketplace build) and with the latest nightly build that you sent a link for.

Other sip clients work fine. We could probably just use one of them, but the team and myself really prefer csipsimple's UI and sound quality greatly over sipdroid and the several others we've tried.
