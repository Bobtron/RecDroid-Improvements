Comment #3

Posted on Sep 17, 2013 by Quick Hippo

1) Developers should not use Windows 2) Developers should read the docs : http://code.google.com/p/csipsimple/wiki/HowToBuild and specially the sections with headline IMPORTANT. 3) The project is currently something I develop on my free time only and I've few time to support. That's why sometimes my replies are quick and short - it doesn't mean to be rude. I just prefer spending time to introduce new features/helping end users rather than teaching development. This is a personal choice, teaching informatics is also something that has a good impact on global people skills, but that's not something I'm good at and I do not take pleasure doing that. Besides, anyone with good linux background should be able to solve the kind of problem mentioned here easily (and it would not be a problem to modify a gnu toolchain steps too). That's why the HowToBuild wiki page indicate to post to dev mailing list where other devs can help on these kind of topic (and maybe when you'll get more skilled, you'll be able to help others). Also, reading the docs and searching by yourself will also help. The problems I see in this issue is not something really related to csipsimple but to some misunderstood of java/jni/c development. So any forum where there is android/jni developers can help to find the root cause of the problem. I know that Windows users are used to not understand how things works, But everything has a reason in informatics, and reading/understanding the error message will make you progress way more than getting a solution that you'll apply like a not understood recipe.
