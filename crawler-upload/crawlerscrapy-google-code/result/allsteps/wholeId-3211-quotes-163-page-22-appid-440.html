csipsimple - issue #1633

Ringer stays silent if a mobile call is answered while a sip call is ringing

Posted on Mar 21, 2012 by Happy Lion

What steps will reproduce the problem? 1. I don't have the dialer integrated. An inbound SIP call rings and I receive an inbound mobile cell call while the SIP call is still ringing. 2. I Answer the mobile call and not the SIP call. 3. When I finish the mobile call and hang up, the ringer is silent and will stay silent no matter what unless I reboot the phone OR make/receive a SIP call. The ringer is then activated again when I hang up from the SIP call.

What is the expected output? What do you see instead?

I expect the ringer to be activated when the SIP call times out, is dropped or the call goes to voicemail.

What version of the product are you using? On what operating system? I have tried it on several versions of Gingerbread with the same result. I'm using csipsimple version 0.03-01

Please provide any additional information below.

When testing I found that if I hit end on the SIP call before the mobile call starts ringing the ringer does not remain silent. Of course this does not help since I can't know when a mobile call will be received while a SIP call is ringing. I'm in a situation where I can receive multiple calls simultaneously, both SIP and mobile and I had a hard time figuring out why my cell phone would suddenly go to vibrate unexpectedly and permanently.
