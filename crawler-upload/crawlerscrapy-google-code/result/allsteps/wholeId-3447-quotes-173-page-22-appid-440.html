csipsimple - issue #1350

Cannot Stop csipsimple from running

Posted on Oct 28, 2011 by Quick Panda

What steps will reproduce the problem? 1. Change network settings so csipsimple shouldn't start 2. Uninstall csipsimple in market 3. Delete all csipsimple*.apk files downloaded 4. Restart phone 5. csipsimple is still listed in running applications

What is the expected output? What do you see instead? My Droid2 has been resetting itself so I have been uninstalling apps to be sure they are not contributing to the instability. I thought I would be able to stop csipsimple from starting without driving a wooden stake through the phone!

What version of the product are you using? On what operating system? 0.02-03 on Android 2.3.3 System 4.5.601.A955.Verizon.en.US

Please provide any additional information below. I have been trying various versions of csipsimple but the one that was persistent is the one listed above.

Comment #1

Posted on Oct 28, 2011 by Quick Panda

In the process of looking things up on this, I discovered that if you go to

Home Menu --> Settings --> Manage Applications --> csipsimple

there is a button to uninstall the app from there. That button seemed to actually work.

Comment #2

Posted on Oct 28, 2011 by Quick Hippo

Yes, this is the good way to uninstall an app installed with an apk that does not comes from the android market. The "android market" can only handle (uninstall) apps that comes from the android market. If it comes from elsewhere (an apk downloaded from a website), you have to use the android OS application manager that is in settings, and not the "google android market" (this market is just an application on the top of android ;) ).

Comment #3

Posted on Oct 29, 2011 by Quick Panda

I am curious, however -- SHOULD csipsimple keep running after a reboot when the network settings don't make it active? Is this unavoidable?

Comment #4

Posted on Oct 29, 2011 by Quick Hippo

No it should not start if network settings does not make it available for incoming on the current network.

However, you should be aware about what "running" means on android. The fact that the app is listed in the tasker apps does not mean it is actually running. Many users are confused with these apps that list other apps loaded in memory by android OS. That's why I precise ;).
