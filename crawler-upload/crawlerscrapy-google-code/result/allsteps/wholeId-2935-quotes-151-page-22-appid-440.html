Posted on Aug 21, 2012 by Grumpy Rhino

What steps will reproduce the problem? 1. css retries registration several times, where 3G is weak 2. css stops registration with errors such as bad gateway or request timeout 3. css and I also can not recover registration from the fails until restarting css after killed.

What is the expected output? What do you see instead?

css should recover it automatically, and also I want to recover it manually and quickly.

What version of the product are you using? On what operating system?

r1804 with Galaxy S2 and CM10

Please provide any additional information below.

Although I tried taking logs, the size of log files leaves 0B. Taking log through the help dialog does not seem working. All other log files are also deleted.

Comment #1

Posted on Aug 22, 2012 by Quick Hippo

Normally it retries each 60 seconds to reconnect after a failure (even a timeout failure). It also indeed retry during the registration process.

Some tip, if you want to restart teh entiere stack, go in Settings and press back.

Normally a network change should also trigger restart of the sip stack.

About logs, there is an alternative way to the help dialog way. To use it switch to ExpertSettingMode (see wiki page). Then in user interface section, turn logging level to 4 and use log to file option.Reproduce the problem, and once done, go in settings and press back (so that it flush the file). The log file will be produced in sdcard, in csipsimple folder in logs sub folder. Logs recorded to file have "pj_" prefix in name.
