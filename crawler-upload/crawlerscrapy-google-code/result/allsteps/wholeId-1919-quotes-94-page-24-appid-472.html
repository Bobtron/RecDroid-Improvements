Dialog Fragment is added to Activity stack on device orientation change

Posted on Jun 21, 2012 by Grumpy Bird

What steps will reproduce the problem? 1. Open the application 2. Type data in dialog 3. Rotate the device's orientation, allowing screen to rebuild 4. Notice the dialog is rebuilt, without the entered text 5. Press back

What is the expected output? What do you see instead? The entered text is expected to remain during orientation changes, and for the alert to dismiss after pressing back, instead of showing the previously spawned dialogs.

What version of the product are you using? On what operating system? ICS, in simulator

Please provide any additional information below. I know this is just a sample on how to build dialogs with fragments, but it would be great to show an example that correctly handles orientation changes. The solution is to not always fire the dialog up in onCreate, and properly call back when the view is dismissed instead of submitted. Right?
