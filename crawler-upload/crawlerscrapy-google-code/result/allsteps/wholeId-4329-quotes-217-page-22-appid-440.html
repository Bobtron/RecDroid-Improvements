csipsimple - issue #470

sRTP mandatory only works outbound - sRTP optional only works inbound with Asterisk 1.8

Posted on Dec 8, 2010 by Grumpy Dog

What steps will reproduce the problem? 1. Install CSipSimple-r416-trunk.apk on Droid 2 (Android 2.2)

  1. Register to Asterisk 1.8 with sRTP support (libsrtp 1.4.4) compiled in.

  2. Only outbound calls work with sRTP mandatory setting. Inbound calls are rejected with a SIP 406 error (Asterisk error message): -- Got SIP response 406 "Not Acceptable" back from x.x.x.x:5060

  3. Only inbound calls work with sRTP optional setting. Outbound calls terminate with CSipSimmple hangup. Although CSipSimple is registered with Asterisk, no error is reported on the Asterisk console.

What is the expected output? What do you see instead?

What version of the product are you using? On what operating system?

Comment #1

Posted on Dec 8, 2010 by Quick Hippo

Your SIP server announce SRTP for incoming calls? (It can support srtp but do not announce SRTP in invites)

If not and pjsip (csipsimple) set to mandatory, it will reject the call since SRTP is mandatory for call.

If you send me logs of incoming calls I can help you to detect whether SRTP offer is done by your Asterisk. I'm not an expert at Asterisk configuration so unfortunately I can't help you more on that conf point, but as it is described sounds simply to be due to the fact Asterisk do not offer SRTP and so as CSipSimple is configured to use SRTP as mandatory for calls it refuse the unsecure incoming call.

Comment #14

Posted on May 17, 2011 by Happy Cat

I too have come across this issue. I have been debugging the root cause of this error. CSipSimple yield the following logcat message:

05-17 12:33:18.842: ERROR/libpjsip(7319): 12:33:18.845 pjsua_call.c Error initializing media channel: Invalid SDP media transport for SRTP (PJMEDIA_SRTP_ESDPINTRANSPORT) [status=220227]

I've tracked the source of this down into pjsip in the following location: transport_srtp.c in pjsip/project/pjmedia/src/pjmedia. Specifically, line 1190:

1185 /* Validate remote media transport based on SRTP usage option. 1186 */ 1187 switch (srtp->setting.use) { 1188 case PJMEDIA_SRTP_DISABLED: 1189 if (pj_stricmp(&m_rem->desc.transport, &ID_RTP_SAVP) == 0) 1190 return PJMEDIA_SRTP_ESDPINTRANSPORT; 1191 goto BYPASS_SRTP; 1192 case PJMEDIA_SRTP_OPTIONAL: 1193 break; 1194 case PJMEDIA_SRTP_MANDATORY: 1195 if (pj_stricmp(&m_rem->desc.transport, &ID_RTP_SAVP) != 0) 1196 return PJMEDIA_SRTP_ESDPINTRANSPORT; 1197 break; 1198 }

What's very strange about this is that the pjmedia layer believes that SRTP is disabled even though I've set the SRTP mode in the user account as "Mandatory". Something isn't being configured or passed down into the lower pjsip shared library.

Stay tuned...if/when I find more, I'll post it here.
