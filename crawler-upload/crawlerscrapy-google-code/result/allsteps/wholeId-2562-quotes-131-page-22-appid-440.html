csipsimple - issue #2246

Program loses its "SIP registration" after long periods of time or change in location and doesn't re-register

Posted on Feb 15, 2013 by Helpful Rabbit

What steps will reproduce the problem? 1. Have CSipSimple running (I'm using it exclusively as my SIP Client) 2. I leave my house (I only use 4G so it's not a WiFi issue) or I go to sleep 3. Observe phone after either case

What is the expected output? What do you see instead? I would expect that the phone would re-register the SIP automatically. If I click on CSipSimple (assuming it's never been "turned off" (i.e. task killer, RAM clear), it still will not re-register the SIP. As a result, I sometimes miss many calls. The ONLY way to get it to register is to hit the menu button (while in the program), hit disconnect, and then re-open the program and it registers just fine.

What version of the product are you using? On what operating system? 0.04-05 r12124

Please provide any additional information below. I would like to find a way that CSipSimple will start and will not be closeable (just like a core app). I have converted it to a System app using Titanium Backup Pro but it still closes down during a RAM clear. I have purchased a program called Memory Booster Pro where I added CSipSimple to its "whitelist". I changed the settings in the program to "balance" so it isn't insanely aggressive when closing apps, as when I had it on aggressive it even killed programs on the whitelist (there might be some sort of override in the aggressive mode that even allows it to kill select whitelist apps). I'm unsure how effective this really is, but if someone has an alternative means preventing this program from closing, please let me know!
