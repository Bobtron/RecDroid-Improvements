csipsimple - issue #1482

"Call directly" filter doesn't work for GSM

Posted on Jan 9, 2012 by Grumpy Rhino

What steps will reproduce the problem? 1. Set a filter to a SIP account (e.g. if the number is exactly **1, call directly with the Freephonie account). 2. Set a filter for GSM (e.g. if the number is exactly 123, call directly using GSM).

What is the expected output? What do you see instead?

Dialing **1 will automatically pass the call using Freephonie. But when dialing 123, the CSipSimple windows shows and asks wich provider to use, but doesn't call automatically 123 through GSM.

What version of the product are you using? On what operating system?

r1176 on Gignerbread 2.3.5 (Galaxy S2)

Please provide any additional information below.

It is a new issue I get with the lastest releases. One week ago it was working fine (I check for updates everyday). I didn't check if the issue occurs with other filters such as Betamax or Skype handler.

Thanks.
