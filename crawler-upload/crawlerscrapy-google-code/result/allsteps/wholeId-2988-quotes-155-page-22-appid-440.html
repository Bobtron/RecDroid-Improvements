Comment #1

Posted on Jul 4, 2012 by Quick Hippo

Replied in private mail, but for reference if anybody else search on the issue list :

Did you try to activate STUN option as described in this entry of the FAQ ? : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#The_other_party_can_hear_me_but_I_can_not_hear_them

Apparently the wifi network you use is NATed so there is something needed to resolve "public" IP address of the phone and send it to remote side so that remote side media can reach you. Take care however, the default STUN server is located western public IP network, so depending on where your sip server/provider is located and whether the contry you are in has a separated network the "public" IP address resolved by the default STUN server may be wrong for your sip server. If so you have to search a STUN server in the same "public" network than your sip provider :)

Comment #3

Posted on Jul 4, 2012 by Quick Hippo

About call quality, you can try to play with codecs options. For example PCMU/PCMA aka G711 is a legacy codec that is supported by all sip providers but uses a lot of bandwidth (64kbits). CSipSimple supports many other codecs that it could be interesting to try/force if supported by the sip provider. (long press a codec row to activate/deactivate it). You can also check the codec in use while in call by pressing the little arrow under the call card and the Info button.

You can also try developement version that contains some user interface improvement and bug fixes, but in case of bad quality it's more likely something with codec in use vs network capability.
