Comment #3

Posted on Jun 13, 2011 by Quick Hippo

What kind of connection are you using? Is that 3G or wifi?

Low bandwidth (3G/GPRS/Edge), need to use a codec optimised for low bandwidth. PCMU and PCMA are the worse codecs (cause actually do no compression at all). iLBC and g729 should be better regarding data usage and have a good quality (however iLBC use many CPU and g729 is proprietary. GSM use few bandwidth but provide a not so good call quality (as worse as on real GSM -- it's the same codec used ;) ) speex@8kHz should not use too much bandwidth but is not the best choice (speex 16 and 32 are good if you have a high speed connection) silk codec (it is the one that Skype use) has very good results @8kHz with low bandwidth. g722 (only if you have a really high speed connection, but call quality is very good if so).

The important thing in all this list is whether your sip provider does actually support this codec or not ! Cause if they do not unfortunately, there is few thing a sip app can do ;).

Comment #6

Posted on Jun 16, 2011 by Quick Hippo

I'm surprised that one should manually set the codecs in accordance with the sip provider (I assume this is pbxes.org in my case and not the trunk-sip provider pbxes routes the call to), I would have thought SIP offers negotiation capabilities to get the best codec.

Actually it should. The default order in CSipSimple, for low bandwidth (narrowband) and wideband should be good as far as the remote side manage it correctly. Unfortunately some providers / sip gateway does not manage some codec implementation very well and as consequence even if the application does it best to announce codecs in the best order and that negociation succeed, if the other side do weird things with the codec... there is few things the app can do.

Besides you talk about the fact you use pbxes.org as gateway to another sip provider. Did you try to directly connection to the sip provider without using pbxes.org? I know that sipdroid (developped by pbxes.org) usually advise to use pbxes.org, but that's not necessary, SIP is a standard and a sip client is able to connect a sip provider without using pbxes.org.

(However pbxes.org provides other cool features, so maybe you did this choice being aware about that pbxes.org is not required ;) ).
