Comment #1

Posted on Oct 25, 2012 by Quick Hippo

Hi,

Maybe these sip providers has limited number of contact that can be registered at same time. In case of switch from wifi to 3g, the app will lost brutally connection (when loosing wifi) and will not be able to send the unregister cleanly. Some sip providers have limited number of possible concurrent registration when when new one comes from one user instead of dropping oldest one they raise an error.

A workaround to this situation is to activate the "Try to clean register" option. You can also try to reduce registration timeout value (so that old not unregistered accounts will timeout naturally quicker).
