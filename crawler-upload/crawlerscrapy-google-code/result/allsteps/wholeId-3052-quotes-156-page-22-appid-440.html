Comment #1

Posted on May 29, 2012 by Quick Hippo

Hi,

It could be very interesting to play with audio workarounds in your case.

Motorola is known to contextualize audio frequency by audio routing mode. They had a very big bug in the first version of the Atrix. So it could go the same on this device.

So, my first thought, is to try to change clock rate to 8kHz instead of 16kHz. To do so, go in csipsimple settings, press menu key of the phone, and select "Expert mode". Then, go in Media section of the settings, and change clock rate/frequency from 16kHz to 8kHz.

Then place a call and see how it goes.

There is other things you could try if this one doesn't help. See the http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting FAQ entry for more details on the possible audio workarounds.

Anyway, keep me in touch of your findings. If you find some working options, I will be able to auto-detect the photon 4G, and to activate correct audio workaround settings automatically in this case. Which will be a great improvement for all owners of this device.

If you have time, you can also test latest nightly build : http://nightlies.csipsimple.com/trunk/ It's the current development status, and I appreaciate any feedback I can get on new devices with this version since it's the next one to be released on the android market :).

Thanks!

Thanks for the reply. See below for results:

Using "WebRTC" solved issue. See below for complete test.

I tried changing from 16Hz to 8Hz. No difference, that is same problem (still bad sound out of earpiece, great sound out of speakerphone)

Installed CSipSimple-latest-trunk.apk from http://nightlies.csipsimple.com/trunk/79 version now says 0.04-00 r1579

Tried changing back from 16Hz to 8Hz. Same problem.

Switched back to 16Hz.

Switched to Expert Mode

Went into Media settings

Turned on "Use Mode API" - couldn't even make call.

Turned off on "Use Mode API" - could now make call but same problem.

Turned on "use routing api" - same problem.

Turned off routing api and turned on use tone hack - same problem.

Turned off tone hack and turned on Use WebRTC implementation - problem SOLVED.

Turned off Use WebRTC implementation - problem returned.

Also, wanted to mention that I also experienced the problem when using the "Audio Test" option - would hear myself choppy (unless I had "WebRTC" turned on).

Comment #6

Posted on May 31, 2012 by Quick Hippo

Thanks for the logs.

WebRTC is a project from Google that aim to bring voice/video over IP directly available in browser. As part of this project they release a lot of media library. I reuse some of these libraries in csipsimple (echo canceller, noise suppressor, some codecs). I've also contributed some patch for android while finding issues.

As they also have some starting thing for android, they have also written a glue with the audio device that setup audio routing. As it's google and they know how manufacturers does things there is a very interesting way to setup audio routing that I've ported to csipsimple. It doesn't work with all devices however, but it usually works well with manufacturers with close relation with google (samsung and motorola). http://www.webrtc.org/
