csipsimple - issue #2150

Multiple accounts but only the last one registered will work

Posted on Dec 27, 2012 by Happy Bear

What steps will reproduce the problem? 1. I have two pbxes accounts, I set them up in CSipSimple 2. Activate both accounts. 3. Try an outgoing call, only the last one registered/activated will work, the other one will throw a 407 error. 4. Deactivate the both accounts and then reactivate them making sure the one that failed previously is the last one registered. 5. Now the account that previously worked will fail and the one that previously failed will work

What is the expected output? What do you see instead? Regardless of which account got logged in/registered/activated, all accounts should function.

What version of the product are you using? On what operating system? Nightlies versions Dec 25, 26 and 27. TMobile Galaxy S3 SGH-T999 running 4.1.1 android.

Please provide any additional information below. Currently I have to run the second account through another sip client such as sipdroid.
