Comment #1

Posted on Dec 20, 2011 by Quick Hippo

Did you manually disable stun server? It could also be because ekiga stun server is down. You can maybe try another public one. For example stun.ippi.fr In fact you get this problem because ekiga does not allow to register with a private ip. Normally stun is there to know your public IP in public network. this operation is not really linked to the provider, that's why you can try with the stun server of another provider. By default the ekiga wizard setup ekiga stun server because of the fact ekiga restrict register to public IP. But possible that there is currently a problem on this server. Normally if the stun resolution goes well ekiga will not cry anymore

Comment #4

Posted on Dec 21, 2011 by Quick Hippo

The stun server of ekiga is DNS alias for another stun servers. So maybe sometimes it load balance on invalid one. And indeed seems that there is some problem with counterpath's one.

So try to remove all but the one of ippi.

By default to make things easier for users stun.counterpath.com is added (many users doesn't know what stun is about and I want to simplify their life as much as possible ;) ). Then if there is a setup from a different wizard the stun server of the wizard will be inserted in the list.

Normally the sip stack behind csipsimple should try all stun server configured but I'm not sure if it goes well in all error cases (maybe a dns error on one of the stun server can completely break stun things).

Comment #7

Posted on Dec 23, 2011 by Quick Hippo

For the record on the issue list after private mails ;) : to be linked to issue 345.

The problem was that a first account did a register to a local account previously to the one to ekiga. As consequence, pjsip detected its public ip to be the one of the phone in the LAN area and did the registration using this IP. The clean way to solve this kind of issue is the one proposed in issue 345. However it should be possible to try to workaround by disabling contact rewrite feature on first account (maybe need to force contact header on this account) Maybe reordering account priority could help for first registration too, but I do not advise this method cause could be broken in next registration.
