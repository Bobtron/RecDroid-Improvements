csipsimple - issue #710

Manage ANR exceptions for in Call state

Posted on Feb 12, 2011 by Helpful Panda

What steps will reproduce the problem? 1. Call ends via glitch or via some other non-normal way

What is the expected output? What do you see instead? I expect silent mode turned back off whenever the call ends no matter how it ended. Instead, silent mode remains on.

What version of the product are you using? On what operating system? 0.01-01 r615

Comment #1

Posted on Feb 13, 2011 by Quick Hippo

It has been discussed in another issue.

First thing is that you should never have ANR (freeze of the app while in call). Else it mean that there is a bug and this bug should be eliminated. When do you get it? What is your device?

Then if you terminate the call by killing the app (whether ANR or using a task killer app), there is unfortunately no ways for me to manage that. (It make sense, you kill the app, the app can't do anything then)... So there is no ways for me to restore previous settings.

Unfortunately it means that ringer setting will remain to vibrate and also (and that's worse cause users are not aware about that), the wifi sleep policy will go to "never sleep". As it absolutely necessary to set these settings while in call cause else you'll experiment worse communication and that there is no way for me to handle crashes; I would prefer treat the origin of the problem. If that's an ANR could you provide me more info about a scenario to reproduce. Maybe your device has the PSP behavior, (read the last entry of the FAQ to know more about that).

For ANR / Forceclose (I don't think that's a force close you are experimenting), there is maybe something I could do. But that's tricky to do. I would prefer by far fixing the root of the issue ;).
