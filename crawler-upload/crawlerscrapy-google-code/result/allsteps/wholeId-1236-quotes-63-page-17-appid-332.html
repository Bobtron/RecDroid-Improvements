android-on-freerunner - issue #81

Stressing SD causes SD lockup

Posted on Nov 23, 2009 by Happy Bear

What steps will reproduce the problem? 1. Find a big enough file (20 megabytes or so) 2. Copy it to SD over USB: adb install the_big_file /sdcard/ 3. Observe the copy getting stuck and the SD locked up. Note: the SD is most likely is corrupted at this point, so be prepared for fsck.vfat motion.

What is the expected output? What do you see instead? Copy the file completely.

What version of the product are you using? On what operating system? android-on-freerunner-cupcake-week-2009-44.tar.gz

Please provide any additional information below. I am not sure the issue is with the android-on-freerunner as it can be related to the microSD card I am using (SanDisk 8GB). If anyone can reproduce the bug using other microSD cards, then write here please.

Comment #12

Posted on Dec 7, 2009 by Quick Rhino

I tried to change the init.rc, but it is regenerated each time on boot! And there is no note that the file is autogenerated...

Problem is that root part ( to which init.rc included ) is a part of kernel.img. To change init.rc you should rebuild kernel. Same problem with SD-based subj. To be able change init.rc I have to rebuild kernel without this root part. And then it is simple to put ramdisk.img to SD for SD-based subj. For NAND-based subj for change init.rc you should rebuild kernel with new version of init.rc.

I does not know a way to start "user-modifiable script" without rebuilding kernel or init ( not init.rc ).
