Comment #5

Posted on Feb 24, 2011 by Quick Hippo

Yes the directory is correct. But anyway you should read the android NDK documentation. Always good to know this kind of stuff ;).

As for android 1.5 to 2.3 ... everything in CSipSimple is already compatible. Do not confuse between the target SDK and the actually compatible SDK version. CSipSimple is compatible from android 1.5 even if target sdk is 2.2. (There is only the gingerbread edition available in nightly builds that is only compatible with android 2.3, all others are fine with 1.5 and upper).

On emulator as I've just added on the wiki, you'll experiment a very bad audio quality. I do not advise to use it using emulator. With Qemu audio quality will be awful and besides it will pledge the CPU. So do not test audio.

Comment #7

Posted on Feb 24, 2011 by Quick Hippo

As for ffmpeg, just for information, already exists (done by other guys) and I re-used it on the video branch. (Could save your time ;) ).

As for library produced, when you'll read the ndk documentation you'll know more about the android arch.

You'll see that for now there is armeabi and armeabi-v7a. That armeabi is compatible with armeabi-v7a device but that it's better to have an optimized version for v7a device (specially for floating point support). The NDK re-distributed in CSipSimple also allow to build for amrv4t. The android ndk also ~should~ allow to build for x86. But never tried it... for now I've no device running this arch. Maybe I should try with the vm x86 port of android.... But anyway, nobody asked for that arch for now... I'll have a closer look if anybody ask.
