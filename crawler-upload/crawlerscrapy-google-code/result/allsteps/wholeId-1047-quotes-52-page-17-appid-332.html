android-on-freerunner - issue #152

Installer branch kernel outdated

Posted on Sep 2, 2010 by Happy Cat

What steps will reproduce the problem? 1.Get installer branch and look at kernel code

What is the expected output? What do you see instead?

Expect to see same kernel code in master, cupcake and installer. Found versions not to be in sync while verifying the glamo-fb.c code looking for the values of GLAMO_REG_LCD_A_BASE2 as suggested in a bug-fix by Paul Fertser and Thomas White.

Cupcake and master contain GLAMO_REG_LCD_A_BASE2, 0x4000 , but installer still contains GLAMO_REG_LCD_A_BASE2, 0x0000. This makes me believe the kernel of the installer has not been updated.

Comment #1

Posted on Sep 2, 2010 by Happy Rabbit
