csipsimple - issue #560

wifi disallows connection over 3G/EDGE/GPRS

Posted on Jan 2, 2011 by Swift Bear

What steps will reproduce the problem? 1. Network settings are set to use 3G, EDGE, GPRS, and Other Networks but NOT wifi 2. Wifi is OFF
3. Registered with voip provider (3G) 4. Turn wifi ON and we get de-registered with this message:

"Selected Network route not available (Activate wifi or change preferences to use 3G)"

  1. Turning off wifi and we get re-registered.

What is the expected output? Setting wifi to NOT be used should not disallow a connection over 3G/EDGE/GPRS when a wifi connection is detected.

What do you see instead? Cannot connect over 3G/EDGE/GPRS.

What version of the product are you using? On what operating system? 0.00-16 r410 on android LG-P500

Please provide any additional information below. If I allow connecting over wifi everything works as it should.

Ok two points here : 1- the label is not correct you're right. I'll fix that :)

2- However, there is something you probably don't know : in android you can't have both a gsm data connection running at the same time than a wifi connection. If you know a little bit about technique you can't have two default gateway at the same time on a device. Besides there is very few OS where this kind of dual connection is possible (unless to be an expert). For example on Windows when you connect both wifi and wired ethernet, you'll see that only one of your connection is used at the same time. And in windows it's really not user friendly in this case cause you don't know which is used ;) ). On Linux, as you can do everything you want you can choose more finely routes but it requires to know exactly what you are doing and a deep understood of network routing.

On android, the guys that have decided the design of the platform choose to give the priority to the Wifi connection (which make sense on a mobile). As consequence on android you can't have at the same time Wifi AND 3G/GPRS/EDGE etc. As soon as wifi goes up, 3G connection goes down. 3G or any data packets is different from GSM voice packets (packet switching vs circuit switching). So you keep circuit switching and data switching of you gsm connection becomes unavailable for all applications on android !! You'll not find any application that can do what you'd like to do unless it is a modified ROM and something that require to really be a true network expert allow you to change the default gateway for some subnetwork. You have probably noticed but when wifi turns on the little 3G or H or G or E sign on the notification of your android disapear and is replaced by the wifi. It exactly mean that your PS (packet switching) of your mobile is turned off !!

So the core of your issue is not a problem with CSipSimple but a misunderstood of how networks works (on Android).

If you'd really like to have this feature, you should have a look if there is not a custom ROM that allow that. But I guess that you'll not find one : modifying this will work with no android application and it will require skills in network configuration.
