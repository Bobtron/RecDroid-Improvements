csipsimple - issue #799

Detect wrong config regarding dialer integration in 2.3 and report it to user

Posted on Mar 13, 2011 by Happy Ox

What steps will reproduce the problem? 1. Place call through native dialer. 2. Receive warning: 'No Internet Calling Account'. 3. Cancel call with no other option available.

What is the expected output? What do you see instead? I would expect native dialer integration with CSIPsimple. The only way known is to dial directly while within CSIPsimple.

What version of the product are you using? On what operating system? Using nightly version r700_ginger and r700 build. CM Nightly, cm_vision_full-18, Gingerbread 2.3.3

Please provide any additional information below. The native dialer defaults to GSM calling or Internet Calling. CSIPsimple is completely ignored as an option with the native dialer.

Comment #3

Posted on Mar 14, 2011 by Quick Hippo

So I guess you have already use the "Remember my choice once" but choosing the stock sip app ;) (which unfortunately is named "dialer" ... ).

To reset this choice : Go in "android settings" > Applications > Manage applications > All > Dialer (the name may depend on your rom but search for application named dialer or telephony). When you are on the screen I bet the Clear default action button can be clicked. Click on it. It will reset your previous choice as default action for outgoing calls.

If it does not help, things to check : * In CSipSimple settings > easy settings : check that integration with android is activated * In android settings > call parameters > Call using internet > choose Only for internet calls.

I'll recycle this issue to include inside the CSipSimple UI a big warning when a bad configuration is detected and that explain how to solve it.
