csipsimple - issue #1154

While making calls CSipsimple / Keyyo / Sonetel showing Complete action using dialog

Posted on Jul 15, 2011 by Grumpy Panda

What steps will reproduce the problem?

  1. Disable integrate native dialer.
  2. Register with a user.
  3. Try to make a outbound call
  4. A dialog box will come and ask by showing installed voip clients.
  5. If i have only sonetel it's not showing this. If i install
    CSipsimple or Keyyo along with sonetel then this problem coming.

What is the expected output? What do you see instead?

I guess if we disable integrate native dialer it should not ask for select dialer. But still it's asking. If i make call it should not ask which dialer we should use for the call.

What version of the product are you using? On what operating system?

Csipsimple from market: 0.02-03 r944 Mobile and OS: Samsung Galaxy POP - GT-S5570, Android 2.2.1

Please provide any additional information below.

I have tested almost three different phones (HTC dezire, HTC nexus and Samsung pop). All the phones i am facing this same issue. I am redistributing CSipsimple for Sonetel. For Sonetel if user registerd call should go only through Sonetel. It should not ask other dialers to use. Please help me whether this is this a issue or some usability/configuration issue from my side.

Thanks & Regards,

M. Yuva Technical Lead, Sonetel software services limited

Comment #1

Posted on Jul 15, 2011 by Quick Hippo

Since all applications are based on the same source code, what you observe is absolutely normal. There is three applications that are able to do exactly the same thing. So android ask you what app you'd like to use.... just normal !! It does exactly the same when there is several mail clients installed on the same phone.

In fact, when android detect that several apps can handle the same "intent" it ask for the user to choose. If you install several web browsers, several home applications, several email application, you'll observe the same "intent" chooser view (note it allows user to remember his choice). This is absolutely not a bug, and this is the way android works !

Besides, CSipSimple is made to cover needs of users that need sonetel / keyyo / ippi and so on... So if one needs not only your service, just advise them to install only one app !! Depending on the CustomDistribution.java you make on your app this app can be yours or CSipSimple if you decide to prevent your users from creating other accounts within your redistribution.

Please think about users first. Keep in mind users are your goal !

If your users download your app they'll not have any problem and it will automatically choose Sonetel application. If they start to install another sip application it means that they need another account (probably...). So either your redistribution allows that (and they do not need to install csipsimple or another sip application), either, let them use CSipSimple instead of your app and uninstall your app. Sonetel wizard is included in CSipSimple... Besides, due to the fact csipsimple is released under GPL, both apps should have same feature level... so not hurting for a user.

So what you observe is just normal, and I really don't think that's a problem at all.
