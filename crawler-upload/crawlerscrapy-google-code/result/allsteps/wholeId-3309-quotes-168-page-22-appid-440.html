csipsimple - issue #1533

Allow to configure TCP/UDP ka interval

Posted on Jan 23, 2012 by Grumpy Rhino

What steps will reproduce the problem? 1. Setup sip account 2. Register to asterisk 3. Monitor on the asterisk (sip set debug, sip show peer)

What is the expected output? What do you see instead?

Send registration request and keep-alive packet periodically.

What version of the product are you using? On what operating system?

r1235-trunk on Galaxy S2 (android 2.3.6)

Please provide any additional information below.

I tried SIP/UDP, TCP, and TLS. They all successfully registered at the first time. However, the registration will expire on the server, since csipsimple never send keep-alive packet and re-registration request.

With csipsimple0.03-01, SIP/UDP send re-registration and KA packet periodically. (However, registration request with old CSeq seems to be doubled.) SIP/TCP send re-registration periodically, but KA is not sent.

Thank you very much for providing the great application!

Comment #6

Posted on Feb 11, 2012 by Massive Monkey

I took a look at the pjsip stack. In sip_config.h there's a #define that controls (at least to some extent) the TCP KA interval - PJSIP_TCP_KEEP_ALIVE_INTERVAL.

I believe that by defining this to something sensible (e.g. 900 (not 90) seconds) in the correct place (pjsua.i????), it should be possible to build the stack without needing to patch it, and have an interval that doesn't kill the battery [even better would be to use TCP-level keep-alives, but that would require patching the source]. See sip_transport_tcp.c for where PJSIP_TCP_KEEP_ALIVE_INTERVAL is used.

A fully configurable KA interval would be great. A hard-coded 900s KA interval would at least be a huge improvement over the unnecessarily small, battery killing 90s default interval.

Anyone (like myself) that uses SIP over TCP would be really grateful for anything that can be done in this area. I'm stuck on an old version of CSS because of it.

from sip_config.h: /** * Set the interval to send keep-alive packet for TCP transports. * If the value is zero, keep-alive will be disabled for TCP. * * Default: 90 (seconds) * * @see PJSIP_TCP_KEEP_ALIVE_DATA */

ifndef PJSIP_TCP_KEEP_ALIVE_INTERVAL

define PJSIP_TCP_KEEP_ALIVE_INTERVAL 90

endif
