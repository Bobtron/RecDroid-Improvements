csipsimple - issue #231

TCP Problems with the new stack

Posted on Sep 20, 2010 by Happy Horse

What steps will reproduce the problem? 1. Enable WiFi and 3G in network settings for both incoming and outgoing 2. Create an Advanced Wizard account for UDP 3. Turn on WiFi 4. Enable the account, see Registering, Registered 5. Disable the account, change it to use TCP 6. Enable the account

What is the expected output? What do you see instead?

Expected: Registering, Registered Found: Registering, a long delay, then failed to register

Please use labels and text to provide additional information.

I watched this with an ethernet tracer (Microsoft Network Monitor 3.4, free). It showed the phone repeatedly registering with TCP, getting the 407 Auth Required, then registering again, getting 200 OK, then registering again, 407 Auth Required, register again, 200 OK, register again... etc. for 56 times in 10 seconds. This triggered a security lockout on my 3CX V9 SIP switch and took down our outside connection. This is really difficult for me to test in more detail as it affects our live system here which we use for our phones :-)))

Normally only one cycle of 407 then 200 is needed and the phone is registered. This is definitely new with the version that has the new SIP stack.

I've also noticed other strange things, and haven't the time to follow through right now: For example, if TCP is disabled in Network settings, but an account is selected for TCP, it tries with TCP anyway, and you can also see a UDP register/OK. Again, I can't be more specific, but they seem to have appeared with the new stack.

I apologize for not having the time to test TCP after the SIP stack update a week ago. I was so busy with my regular work. I can send you a trace file if you happen to have the (free Network Monitor 3.4).

UDP seems to be working fine on WiFi and 3G. The troubles are with TCP.
