csipsimple - issue #2960

Wrong port in contact URI

Posted on Jul 28, 2015 by Happy Rabbit

What steps will reproduce the problem? 1. Register to SIP server with TCP transport 2. Drop TCP connection 3. Wait for reconnect

What is the expected output? What do you see instead? CSipSimple should reregister with the same port, the TCP session is initiated. The contact URI uses the old port, before the connection was dropped. Incoming calls try to connect to the old port and fail.

What version of the product are you using? On what device / operating system? CSipSimple_GT-I9100G-17/r2459 on Android 4.2.2 Latest nightly build exposes same behavior.
