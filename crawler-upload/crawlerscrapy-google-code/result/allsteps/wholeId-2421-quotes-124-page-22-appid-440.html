Comment #1

Posted on May 15, 2013 by Quick Hippo

As stated by this log it's not a bug of CSipSimple.

It's a bug in your rom stock OS, in some component "BatteryStatsImpl" that is absolutely not inside csipsimple.

In your logs, the only things produced by csipsimple is :

05-15 08:16:22.052: E/libpjsip(873): 08:16:22.057 pjsua_core.c !.STUN resolution failed: STUN transaction has timed out (PJNATH_ESTUNTIMEDOUT) 05-15 08:16:22.162: E/libpjsip(873): 08:16:22.168 pjsua_media.c !..NAT type detection failed: Invalid STUN server or server not configured (PJNATH_ESTUNINSERVER)

It's NOT a crash. It's a log of an error that happens because the network is currently unreachable. (And I guess the crash of the other ROM app is also linked to some problem with network). But anyway, it's not a bug in csipsimple, it's a crash in the ROM. I guess it's a custom ROM (or one from a manufacturer that doesn't make robust stuff) and if there is a fix to do, it's in the rom
