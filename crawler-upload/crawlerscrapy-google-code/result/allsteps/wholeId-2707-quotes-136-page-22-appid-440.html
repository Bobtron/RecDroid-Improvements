Comment #5

Posted on Dec 15, 2012 by Quick Hippo

There is no real other way to know what is blocked unless you test it or you ask IT ;).

About the way to workaround : The general idea is to have the SIP channel encrypted. So using SIPS could help with that. To use SIPS (SIP + TLS) your SIP provider must support TLS transport. If they allow other SIP port it could also help. Then once you'll have the SIP (signaling) part solved, you'll have to test media channel. There is less way to workaround this part if blocked too. But maybe it's not blocked.
