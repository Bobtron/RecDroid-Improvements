csipsimple - issue #2022

Displays "Incoming call" after call has already been answered

Posted on Oct 13, 2012 by Happy Cat

What steps will reproduce the problem? 1. Receive call (or make call, considering I'm using Google Voice Callback to call me back on outgoing calls.) 2. Slide green answer icon to answer call.

What is the expected output? What do you see instead? It should show me a call duration counter along with an option to turn on the soft keypad. Instead it still displays "Incoming call" in the upper right. Within the menu at the bottom right, I see "Answer" (pressing this has no effect) and "Ignore" (hangs up the call).

I can hear and talk to the other caller but I can't use the keypad and sometimes (always?) the calls get dropped shortly after because, I guess, it decides I'm not answering this 'incoming call'.

It worked correctly before. I have updated the version and uninstalled and reinstalled it since, but I'm not sure exactly when it started working incorrectly.

What version of the product are you using? On what operating system? Using CSipSimple 0.04-04 on a Galaxy Nexus with android 4.1.1.

Please provide any additional information below. I use a Callcentric account with CSipSimple.

Comment #1

Posted on Oct 14, 2012 by Quick Hippo

It often happens when we don't get the ACK from server on our 200 OK. Normally the remote side should acknowledge the fact we reply. If not, the app remains in mode incoming call until remote side acknowledge that we replied, and will automatically end the call after timeout (30 seconds). It's the way sip behaves.

So what's to be found is why the sip server doesn't acknowledge on the ACK.

Callcentric recently had a DDoS attack and they probably modified things on their server / or are still not fully stable back to normal. If you collect logs (see HowToCollectLogs wiki page) maybe I'll see a clue. But I think we should wait a little for callcentric becoming stable again before. I did a lot of tests with callcentric guys before the stable release 0.04-04 (because of one unsupported thing there were on their network and had to be workaround on my side) and everything were working fine.

So if you have time to collect logs, I'll have a look. If you noticed the problem only recently (latest 2 weeks), probably not the app the culprit but rather callcentric network that had some problems recently.
