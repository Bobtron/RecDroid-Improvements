csipsimple - issue #1137

Numeric keypad buttons are in the wrong order (because of RTL)

Posted on Jul 10, 2011 by Happy Panda

What steps will reproduce the problem? When launching the application on LG Optimus One phone with Hebrew UI, the numeric keypad buttons appear as [3][2][1] instead of [1][2][3].

What is the expected output? What do you see instead? Even though Hebrew and Arabic speakers got used to UIs with some layout issues, having the numbers to appear reversed is unexpected, and make difficult to type numbers to dial.

Switching to English (US) UI is indeed solves the problem, but the phone owner prefers to use it in Hebrew.

What version of the product are you using? On what operating system? LG Optimus One LG-P500h, Android 2.2.1, csipsimple 0.02-03 r944

That's really weird.

When I test on emulator I don't get this problem even on Hebrew (even with Hebrew actually translated -- for now it's disabled cause someone told me the translation I got from Rosetta translation project was not valid -- if you want to contribute to it, it will be welcome ;) ).

It sounds android OS has many problem with RTL. I guess LG rom does things to globally support RTL in android but it will impact all applications that behaves normally by reversing all layouts :S.

For now, just for reference, my first findings about that : http://code.google.com/p/android/issues/detail?id=8626 (as screenshot what I get on my galaxy S in hebrew).

For now I don't want to break things for devices that do actually support correctly RTL regarding android way to support RTL. So I'd say, that's more likely a bug on LG rom for now that just invert layouts. Since it seems to be fuzzy on android side, I'll do more search about the correct way to handle RTL, but I think the one done by samsung on SGS is the good one since it does not require developpers to create new layouts inverted just to plan that it will be inverted once again by android layouting.
