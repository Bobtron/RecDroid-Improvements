csipsimple - issue #1963

Continues ringing after call answered elsewhere

Posted on Sep 16, 2012 by Swift Elephant

What steps will reproduce the problem? 1. Register multiple SIP clients to server 2. Receive incoming call 3. Answer it on a client other than csipsimple

What is the expected output? What do you see instead?

Csipsimple should stop ringing when it receives the CANCEL. And should note the Reason: header therein and not report a missed call (qv). But for the last few weeks it's just kept ringing! This is a regression in the release version (on ICS Sgs2) in the last few weeks and persists in the nightly build.

What version of the product are you using? On what operating system?

Please provide any additional information below.

Tcpdump has been sent in email.

It's helful. Means that the app correctly receive the CANCEL but that the CANCEL is considered as not matching previous request. That's why the sip stack reply with "Call/Transaction does not exists" and considers the sip call is still active.

The question is ... why it doesn't match? Apparently the remote side changes the "From" field, which I'm not sure is SIP valid. I'll check that and see if there is any recent changes in pjsip regarding matching sip requests. Did you try with another sip client on the other side? Did you notice the problem only when starting using this remote side?
