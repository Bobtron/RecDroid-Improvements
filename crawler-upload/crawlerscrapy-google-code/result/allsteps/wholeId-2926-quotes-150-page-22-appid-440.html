csipsimple - issue #1888

Wrong audio-redirect on HTC One X

Posted on Aug 20, 2012 by Quick Dog

What steps will reproduce the problem? 1. Install app on HTC One X 2. Use correct settings. 3. Make a call, or use test-audio. 4. You see that the incoming audio bars are responding, but you don't hear any audio.

I'm using the latest version of CSipSimple. On my Galaxy Nexus it is working properly. On the One X there is no incoming audio. But with the volume settings, the volume bars are moving with incoming audio...

So... there is incoming audio, but not directed properly to the speaker on the One X. (Yes, i have bluetooth completely disabled)

Comment #1

Posted on Aug 20, 2012 by Quick Hippo

Thanks for the detailed feedback.

Indeed looks like something with audio routing. What is usually very helpful in this case is to try to use the audio troubleshooting options. To do so go in Settings, press menu and switch to "Expert mode" (if not previously done). Then go in media section and at the end of the list there is a subsection for audio troubleshooting.

The options here could be very helpful. The first thing you could try is to change the audio implementation from openSL-ES to Java (it should be latest option) : Java was the default in previous version of the application.

The option to use mode api and/or routing api could also help. The mode while in call can also be tested (if not using mode api).

If you find any option helping and works properly when routing through speaker/earpiece while in call, let me know. It will help a lot and I'll be able to integrate the option you find as default for HTC One X as I explain in this FAQ entry : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting

Actually the r1808 just by default disable "Focus audio". If you re-enable it manually... obviously will just result same things as before !

It's some feature that is probably not handled properly by HTC. Maybe there is another workaround that could lead to something working; Usually this could be linked to the audio mode while in call.

So if you are using HTC One X and want it to work... DISABLE Focus audio. If you manually enable it... it will fail.. it's not csipsimple... it's HTC... the "audio focus" feature works well on other devices ! (the goal of this feature is to mute other streams than our stream -- for example music or GPS voice guidance). I don't know what HTC did on this device, but apparently they also mute the application stream....
