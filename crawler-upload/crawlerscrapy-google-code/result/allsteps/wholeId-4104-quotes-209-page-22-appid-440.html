Posted on Feb 9, 2011 by Quick Hippo

Regarding your other issue there is maybe another sip app installed on your phone at the same time. Could be annoying on which app is holding the audio resources.

So worth to try to disable completely others apps, and to reboot phone.

Also cynagen 3.2.3 is not yet stable. Audio drivers has been changed a lot cause of the new opensl layer and the new stock sip app.

I'm currently implementing opensl which should be much more reliable than using the java audio api...

So stay tuned for opensl driver on gingerbread.

Keep me in touch if reboot + disabling other voip apps helps.
