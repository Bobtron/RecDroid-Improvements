Comment #1

Posted on Dec 2, 2010 by Quick Hippo

Android API doesn't provide a way to use BT A2DP devices as micro input.

CSipSimple (for now), doesn't activate automatically BT. And when it activate BT it does using SCO (which is the only way to get micro input from the headset on the current API). (Frequency has to be set to 8kHz in csipsimple settings to get something that works properly with all devices - I'll soon do a fix for automatically switching to 8kHz).

What you observe with the AD2P device is just the standard output done by android. Just like if you have plugged a headset. So micro in use is phone micro as AD2P on the api doesn't provide a way to get back micro. And I code nothing for that, just like for a wired headset, it's done by android... would be nice if android OS provide a way (if technically possible but I guess so), to automatically get micro input from the AD2P headset just like they already do for wired headset on 3'5 jack plugs.

Hope it make things more clear for you. To sum up : android API doesn't allow AFAIK to use A2DP as input mic. I've to dive in the android phone app code for this subject again, but after my first look on their code, my conclusions was that telephony app use SCO mode only (anyway gsm calls are 8kHz and mono I think, so there is no big difference ;) ).
