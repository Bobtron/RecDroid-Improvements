Comment #1

Posted on May 5, 2011 by Quick Hippo

What is the configuration of CSipSimple? Did you set to "Always available" + 'not enabled allowed over Mobile data'? (Or if you go in CSipSimple > settings (press menu and activate expert mode) > network > what is the status of Lock wifi?)

If so CSipSimple take a wifi lock. This wifi lock should be taken into account by every other apps and by the android OS to avoid wifi to be switched of unless user do an action. If not, CSipSimple does not take wifi lock at all. It means that CSipSimple does not retain wifi. It is the case if "Only for outgoing" or "Only when wifi is on" is selected. And can also occurs if use mobile data is allowed (cause obviously no reason to take lock on wifi if possible to register over 3G).

If another app do not take that into account, that's obviously the fault of this app (or a feature of this app ;) ). If this app do not take wifi locks into account... nothing can be done on my side...

Keep in mind that locking wifi on may drain battery faster. But well, if you need that to be connected to your SIP account ... there is no magic ;).

So : * check your config to be sure you did not miss something in the conf and that you configured the application to actually do not retain wifi ;) * if that's actually some feature of Juice Defender to cut wifi even if other apps are retaining wifi lock... There is absolutely nothing I can do and you should complain to Juice Defender developers if they don't provide a way to disable their "feature" to disable wifi (actually the phone if you didn't change the default settings should cut wifi after a while when screen goes off - it's in android wifi settings > menu advanced > wifi sleep policy).
