csipsimple - issue #2752

Calling doesn't work

Posted on Jul 31, 2014 by Helpful Lion

What steps will reproduce the problem? 1. Dial any number 2. Call

What is the expected output? What do you see instead? Number should be called. Instead, nothing happens (stays on dialer)

What version of the product are you using? On what device / operating system? CSipSimple from market (same issue on latest trunk as well) running SlimKat rom on Huawei Y300

Please provide any additional information below. Digging through log file, I noticed bunch of errors regarding OpenSL-ES and audio drivers dying and then finally "Unable to open sound device: Invalid or unsupported audio device operation (PJMEDIA_EAUD_INVOP) [status=420009]".

Very rarely it would actually call, but sound comes off loud speakers, and when I click speakers icon app then crashes.

After switching to Java backend, call can be initiated, but caller nor person called can hear anything.

Interesting is that Zoiper works right out of the box using OpenSL-ES backend.

CSipSimple works fine on stock ROM that came with the phone, but given no such issue is present in similar application (Zoiper) I chose to report this, perhaps I can find out how to fix this.

Comment #1

Posted on Aug 4, 2014 by Quick Hippo

Hi,

Did you try with a different clock rate option? By default csipsimple set to 16kHz which gives better audio rendering, but that might not be properly implemented by all. You could also try to change the way the app open audio driver. See : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting

Any working combination of option is of course very welcome for reference to other users.

However if stock rom works and not the modified one, I would say there is a problem on modified one : a ROM should not prevent a way to open the audio layer. It's not to blame them because I know very well how hard it's to try to hack proprietary drivers and get things working perfectly in all situations. Even official ROM sometimes does very weird things. But trying to get at least same feature level than the stock ROM is the work to be done by people making custom ROM (and not to workaround in all apps ;)... because even if we could find something to workaround in csipsimple, would probably still be present in other apps using audio ).
