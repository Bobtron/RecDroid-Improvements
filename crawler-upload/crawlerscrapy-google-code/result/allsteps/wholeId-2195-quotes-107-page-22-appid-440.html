Comment #4

Posted on Jan 12, 2014 by Quick Hippo

Thanks for the logs. It looks like a problem with the ROM, but not the problem I was thinking about initially. Can you try the following : * Switch to ExpertSettingMode (wiki page) global setting section. * in Media settings > Audio Troubleshooting section > Audio implementation : switch from openSL-ES to java-jni.

Java-jni implementation is the legacy one, used by most sip client but that is not very good for performance. However, if the rom does not support opensl-es properly, java-jni is the only option that is suitable. If this suggestion works, please report the problem to people providing the ROM you use. They have a bug in their opensl-es implementation which might make apps using opensl-es optimization crash.
