Comment #1

Posted on Feb 5, 2012 by Quick Hippo

Did you try to play with audio troubleshooting options (see http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting)? Also try to disable bluetooth (if activated on the galaxy S II).

Another thing to investigate is codecs. In fact if you didn't mention another device working good in the same configuration it would have been my first recommandation. But for some reason both device could select different codecs anyway, so worth to check too. In fact, the application support several (many) codecs. Not all codecs are suitable for use on mobile network. Mobile network have very variable bandwidth. Even if you get a high signal, it often depends of the place you are and the number of other users connected to the same radio antenna. Not all codecs are suitable for such a connectivity. The one supported by most sip provider, g711u/a aka PCMu/a is really really really bad for that. It does not compress data and send raw audio packets, which make it 64kbit/s. CSipSimple supports many other codecs that are more suitable for low bandwidth situation (most of these codecs are around 10kbit/s.

When using a codec that use more data bandwidth than what's supported by the network, you'll get exactly the symptom you describe, robotic + high latency, and sometimes it cuts too.

So you should check what codec is in use while you are in call. To do so, press the little arrow under the call card, and select the information button. You'll see here the codec in use. If it's PCMU/PCMA don't search anymore. That's your problem.

Then how to solve. It's more likely not a problem of the client : by default csipsimple order codecs for low bandwidth situations so that codecs that consume less are selected in priority. However, the client is not the only one, the remote side (in your case your sip provider), can decide to use another codec. So, you should check what are codecs supported by your sip provider, and then verify in csipsimple codecs settings that it has higher priority (or is activated, for example due to licensing fees, G729 is not activated by default). To be sure you can disable all codecs but the one you want to force (to disable a codec, long press on the row).If your sip provider does only support PCMU/PCMA (g711), ask them to support more codecs.
