csipsimple - issue #2054

Can't Create Widget on Nexus 7 Tablet

Posted on Nov 3, 2012 by Helpful Wombat

What steps will reproduce the problem? 1.Install CSipsimple 2.Try to create a widget for account switcher.

What is the expected output? What do you see instead? It should ask you for which account and then create the widget. Instead, it says 'App isn't Installed'

What version of the product are you using? On what operating system? 0.04-05r1994 Android 4.1.2 (A CM10 based rom)

Please provide any additional information below.

I'm using the Apex Launcher (v1.3.4beta1) on the tablet. Interestingly, I use the same launcher on my phone (T-Mobile Mytouch 4G) but I am able to create and use the account switcher widget. So whatever the issue is, it has to do with the Apex Launcher + csipsimple + tablet (N7).

Apex Launcher + csipsimple + MT4G works great.

The default launcher from CM is able to create the widget just fine on the tablet.

Comment #1

Posted on Nov 4, 2012 by Quick Hippo

It's a problem with permissions of the launcher (the permission "BIND_APPWIDGET").

I also have exactly the same with my own build of the AOSP launcher.

The point is that in android 4.1 the BIND_APPWIDGET permission has now protection level 18. And so is not granted to Apex.

I'm not sure on how to treat that. If I remove the request of the permission in my app would be a security breach. BTW, any app you might found that have a configure view and works with your app launched on 4.1 has a security bug.

For now, my reply would be to say the culprit is android 4.1 that increased the protection level of this permission and doesn't allow to grant to third party apps (such as mine and such as apex, and probably any apps not "system").

I have to evaluate what would mean to open widely the permission of the activity to configure the widget and if there is not an alternate permission that could be granted to third party apps.

Comment #2

Posted on Nov 4, 2012 by Quick Hippo

Well, after reviewing code of the app widget configure and reading a little bit more about this permission, seems I can grant other apps to configure CSipSimple widget. As it normally doesn't leak any other permission it will be fine to be more permissive about widgets configuration from other apps.
