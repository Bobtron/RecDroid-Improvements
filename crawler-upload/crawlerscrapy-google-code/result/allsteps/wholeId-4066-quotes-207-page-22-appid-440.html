csipsimple - issue #755

Caller ID not sent when dialing from native dialer (using pbxes/gtalk)

Posted on Feb 28, 2011 by Helpful Panda

What steps will reproduce the problem? 1. Check "Integrate with Android" option 2. Place call from native dialer

What is the expected output? What do you see instead? The other end should see my number. They see "unknown caller" instead. However, if I use the CSipSimple dialer they do see my number. The difference seems to be that in the native dialer, I am dialing "<phonenumber>", and in the CSip dialer, I am dialing "<phonenumber>@pbxes.org".

What version of the product are you using? On what operating system? I am using r615 on Android 2.2.1. I have a GTalk trunk set up in PBXes.org. I am dialing a (different) Google Voice number that I have set up for testing purposes and answering in the associated Gmail account.

Please provide any additional information below. I tried adding my number to Calls options>Caller ID but it didn't seem to make any difference. Using GVoice Callback for now as a workaround.

Comment #2

Posted on Mar 1, 2011 by Quick Hippo

I guess that's something with server side configuration. Pbxes.org allow you to configure on their side the caller id independently of the client side configuration. So maybe something that was broken on the server. (Sometims pbxes.org restart accounts which may help on this kind of issues).

Cause "normally" using the native dialer, the csipsimple digit dialer or csipsimple text dialer leads to dial @pbxes.org if you choose to dial through the pbxes.org account. Be careful however that if csipsimple can't use the current connection to make a sip call the phone automatically choose mobile to make the call.

However if you reproduce it feel free to re-open. It's hard to know what is the root of the problem with this kind of fuzzy behavior.
