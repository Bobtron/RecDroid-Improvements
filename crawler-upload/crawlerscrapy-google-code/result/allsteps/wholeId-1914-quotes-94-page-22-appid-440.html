csipsimple - issue #2817

"error while registering - forbidden" with MUFF.PL provider

Posted on Oct 26, 2014 by Quick Giraffe

What steps will reproduce the problem? 1. use basic wizard to add an accounting using this provider 2. give the account a name - I used "MUFF" 3. enter the username and password provided by VoIP operator 4. enter server name sip.muff.pl

What is the expected output? What do you see instead? I used the basic wizard (as no provider-specific wizard exists). I expected the same result as with the previously set up accounts, i.e. account is added to the list and marked "active".

In this case, as soon as I click "save", the account is added to the list of accounts, with a comment "error while registering - forbidden".

Attempting to use the Advanced Wizard ends up in the account being added but marked "Inactive".

I used the same account info (without using any advanced options) in a different SIP/VoIP client - everything worked, no additional details were required.

What version of the product are you using? On what device / operating system? Product version: 1.02.00 r2330 Device: Nexus 4 Operating system: Android 4.4.4

Please provide any additional information below.

Comment #1

Posted on Oct 27, 2014 by Happy Giraffe

With muff.pl you have two sets of user names with two seperate passwords:

  1. to login into your profile (name and password used to login on muff.pl/panelklienta)
  2. to access and register your SIP account (accessible only once you have a verified account with muff and have logged in to your profile).

The account details required for CSIPSIMPLE are the credentials described in #2 above.

Looks as though CsipSimple is not registering on to the muff network. I think we need a dev person from CsipSimple to look into and advise on this one.

If X-Lite and Yate work, I can only assume that CsipSimple is not sending out the request in the correct format or is not interpreting the response from the service correctly. I personally have come across a simlar problem with APPs that try to overwrite a config that has been defined in the native phone app.

The work around is to delete the muff.pl SIP account in the native phone APP setup and then use the CsipSimple wizard to setup the account. The only problem with this method is that it other APPS on the phone that use the initial SIP account config may stop working and will require to be reinstalled ;(.
