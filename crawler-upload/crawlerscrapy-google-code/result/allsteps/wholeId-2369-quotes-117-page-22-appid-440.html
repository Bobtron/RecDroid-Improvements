Comment #1

Posted on Jul 27, 2013 by Quick Hippo

When you configured on destkop, did you enter a different username and authentication name? Sometimes, some sip server uses a different value for username (the user in sip:user@domain.com AoR) and the authentication name (what should be use to challenge authentication).

If the sip server is not configured to be a proxy, it could be interesting to clear the "proxy field" (available if you long press the account row > choose wizard > advanced and edit values here).

The comparison with what's send and works on pc sip client might be useful to deduce how to configure csipsimple and restricts features it uses to be sure the server is not lost with a feature they do not support correctly (sip is normally designed so that extra feature should be silently ignored by a server not supporting it, but some servers are buggy and just get lost if there is something they do not expect).
