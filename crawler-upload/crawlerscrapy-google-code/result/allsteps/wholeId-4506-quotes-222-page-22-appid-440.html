csipsimple - issue #327

iLBC Dropped Calls

Posted on Oct 30, 2010 by Swift Monkey

What steps will reproduce the problem? 1. Register line with iLBC codec. 2. Place call 3. Call drops

What is the expected output? What do you see instead? Successful call. Call drops immediately on outbound. Inbound receives an error "incompatible media parameters or not enough resources"

What version of the product are you using? On what operating system? 0.00-15, Android 2.2, Phone system Epygi Quadro 4x with iLBC enabled.

Please provide any additional information below. PCMU, PCMA function as expected with choppy audio over 3G.

Comment #2

Posted on Oct 31, 2010 by Quick Hippo

@jake Tobias is right, but some other details : If you are on a old device ilbc is by default deactivated in settings cause the ilbc implementation is a floating point implementation which is supported only on armv7 and higher. So could be the case here : both endpoint must be able to do ilbc else it will try to fallback to another codec if available on your server/your remote contact.

So first thing to check is if ilbc is activated in settings > media > codec. If not you could try to activate it but I advise you to not use ilbc cause on your device that doesn't support floating point call quality will be awful! If already activated (and was the default value of course)... well you are on a recent device and problem is probably on the other side that don't support ilbc. Cause what you observe is typical of a codec negotiation that fail : both ends do not found any common codec to talk with
