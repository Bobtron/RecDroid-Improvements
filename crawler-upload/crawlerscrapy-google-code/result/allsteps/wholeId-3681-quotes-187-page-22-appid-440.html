Posted on Jun 27, 2011 by Quick Hippo

:)

You should read this FAQ entry : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#When_screen_goes_off_sound_quality_is_bad

You can also search across this issue list for the "PSP" keyword. You'll learn that many HTC devices has a wifi mode called "PSP mode". When the driver has this """feature""", when screen goes off, wifi packets are recieved with a very high delay.

As consequence, to workaround this behavior CSipSimple implements a workaround (which is automatically activated in your case since CSipSimple detect you are running a HTC Sensation which was reported to have this behavior). The workaround is to get the screen locked on, but to improve things I add an overlay to avoid you from doing unwanted actions.

Normally android 2.3 should bring something that ~should~ allow apps to workaround the PSP behavior. Unfortunately, from users feedback, sounds that the HTC Sensation (or at least some models) does not have this new API working.

However, on the HTC sensation I was able to test on, the android 2.3 functionnality was working correctly (but it was an HTC sensation with a very very recent ROM - not sure if officially released yet).

As consequence, you can try to disable the PSP workaround (keep awake while in call option, see the FAQ entry it explains how to activate...so should be easy to know how to disable ;) ).

If your ROM support the 2.3 feature, when screen will go off you'll still have a good audio quality, if not you'll observe no sound at all. Obviously since some users reported me that on their device PSP workaround was needed I decided to activate it by default on csipsimple for HTC sensation owners. I prefer users unhappy with the proximity sensor than users disappointed with no sounds at all. All the more so as, hopefully users wondering why proximity sensor does not turn off screen will ask on issue list just like you :) And I'll tell how to disable "keep awake while in call" workaround... and besides user will understand what's happening :) and why the app do that, and will be more happy to see that CSipSimple support their phone and take into account how manufacturer to things the best way possible based on users feedbacks :)

If you want to learn even more on this problem, there is an android issue on this point (I'm very active on) : http://code.google.com/p/android/issues/detail?id=9781

Keep me in touch on what you observe with your ROM. I'm pretty interested with users feedback on this kind of point. When most HTC sensation owners will confirm that the updated ROM from HTC fix things, I'll by default disable the PSP workaround for their device.
