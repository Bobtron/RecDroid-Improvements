csipsimple - issue #2585

Can not response 180 Ringing after 100 Trying ( With ICE and STUN enabled )

Posted on Dec 9, 2013 by Helpful Rhino

What steps will reproduce the problem?

  1. Using CSS as CallEE
  2. CSS CallEE with STUN ( stun.counterpath.net ) and ICE ( In global setting; left default in account setting ) enabled; CSS CallEE use TCP transport ( specified in account setting; allow both udp and tcp in global setting )
  3. Whatever you use New STUN Format or not.
  4. Call to CSS CallEE

What is the expected output? What do you see instead?

EXPECTED:

After CSS CallEE received INVITE and response 100 Trying msg, it should response either 180 Ringing or 183 session progress

INSTEAD:

CSS CallEE never response 180 Ringing and ended with 487 Request timeout by remote side.

What version of the product are you using? On what device / operating system?

( both callER and callEE )

CSipSimple Version:

● CSipSimple Stable on market ( r2330 ) ● CSIPSimple Night Build ( r2339 ) ● CSIPSimple Night Build ( r2342 )

Device and Android Version

● Xiaomi MI S2 on Android 4.1.1 ● HTC ONE X on Android 4.2.2 ● Samsung Note 2 on Android 4.1.2

Please provide any additional information below.

● CSS can not recognize optional STUN attribute type 0x802b 0x802c ( RFC5780 ) as described at: http://lists.pjsip.org/pipermail/pjsip_lists.pjsip.org/2013-February/015759.html But I guess it is not the major causes. Just for your information.

● If lucky, CSS CallEE can response 180 Ringing, but rarely. If the SIP stack create ICE session successfully, then it can response 180 well.

Comment #3

Posted on Dec 14, 2013 by Grumpy Camel

UUID is related to SIP Outbound RFC5626 (see here https://groups.google.com/forum/#!topic/csipsimple-dev/rcwdXSsDq_0).

Previously CSipSimple was creating the same UUID for every installation. Regis fixed it now so each UUID is unique as it should be.

Also, you need to tell us what your server setup is. Do you have a sip proxy like Kamailio, repro, etc or a B2BUA like Freeswitch? The reason I ask is that you should enable RFC5626 in whatever server setup you have.

Hope this helps.
