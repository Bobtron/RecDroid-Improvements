csipsimple - issue #360

Galaxy S: Media sliders on 0.00-15-10 seem to be incorrectly scaled

Posted on Nov 8, 2010 by Happy Bear

What steps will reproduce the problem? 1. Make a call 2. Open the Media settings from the menu 3. Adjust media settings (volume/microphone)

What is the expected output? What do you see instead? I would expect to find a useable setting somewehere between min and Max. Instead, all the useable settings seem to be concentrated around the first 1 or 2 mm of the sliding scale. Anything beyond that will generate excessive sound and distortions. This makes it very difficult to try various settings. BTW - Checking echo cancellation does not seem to provide any echo cancellation. To prevent echo one has to very carefully set the earpiece volume to be fractionally above zero.

What version of the product are you using? On what operating system? CSipSimple_0.00-15-10.apk Samsung Galaxy S, Stock Froyo, UK version XXJPO

Please provide any additional information below.

Comment #1

Posted on Nov 8, 2010 by Quick Hippo

Samsung did things extremly crappy with this device audio stack. What you see here is a workaround I add to implement to control the volume using software volume. The scale you see is intended to work on all device with specific implementation. Default values on galaxy S are the good ones to avoid echo, and if you set echo cancellation there is actually an effect, but with this awful implementation from samsung it has no effect due to the fact the speaker is really really too loud to be cancelled by the sip stack.

If you are interested by more details, search on this issue list about galayx S i9000, you'll find a lot of entries about this device that explain why it's now like that. Besides, making it going through earpiece and not rear speaker was also really hard with galaxy S. I found a workaround (that I shared with sipdroid devs), but that's just a workaround, and users should cry against Samsung... We (VoIP application maker) do our best with what is provided by manufacturers and sometimes... it's really hard. So don't be too exigent on that point, how it works is the best way to be the more compatible on all devices (and actually galaxy S is an exception on the android world).
