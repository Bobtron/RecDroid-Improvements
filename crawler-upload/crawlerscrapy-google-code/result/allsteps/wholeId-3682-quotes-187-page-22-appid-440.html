Posted on Jun 27, 2011 by Quick Hippo

Ok, maybe first thing that could help me is to get some logs from the app (the force close is usually very interesting).

So see : HowToCollectLogs wiki page.

About the Gingerbread and the fact that it does not record. There is something you can try. Actually, gingerbread (should.. at least ROM that does correctly things), bring a new micro mode that should improve things regarding echo cancellation (using hardware echo cancellation).

However some ROM does not support that correctly and it could lead to bugs.

So you can try to use antoher micro mode than the one used by default.

See : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting

What could be interesting to test in your case is : * micro source * audio mode for sip calls.

Beside the SGS tab which I usually know with codename : "GT-P1000" has the Galaxy S hack activated by default when detected in CSipSimple, if it does help for you, I can add the SHW-M180S codename to the list of device that need the GalaxyS hack... (not only samsung do crappy audio driver... but they also do inconsistant naming of their products :S).
