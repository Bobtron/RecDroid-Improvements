Comment #3

Posted on Jan 29, 2011 by Quick Hippo

I think that your provider does not support standard caller id. Or rewrite it... some SIP provider rewrite caller id as set by the SIP client. If so they usually provide you a way to set it. Another reason could be that they have a gateway that use P-Preferred-Identity (This is not yet supported by CSipSimple : that's an extension of SIP protocol, standard SIP behavior is to use display name in contact) CSipSimple actually support Display name. There is two ways to set it. In latest devs version it can be set globaly in Settings > Default caller id or in account setting if you choose advanced or expert wizard (but in expert wizard you have to know exactly what you do and have knownledge in sip protocol ;) ). If you check SIP payloads, you'll see that display name is correctly sent.
