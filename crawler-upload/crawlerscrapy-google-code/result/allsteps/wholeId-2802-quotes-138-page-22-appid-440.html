Comment #1

Posted on Oct 19, 2012 by Quick Hippo

In fact there is something that could be confusing at first look in the app.

Global settings allows to enable or not UDP and TCP (and TLS) transports. It's global settings, so global transport available for the application. Normally you can leave TCP and UDP enabled, that's fine.

When you want an account to use TCP, it's the account that you want to have using TCP, so it's account settings and configurable in account view. If you want to enable TCP, wizards that supports that are Advanced wizard and Expert wizard (other providers that have known tcp support also have a checkbox to select tcp). So, in your case, Go in accounts view, long press your account row, and select choose wizard. Then go in generic section and pick the "Advanced" wizard. Here you'll see an unique checkbox to enable TCP transport for this account. (If you want to have invite going through tcp too, be sure that proxy is setup for this account too). Save the account.

You can also edit the account in expert mode if you want to try the tls transport.
