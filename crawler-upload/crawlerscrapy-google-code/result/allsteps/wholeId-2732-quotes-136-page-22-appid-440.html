Comment #11

Posted on Nov 16, 2012 by Quick Hippo

Ahah ok great :)

So if you are interested in the technical part : The android stock phone app : https://android.googlesource.com/platform/packages/apps/Phone/+/5c24ba2eaee30b1eb232b3a22a925240d25857fc/src/com/android/phone/PhoneGlobals.java

It's line 493. It's private API of PowerManager class that needs to be solved using Java reflexion. mPowerManager.isWakeLockLevelSupported() had previously a different name (previous approach was to get some flags and compare).

If viber (or another app) do not this check as it's done and was done using previous method it's very risky. Because they use a private API (which is already risky) and beside use it without reading actual code of google doing that;.... they maybe just found some hint on stackoverflow and didn't try to search the source. (many developers thinks stackoverflow is a reliable source ... even in big companies lol).
