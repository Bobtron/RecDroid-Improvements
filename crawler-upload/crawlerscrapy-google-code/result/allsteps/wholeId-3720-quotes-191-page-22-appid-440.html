Comment #2

Posted on Jul 7, 2011 by Massive Wombat

I was experiencing the exact same issue: Vono calls being dropped whenever using WiFi. Interestingly, it worked perfectly on 3G.

Since I have a 2.3.4 Android, and my issue was with WiFi, I enabled the stock SIP interface (to eliminate the possibility of issues on my network, preventing the call). Once it worked fine, my attention went back to CSipSimple over Wifi.

To cut the story short, here's what you have to do to make it work:

Disable all the codecs (for both wideband and narrowband), leaving only 722 and 729 enabled.

Hope this helps anyone else with this issue.

Comment #3

Posted on Jul 15, 2011 by Quick Hippo

Yes, usually too much codecs support may lead to have the server on the other side completely lost.

These for several reason : - Too much codecs means big SDP offers => so big UDP packets and some routers does not allow that - Too much codecs may lead the server to reply with multiple codecs in the final offer which is not supported that pjsip, and as consequence pjsip will try to fix codecs by a update/re-invite. Unfortunately, some sip server announce to support update while it actually does not, and then everything goes very wrong. (I recently introduced a workaround to force re-invite method regardless what is announced by sip server).

So for these reason, you should try to reduce codecs at strict minimum of supported ones by the sip provider. Obviously this kind of issue will not affect sip clients that does not support many codecs ;).
