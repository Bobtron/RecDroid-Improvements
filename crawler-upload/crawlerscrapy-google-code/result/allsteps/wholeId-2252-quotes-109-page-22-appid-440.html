Comment #6

Posted on Feb 8, 2014 by Quick Hippo

Hi,

The fact the ipv6 port is offset by 10 is intended. (It's set in code here : https://code.google.com/p/csipsimple/source/browse/trunk/CSipSimple/src/com/csipsimple/pjsip/PjSipService.java#563)

At first implementation of ipv6 I was unable to make tests myself and was asked to me to offset the port in order to ensure it's correctly using ipv6. Maybe it's obsolete now. Or maybe we could introduce a dedicated ipv6 binding port setting? What do you think, would it be better to have an extra setting or to set to the same value? The latter is more configurable way to do but might be error prone for users. In any case, yes would be a new issue. If you can open one and in description of it give your feeling if an extra option should be introduced or the current option reused.

Thanks!
