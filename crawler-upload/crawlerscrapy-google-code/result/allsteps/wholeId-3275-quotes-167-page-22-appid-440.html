Comment #2

Posted on Feb 8, 2012 by Quick Hippo

The dtmf tones (when sent using rtp or sip info) are not feedback to users because in many case the echo introduce may make think the remote party of a dual press.

In your case I think that the feedback is not really the problem but rather the fact that the sip server does not transmit the dtmf tone to the bank.

The first thing to understand is that there is several ways to send dtmf tones in SIP. There is RTP, sip INFO and the legacy analogic way that just generate a tone and send it as it was voice. CSipSimple tries to detect capabilities of your sip provider and if the sip provider announce to support RTP csipsimple will use RTP, if not it will fallback on analogic generation. However, some sip server are buggy, and announce to support sip rtp while the gateway to landline calls does not transform it into legacy analogic tones !!! In this case, it's not a bug in csipsimple but rather on your sip provider.

But... good news, in order to woraround this case, there is an option that you can configure to force csipsimple to use the legacy way to send dtmf (many sip client only support this way...). To do so Go in csipsimple Settings > Media > DTMF modes (in misc). Instead of auto, select analogic generation.

Press back twice and try to make a new call.
