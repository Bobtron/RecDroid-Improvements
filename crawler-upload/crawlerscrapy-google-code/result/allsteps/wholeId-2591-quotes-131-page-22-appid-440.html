Comment #2

Posted on Feb 2, 2013 by Quick Hippo

It could be interesting to test the settings : - clock rate (in media) -- motorola is known to have some bugs with clock rate management so you can try with 8kHz which is more legacy option.

In audio troubleshooting section : - The micro source - The audio implementation (you can try to use the Java-JNI one instead of the openSL one.

In each tests, to be sure there is no problem with network config and that it's actually a problem with the audio layer, it's good to check the "media settings" (the last option at the bottom of the call screen). The popup displays an indicator bar that shows what is recorded by the device before sent to network :)

Let me know if one of these option helps, it can be added as default workaround. Thanks !
