Comment #3

Posted on Jul 14, 2012 by Quick Hippo

So if caller-id of bria works, the caller-id is probably in fact the sip username of the AoR.

To simulate the same in CSipSimple :

Long press the account row and select choose wizard. Then select "Advanced" wizard in generic section. Here, leave caller-id field blank. It's not the caller-id that is taken into account by your gateway server. Instead put your caller id in "username" field. Don't leave blank "SIP Authentication ID" and fill it with your sip login.

Save the account.

You know, sip configuration and label naming always depends on sip server/client. In CSipSimple my approach is to simply as much as possible things for mainstream users but leave people that knows SIP free to configure as expert each fields of the official SIP protocol. So, the application allows almost everything supported by the SIP protcol and gives a direct access to the underlying sip stack that is widely used by other projects (even commercial projects). If you use the expert wizard, you'll get any settings possible from very low SIP protocol point of view which should give access to all features you may need. It's just about to find the correct setting then ;)
