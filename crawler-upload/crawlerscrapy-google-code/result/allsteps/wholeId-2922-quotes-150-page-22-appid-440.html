Comment #4

Posted on Aug 22, 2012 by Quick Hippo

Indeed that's very strange that the behavior depends on the wifi type (home/work) and of screen on/off. Maybe could be interesting to try to enable STUN option or to reduce the keep alive interval (in Network > Keep alive section for UDP on wifi).

The weird thing is that normally from application point of view there is no distinction between the wifi of your work and the one of your home. So it's strange that it works well at work and not at home. Sometimes it could happen that it's something with network topology, but in this case it's not linked to screen state and the problem is consistent whatever screen state is (this problem is usually solved by the stun option). The only thing it could be is that the network requires more frequent keep alive (because wifi router of your home cuts connection faster). But in this case it's also not the screen state that leads, but it's more about the delay after registration (this problem is usually solved by reducing keep alive interval).

About the point of comment #2 with direct on hold indication, can you try to download and install latest nightly build : http://nightlies.csipsimple.com/trunk/CSipSimple-latest-trunk.apk ? Also try to disable all codec and enable only g729.
