Comment #5

Posted on Apr 9, 2012 by Quick Hippo

If you read correctly the howtobuild wiki page you'll see that there is a step that generates native libraries of the project. If you don't generate this, size will indeed varies for the obvious reason that you actually forgot to build something. And for the video plugin everything is the native library. The Java code is a fake broadcast receiver and the important part is the manifest and the native library.

There is absolutely nothing missing in code but a step you did miss in the how to build process. If you don't follow steps of the how to build wiki page you'll never reach something working and will get what you observe (only seeing the cam preview) So follow steps of how to build for csipsimple it will produce the native libraries and dispatch each lib in the corresponding Android plugin project (the one for video, the one for codec pack etc)

Again there is no hidden code. All you have in svn allows to produce everything a nightly build and video plugin

So read again the how to build and follow its steps. If at any time you get an error building, join the dev google group (the link is also on the wiki page), search if nobody already has the pb and ask if you find nobody having your problem

You are not alone building CSipSimple by your own, and other succeed and are on the dev group... so it's something absolutely possible for anyone that read with care the how to build wiki page
