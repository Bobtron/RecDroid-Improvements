app-recording-lib

App recorder and analyzer for Android

Little swiss knife lib for recording application runs, performing analysis and reporting problems. All from inside of the app, without root, Android 2.2 and up.

The lib is in alpha stage, so stability might not be perfect, UI looks crappy and the feature set and method names are changing rapidly. There's even no name nor license for the lib yet. In case of any questions or requests feel free to ask. I will gladly see my lib being useful for anyone.

I've seen many crash reporting libraries for almost any moblie OS. Some of them allow recording UI, but it's very slow. Some of them offer application stats, but usually all static. Crash and problem reporting doesn't have any reproduction mechanism, so it's hard to call it helpful in case of non-trivial bugs.

Current features:

  • tracing activity state

  • gathering cpu usage, memory usage, storage space and some system stats

  • system stats graph

  • simple integration

  • catching crashes and sending them via emails

Plans:

  • recording touches and keys

  • unused gesture recognition

  • more system stats (device name, geo location, etc.)

  • recording user to app interaction (user clicked on 'OK' button, SecondActivity started, etc.)

  • suggesting steps to reproduce crashes

  • recording UI

  • replaying UI with touches, keys and device usage graph

  • catching logs
