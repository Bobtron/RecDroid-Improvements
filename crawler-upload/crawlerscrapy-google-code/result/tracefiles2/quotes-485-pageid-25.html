borgstrom-heightmap-extractor

Tool for scaling heightmaps

Borgstrom Heightmap Extractor (BHME) is a tool used to scale heightmaps but may just as well be used to scale images. The library is still in development but is scheduled to get first release before february 2015. Each algorithm will future a CPU implementation and possibly a complementary OpenCL implementation to accelerate the extraction. The utilization of OpenCL will be done through JOCL. If the OpenCL implementation for some reason fails it will fall back to the CPU implementation. The CPU implementation can be multi-threaded but generally is not as it is just meant to be a fallback.

WARNING: DUE TO HARDWARE DELAYS IT MAY TAKE MORE TIME BEFORE LAUNCH

Features

  • Algorithms can be GPU accelerated.
  • CPU implementations uses pure java.
  • CPU implementations require no graphics library such as awt.
  • Fallback system.
  • Platform consistency, same result no matter what OS or CPU/GPU is running it.
  • Fully cross-platform
    • Android (CPU)
    • Linux 32/64bit (CPU & GPU)
    • Mac OS X 32/64bit (CPU & GPU)
    • iOS (CPU) NOT TESTED Should be possible through RoboVM, J2ObjC, Codename One or similar software.
    • Solaris 32/64bit (CPU)
    • Windows 32/64bit (CPU & GPU)
    • GWT (CPU)
    • Any other system that can run java bytecode (CPU)
  • Completely modular.
  • Extendable.
  • Differentiate between algorithms that need to generate the whole scaled image at once and algorithms that can generate pixels and areas.
  • Multiple stacked algorithms.

Usages

  • Scaling down heightmaps to give thumbnails.
  • GPU accelerated image scaling.
  • Saving memory or bandwidth by only transferring a small heightmap than scale it to any desired size.
  • On demand generation
  • Work faster by making small heightmaps with rough design and less consideration for realism and let the algorithms do the hard work.

  • Applied on real world data to make it look more natural when exact similarity is not required.

    FAQ

  • Why not just generate terrain using algorithms directly? Doing so is a great approach if one just wants a realistic terrain. The problem arises when you want a special shape or layout of a terrain. Then this library will work where you make a small heightmap than scale it at run-time.

  • Why borgstrom? Borgstr√∂m is my surname.
  • Will there be a closed source license released? I will most likely offer a "commercial" license in the future. If you are inpatient feel free to contact me directly.
  • Why name it extractor and not scaler? While it does certainly scale it does not just do that. Algorithms may also add new details to make the terrain more realistic.
  • Who is this for? At the moment this is meant to be used as a library by games and other systems which requires heightmaps. Once CLI and/or GUI is implemented it will be usable as a standalone tool to be used by heightmap designers at development time.
