csipsimple - issue #1968

Speex 8kHz Codec Broken When Receiving Calls

Posted on Sep 16, 2012 by Swift Monkey

I was trying Speex 8kHz codec with pbxes.org. The codec worked fine when calling out. However, if the call was an incoming call, the outgoing voice was broken (not heard) with occasional voice that sounded like hissing but intelligible.

Tested with version 0.0.4-01, 0.0.4-02.

Comment #1

Posted on Sep 17, 2012 by Quick Hippo

Speex codec implementation is known to be buggy on pbxes.org. Use another codec to use their server.

Comment #2

Posted on Jul 10, 2013 by Helpful Rhino

I have the same problem on pbxes.org Their sipdroid app also uses speex without problem. Is it possible to implement their speex code into CSipSimple?

Comment #3

Posted on Jul 10, 2013 by Quick Hippo

Just for information, sipdroid is developed by pbxes.org guys. The implementation of speex codec used in CSipSimple is the full implementation from people that defined the speex codec instead of an extract of just parts for the narrow band only. So implementation in csipsimple supports speex 8kHz, 16kHz and 32Hz while AFAIK, the one reduced version used in sipdroid only supports 8kHz. From what I understood, the problem is around the fact they assume everything should be 8kHz whatever they announce, which obviously break something that is able to manage every codec clock rate correctly.

If my analysis of the problem is correct, the good thing to do is not to complain to CSipSimple to reduce its features but rather to complain to pbxes.org so that they fixes the problem (and maybe try to support 16kHz and 32kHz).

This said, I might be wrong in my analysis, and if it's the case I'll be pleased so that somebody point out the root cause of the problem inside pjsip code. For information, CSipSimple get speex codec feature from pjsip stack which seems to have been the choice as base for Asterisk version 12, a very widely used ippbx server... that's why I'm pretty confident in the fact the bug is not on this part ;).
