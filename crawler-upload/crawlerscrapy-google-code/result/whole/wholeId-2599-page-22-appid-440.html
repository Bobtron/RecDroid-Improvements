csipsimple - issue #2223

[enhancement] RFC 3680 pjsip plugin

Posted on Feb 5, 2013 by Grumpy Bear

It would be great to have support for the SIP event package 'reg' (rfc3680), as it enables support for network de-registrations.

Comment #1

Posted on Feb 5, 2013 by Helpful Cat

I'm not sure if this is related but issue 2198 involved de-registration when the SIP stack is stopped or any reason. Have you tried a nightly build >= r2095 to see if that helps?

Comment #2

Posted on Feb 5, 2013 by Grumpy Bear

No sorry, it's not related. The scenario is pretty much:

1) UAC sends SIP subscribe for 'reg' event, targeting his own SIP URI 2) UAS goes offline (or decided to throw out the UAC) 3) UAS sends a SIP NOTIFY to UAC, telling him that his registration has been cancelled (and for what reason)

The same subscription service is used to keep track over others registration's status as well.

Comment #3

Posted on Mar 9, 2013 by Quick Hippo

This should be addressed by a pjsip plugin. So it's not directly an issue for CSipSimple but rather for pjsip. (and should be possible to do in pjsip thanks to their modular approach and will benefit all other apps based on pjsip).

If someone create such a plugin will be possible to include it inside csipsimple.

I still keep the issue open here in case some developer has time to implement rfc 3680 and want the pjsip plugin to be hosted in csipsimple project.
