Comment #3

Posted on Oct 31, 2010 by Quick Hippo

Could you send me logs for the crash? If you can install the latest version (available on download section on this website) there is an helper to send logs. Else follow the instruction on HowToCollectLogs wiki page.

You can also disable the integration to contact feature (there is an option in CSipSimple settings). But if filtering works, would be probably a better solution for you. All the more so as the app only start up if you are initiating a call (and in the future will automatically stop if you don't choose any Sip account). Of course if you are configured "only for outgoing". I think the way it's done by sipdroid is just awful. It has spoiled generation of user with an non-ergonomic way to use a SIP app. What is the link between sms & calls?...

What should be a good solution in your case, would be a true third party app, that would be integrated to 2.x android contacts and add to the list a button "sip call". But this feature would be nice in a third party app, for two good reason : 1- It would be compatible with all sip application (sipdroid, linphone, csipsimple and so on). 2- It should be available only for users with a 2.x or greater device. Here csipsimple is available from 1.5 to 2.2...

This kind of app could do nice things such as being truly integrated in your contacts forms and allow you to add sip entries to some contacts.

So try to send me logs. If it's a force could will be easy for me to fix :)
