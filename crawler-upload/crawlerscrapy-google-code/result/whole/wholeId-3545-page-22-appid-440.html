Comment #1

Posted on Sep 6, 2011 by Quick Hippo

By "call log", do you mean the one in the stock call log application or the one inside CSipSimple.

In the call log application it should add (if the option to sync with stock call logs is activated) the phone number. On CSipSimple, it will add the sip uri. The good reason for that is that you actually called a sip uri, not a phone number. So the SIP application call logs should be sip uri. And the same calls logs should be added in the stock phone application calls logs as simple phone number (if detected as a phone number - cause obviously could be a sip user).

So if you want to call again the same number using something else than the sip application, you should activate the sync with stock dialer call logs and go in the stock call logs application to see the phone number only.

In fact, the goal of csipsimple is to try to allow you to use as less as possible the application dialer and application call logs and application integration with contacts. CSipSimple's integration with the rest of android system is thought in order to make easy to use the app without having to change your habits when dialing somebody. Features such as rewriting rules and filters are made for that purpose.

However, about this particular issue, if it has to be fixed somehow, it will be fixed by issue 184 (allowing to back-rewrite sip uris to things to store in call logs).

Comment #3

Posted on Sep 6, 2011 by Quick Hippo

If you have several sip providers and don't wont to see these providers in choices when making call from the stock dialer or from stock call log apps :

Go in settings > filters > Click on accounts that do no support +7xxxxx, add a rule "don't call" "start with" "+7". Selected provider will not anymore appear in choices when you'll make a call from stock dialer.

Optionally, if you always want to use this sip provider for phone number that start with +7, you can also create a filter in this account : "Force call" "start with" "+7". If you do so, if SIP is available for outgoing on current network and you place a +7xxxxx call, it will automatically choose this account (even over GSM choice).
