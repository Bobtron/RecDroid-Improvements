Comment #1

Posted on May 17, 2011 by Quick Hippo

For now expert mode does support only one realm.

I guess your authorization name for the proxy and for the registrar are different?

Else you can simply change the "username" in the authentication section (in fact it is the authorization name). Each sip provider has its own naming of things ;). But the username in authentication section of the expert wizard is used to authenticate you.

For now, CSipSimple does not present way to reply with different credentials for different realm challenge. It's allowed by pjsip (the underlying sip stack used by csipsimple), but there is no user interface to set it, cause for now, as far as I know there is really no provider with a different authentication name for their registrar and for their proxy.

So first, check if the authentication name that is allowed to change in csipsimple is not suffisant. I really guess so. And before you ask that, I make a difference between username (the one that is in SIP ID in csipsimple expert wizard), and the username (actually authentication name = Username(login) ).

So 80% of sip providers has same username and auth name And 20% have different username and auth name. And ,for now, 0% of sip provider with : * an username, * an auth name for registrar, * an auth name for sip proxy.

I really don't think so, but if that's your case (the 0% ;) ), what is the best solution is probably to add a wizard specific for your provider. If so tell me : * the registrar uri * the proxy uri * the realm of the registrar * the realm of the proxy * other settings that the wizard should automatically set

And I'll create a special wizard for this provider.
