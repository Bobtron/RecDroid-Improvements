road-trip-test-upload-1 - issue #4

Pull current source code from server, add your name, and push back up (all through the android studio)

Posted on Jul 14, 2013 by Swift Elephant

What steps will reproduce the problem? 1. Open android studio, to view default welcome screen 2. press "Check out from Version Control", then select mercurial as our vcs 3. on the next screen, where it says Mercurial Repositiory URL, past the url found on our google code website. To find this url go https://code.google.com/p/road-trip-test-upload-1/source/checkout, and the url on the page will look something like https://robertmurrayburke@code.google.com/p/road-trip-test-upload-1/ 4. then choose where you want to store the test source on ur pc (Dont name it RoadTrip, cause this is only a test). 5. It will then pull the code from our site. (if a window opens asking if you want to open the project, press yes) 6. if it asks if you want to add the .iml file to mercurlial, say no 7. open the project explorer (i think its alt+1 or ctrl+1), then navigate to test>TestUpload1>src>main>res>values and double click a file called strings.xml. this is a file where string variables are stored. you should see my one in there called "robs_quote". Go ahead and add your own one. 8. now go to test>TestUpload1>src>main>res>layout and you should see a cool nexus 4 image of our page. if its in the designer view, right click on the white space, and press "Go To Definition". You should now see a split page, with an image of a phone on the right, and xml code on the left. 9. you will see a textview i have created with reference to my variable defined in strings.xml. do the same for you, so you see your message on the screen too. 10. as you have made changes to the code, now its time to push it back to the server. open the "changes" widow (i think its alt+9), to see what changes you have made to the code. you will see two drop down items, "Default" and "Unversioned Files". Default are the files that are on the server that you have changed, where as unversioned are the oned that are new on your system. you will see about 13 files in unversioned because they are in the "build" directory. we dont want to sync this folder, as its unique to your debug sessions. in the changes window, press the "Configure Ignored Files" icon. add the folder "test>TestUpload1>build". 11. in the changes window, press the commit changes icon. every time you commit changes, you should add a comment about what you are uploading. instead of pressing the commit button, press the drop down and select "Commit and push...". A window will open, and make sure its pointing to the right server. Once you push, it will ask you to provide a user name and passowrd. hopefully the user name is auto populated. the passowrd has been auto generated, and can be seen at https://code.google.com/hosting/settings Enter this to complete your push process. (it will probly ask u to enter some master passowrd, i just used "TestPass11"). 12. double check your code has been pushed correctly, but looking on our code on the google code webpage https://code.google.com/p/road-trip-test-upload-1/source/browse/

remenber to mark this as done when you have finished
