Comment #1

Posted on Jan 7, 2011 by Quick Hippo

Well not sure I understand your issue, nor that you understood what is actually SIP. Are you trying to use my app or are you trying to do another app that is not a SIP app and try to use parts of my code or understand how android works?

When you do a GSM call.... you do a GSM call and that's not managed by CSipSimple. The GSM option is CSipSimple dialer is just a shortcut to the standard GSM application of your phone. CSipSimple integrate both ways : it intercept gsm intents for outgoing calls and allow user to choose between GSM and one of his sip accounts. And it also allow, within CSipSimple to dial using GSM or one of available SIP accounts (if correctly registered). When you make a GSM call.... you use GSM application. There is a way to bypass telephony application, but that's absolutely not done in CSipSimple (I just know that it is possible cause I've read the android source code of the telephony application), but it requires high skills and knownledge of android platform (if that's your goal, I'd advise you to read the android source code and not CSipSimple code). When you make a SIP call... you use CSipSimple application. What you observe is not the Telephony application but actually CSipSimple application. The look and feel is the same than the Froyo stock GSM application but it's not the GSM application.

Last point, but not sure that's relevant cause I don't know exactly what is your goal: in CSipSimple I've developped a full API that allow other third party apps to overwrite / use my SIP service. But it will be relevant only for SIP calls ... not for GSM.
