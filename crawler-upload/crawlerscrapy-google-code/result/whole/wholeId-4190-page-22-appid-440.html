Comment #5

Posted on Jan 28, 2011 by Quick Hippo

Ok got it. The weird thing is in pjsip. It does not take into account the echo mode when starting the stack. As consequence it always use the default (for pjsip), which is in latest dev version speex cause I've activated it. The default for CSipSimple is Simple echo mode which is a good choice cause does not need floating point support.

So now r597 (which will be on nightly builds server soon, I've launched a manual build), will force the choice of the echo canceller option when call is established (this method seems to be valid :) ).

So for armv6 devices, should now be fine with echo canceller (but you should never activate speex echo canceller cause it will be efficient only with armv7 devices).
