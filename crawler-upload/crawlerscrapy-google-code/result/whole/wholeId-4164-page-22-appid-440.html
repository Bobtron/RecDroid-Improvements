Comment #1

Posted on Jan 30, 2011 by Quick Hippo

Well good idea.

First source code is up to date : http://code.google.com/p/csipsimple/source/list we are at revision 612 and you can see in nightly builds http://nightlies.csipsimple.com/trunk/ revision 606 and will be automatically build tonight :).

You probably miss the HowToBuild wiki page cause building CSipSimple is not so simple ;) .

Then as per your request. I'd advise you to read : issue 586 issue 267 and issue 279

As you'll probably read my approach for this kind of request is to advise to do a third party application specialized in those kind of stuff. The main reason is that doing a third party app is much much much more powerful than trying to do that within the SIP client The second reason is that it will make much more complicated the settings of CSipSimple while most users may not use the feature The third reason is that the approach I propose is the approach of the overall Android OS. We are EXTREMLY lucky to have an OS that is very very well designed by Google engineers. The interop of applications with each others is something that is just amazing in Android. For example iphone owners can't have such an experience ... everything that iphone can do is handle uri... poor... Here Android can do almost everything we could imagine in terms of exchanges between apps.

CSipSimple already provide a full API to allow a third party app to control it. (see the package com.csipsimple.api). Some SIP provider will use it to provide a third party app that rely on the service provided by CSipSimple. But it can be used to address your needs.

If you'd like to implement yourself a quick application that just activate / deactivate accounts according network, I can host it on CSipSimple project and distribute it too if you'd like (but you can also distribute and host it where you want). But I think that the best approach for the future remain a true opensource context aware application with plugins. If there is such a project (if somebody starts such a project) I'll do "output" plugins to control CSipSimple... and then we will be able to use all input plugins (those that detect the wifi ssid, those that are aware of time, etc etc :) :) ).
