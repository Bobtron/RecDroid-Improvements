csipsimple - issue #891

Failure to Register over 3G / Blank Screen Hang

Posted on Apr 15, 2011 by Massive Rhino

What steps will reproduce the problem? 1.With WiFi disabled Install app. Choose "only for outgoing" and leave 3G unchecked in the Easy Configuration... save. 2.Add an an account, save and back out to dialpad. 3.Settings->Network->Outgoing Calls.. check mark 3G and back out to dialpad.

What is the expected output? What do you see instead? CSipSimple should attempt a register via 3G.

Instead, receive "Another application is currently using SIP port! Can't start !!!"

If you back out to the main Android screen and attempt to get back into CSipSimple by re-launching (it's still running), you get a blank screen which hangs. Android brings up the "Wait" or "Force Close" dialog after a short period of time.

What version of the product are you using? On what operating system? Latest nightly build and r797 from SVN.

Please provide any additional information below. Android 2.2

Comment #3

Posted on Apr 16, 2011 by Massive Rhino

That is what it seems like... but to test this again I did the following.

  1. Uninstall the app (via adb uninstall).
  2. Verify no CSip processes or other apps that might seize port 5060 are running (via "ps" in the adb shell).
  3. Repeats steps in initial post, same result port is in use and lockup if you go back home and re-enter CSip.

After doing a "kill -9" on the CSip process and relaunching... phone registers fine over 3G.

To be double sure, I performed the same after a power-recycling/cold boot of the phone (thus ensuring no zombie processes), but still the same result.

I haven't looked deep into the code, but I would think after initiating a network preference change, the SIP stack should be stopped and restarted (perhaps the code is there, it's just not be given enough time to release the port?).

Comment #6

Posted on May 4, 2011 by Quick Hippo

This problem should be fixed with latest nightly build. Now port is dynamically set instead of reusing port 5060 as local binding port.

This fix will allow to avoid slowness of the android OS to close the binding ports that may sometimes leads to some blocking situation. So with latest nightlies it will set local udp and tcp port to 0 by default which now means "auto detect available port"
