csipsimple - issue #2551

"Fail to init pjsua Unknown error 320044"

Posted on Nov 13, 2013 by Happy Horse

What steps will reproduce the problem? 1. Make sure WIFI is off an LTE/4g/3g is on 2. Install CSipSimple application and open the application with defaults 3. While configuration wizard is shown a message appears: "Selected Network route not allowed with current settings you have setup" 4. Disregarding the message, continuing with the configurations as: - Integrate with Android - Always available - I'm allowed to use mobile try to configure any VOIP account. 5. While registering the account it fails with the message: "Fail to init pjsua Unknown error 320044"

What is the expected output? What do you see instead? To be able to register VOIP accounts.

What version of the product are you using? On what device / operating system? CSipSimple version 1.02 Updated on Nov 11, 2014 Nexus 5 Adroid 4.4 (KitKat)

Please provide any additional information below. As a note the steps below are fine on my Samsung Galaxy S4 (Google Edition) I9505G with Android 4.3

Comment #1

Posted on Nov 13, 2013 by Quick Hippo

As the device in this situation uses ipv6 only you have to disable DNS SRV feature. Go in settings > network > DNS SRV and untick the checkbox.

Comment #2

Posted on Nov 13, 2013 by Quick Hippo

Note : this only occurs on your Nexus 5 as this one only uses LTE/4G that is apparently provided with ipv6 only dns servers. On your SGS4, 3G is used and DNS server provided to system is not ipv6.

If you absolutely need DNS SRV feature, you have to manually enter a custom ipv4 DNS server in csipsimple settings. To do so, switch to ExpertSettingsMode and in settings > network > custom dns server enter a ipv4 dns server such as 8.8.8.8 (or 8.8.4.4) (that's google public dns servers).
