MMmhhh weird... I'd not be so surprised if Motorola did not things cleanly regarding the new android 3.0 audio API.

Using r804, you'll find an option in ExpertSettingMode, in Media > Micro source. By default on android 3.0 there is now a new source that is there just for voice over IP :) . On my Acer Iconia the result of using this source is just amazing... it use the hardware micro echo canceller ! And even with speaker at maximum I get perfect sound.

So, what would help me if you can do some tests : First test : Try to change the micro source to DEFAULT instead of VOICE_COMMUNICATION and see how it goes with your bluetooth headset. Second test : set again VOICE_COMMUNICATION as micro source and let me know how it goes without bluetooth headset. It's highly possible that I should use DEFAULT micro source when trying to route to bluetooth headset. (This case is fuzzy in the new android audio API).
