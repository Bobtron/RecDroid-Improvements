csipsimple - issue #2563

Suggestion: Show image (or video stream) from an HTTP source on incoming call

Posted on Nov 26, 2013 by Quick Monkey

This feature would be very useful for SIP doorbells.

If someone pushes the button on the doorbell, csipsimple could show the camera image (or maybe optional, an RTSP video stream) as the contact person's icon before the call is answered.

Like we discuss here: http://superuser.com/questions/679313/android-sip-voip-with-doorbell-view-video-before-answering?noredirect=1#comment861552_679313 SIP protocol already seems to offer a header option for it, but this is not used in any Android SIP client at the moment...

Anyway, it would also be a great alternative if you could set a static HTTP image link to be displayed in the client on an incoming call.

Comment #3

Posted on Dec 1, 2013 by Quick Hippo

Hi,

It's "ic_contact_picture_180_holo_light" (it's present in the different densities folders, hdpi, mdpi, xhdpi).

What loads the picture is : - if contact not found in android contacts: it's the xml file of callcard that defines the resources and it's not changed. - if contact exists, the ContactLoadedHandler of the InCallCard. It does an asynchronous resolution of contact inside contacts and tries to apply the contact's photo if any or fallback to the picture_180_holo_light.

Basically if we want to get the feature implemented, we need 1- parse the header and add the photo uri in the SipCallSession object. 2- In the InCallCard implementation, in updateRemoteName function, apply the photo uri to ImageView if no one available locally in android contacts. This should be done asynchronously as android UI thread should not be blocked by retrieval of the online photo (maybe ContactsAsyncHelper could be extended to get something working quickly this way).

Posted on Dec 5, 2013 by Quick Hippo

Hi, Yep it's not obvious indeed. Adding an image from a remote url is not easy because it involves network and has to be done asynchronously. There is even dedicated android libs to show remote picture. You can maybe have a look to this very simple one : http://www.androidhive.info/2012/07/android-loading-image-from-url-http/

With these classes imported, it would finish with a
imgLoader.DisplayImage("http://domain.com/your_image_uri.jpg", loader, photo); instead of the thread line 456 : http://code.google.com/p/csipsimple/source/browse/trunk/CSipSimple/src/com/csipsimple/ui/incall/InCallCard.java#456

This would not be suitable for a final implementation as async mechanism already exists in csipsimple (to show image from contacts) and could be adapted instead of introducing a new lib. But if your purpose is just a demo, it should do the trick pretty easily. Unfortunately I've not enough free time to help you more to do this demo and will rather focus on real implementation if I've time for this topic :). If you have blocking points, do not hesitate to join the csipsimple-dev group and post there your problem (try to elaborate the more possible the problem you face and add the logs from adb if possible). I try to reply on csipsimple-dev group and it's better to reply there when it's a reply that could help somebody else as it's the place to search :)
