csipsimple - issue #904

Direct IP Calls

Posted on Apr 22, 2011 by Grumpy Rabbit

Feature request. I'll describe the idea, let me know what you think.

What steps will reproduce the problem? 1. Open CSipSimple, select new tab 'Direct Call'. It shows two buttons: 'Create Conference' and 'Enter Conference'

2a. 'Create Conference': (Server Role) Listen for incoming clients, optionally allow the user to set a conference password. It'd be good to show the current IP address, to easily distribute it. Add a client to the conference for the local user. 2b. 'Enter Conference': Let the user input the IP of the remote conference server (maybe with a default configuration and allowing customization), and connect to it.

  1. On the server node, whenever the number of connected clients is >= 2, establish the call/conference. When it drops below 2, close it. (maybe it could be faster to start with a one-to-one call and later add the conference implementation)

What is the expected output? What do you see instead? When users are well known and in wifi range, it would allow to talk without needing Internet connection, SIP accounts and network delays. I came to this idea searching for a bike-interphone app: thanks to Froyo's hotspot feature, users could talk when they're 'on the move'.

What version of the product are you using? On what operating system? I'm on Huawei Ideos u8150, Android 2.2 (Froyo).

Please provide any additional information below. In the future it could be nice to add a security layer

Posted on Apr 22, 2011 by Quick Hippo

I think that for client side it can already be performed with :

1- Create an account of type "Local" (select local in wizard, it will just ask you the display name you'd like for local account) 2- When you want to initiate a call to another SIP client (without the need of a sip server), just type the entire remote IP (actually, you have to add a fake username), in the field. It will directly contact the remote SIP client. I do almost all my tests of video calls like that for now, and it works flawlessly.

Then for conference, if you try the nightly build (http://nightlies.csipsimple.com/trunk/), you'll see that's already there, just drag and drop the contact card to hold / retrieve / conference calls.

And as for security layer, have a look to the TLS build (http://nightlies.csipsimple.com/tls/), you'll see that almost everything you'd like to have is already there ;). (ssl/tls, srtp, zrtp)

Comment #7

Posted on Nov 22, 2013 by Quick Hippo

Often 3G network is complex and NATed. In this case the sip client has no 'public' IP. As consequence, in this case, it's not possible to use direct SIP call (as direct sip call requires the client to be reachable directly). So it's suitable when you are on same LAN network, but as soon as there is a router in the middle, it will fail. That's not specific to csipsimple : that's how sip works and how ipv4 network works.
