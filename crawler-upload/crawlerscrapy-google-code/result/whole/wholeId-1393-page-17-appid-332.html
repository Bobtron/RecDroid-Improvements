android-on-freerunner - issue #16

Not booting

Posted on Sep 24, 2009 by Swift Camel

What steps will reproduce the problem? 1. Install v0.1, v0.0.1 or beta7 from either uSD or dfu-utils 2. Try to boot.

What is the expected output? What do you see instead? Boot stops at the green android logo. I compiled my own kernel to disable initlogo and turn on debug logging, and noticed these errors: init: Unable to open persistent property directory /data/property errno: 2 (At this point the /data/-directory seems to be empty) A few missing files in /system/bin: playmp3, flash_image (and maybe mountd, can't remember)

After this, zygote is started and killed and restarted endlessly with message "process '%s' killing any children in process group" in wait_for_one_process() in /system/core/init/init.c.

I've never touched the kernel before and don't know where to go from here, hope someone else can make something out of it.

When booting the kernel in debug mode I see a lot of messages regarding "CLEANMARKER node found at 0x00000000 has totlen 0xc != normal 0x0" and "Empty flash at 0x00003ffc ends at 0x00004000" (not sure about the memory addresses, copied that message from the web). These messages are explained here: armlinux.simtec.co.uk/faq.html , but I don't know what that means and what to do about it.

What version of the product are you using? On what operating system? Occurs on 0.1 and 0.0.1 (and probably beta7, never tried to run a debug kernel with that). Panicking v15 works fine.

Comment #3

Posted on Sep 28, 2009 by Swift Camel

Sorry.

Michael's posting put me in the right direction, changing mount yaffs2 mtd@userdata /data nosuid nodev to mount ext3 /dev/block/mmcblk0p2 /data nosuid nodev in init.rc fixed it.

Comment #7

Posted on Nov 20, 2009 by Swift Camel

I don't think this is SD card related. The problem occurs when we try to mount the data partion on NAND, the workaround mounts it to the SD instead. I had exactly the same problem on my 8GB kingston card and my 512MB transcend card. Also, the problem occurred both after automatic sd installation and manual dfu-util flashing.

I don't know what really causes the problem, but if I should guess I'd say that there are two different types of NAND or some related hardware, where one works and the other fails.

Somebody with more knowledge on these topics might learn something from the fact that my error messages were the same as question 4 and 5 here. The cleanmaker error were repeated (with different addresses, if I recall correctly) very many (~100?) times.

I hope someone can figure out anything from this.

Anyways, the workaround works, but basically requires you to get the source, edit init.rc as described earlier and flash by either using dfu-utlil or replacing kernel.img in the regular release.
