csipsimple - issue #608

Restore ringer settings if incoming GSM & SIP incoming at the same time

Posted on Jan 21, 2011 by Quick Panda

What steps will reproduce the problem? 1. Receive a phone call via voip 2. hang up when finished 3. phone sound is set on silence / vibrate only

What is the expected output? What do you see instead? -- I expect phone sounds to return to wherever they were set before the call

What version of the product are you using? On what operating system? --csipsimple 0.00-16 (recently upgraded via market) on a Verizon Droid2, Android 2.2 System Version.2.3.20.A955.Verizon.en.US

Please provide any additional information below. -- As it happens, the calls that MAY be triggering this problem are to my Google Voice number which is currently set to ring both my cell and voip line at the same time. SO it is possible this bug is triggered only when there is an incoming cellular and csipsimple call at the same time.

Please let me know if any other info would be helpful.

Comment #4

Posted on Dec 24, 2011 by Happy Monkey

I'm very impressed with the progress on CSIPsimple over the past year. On my old Droid 1 running Cyanogen 7.1, it's working great, aside from this issue. (I do need to prevent the phone from sleeping during a call, but that's no big deal)

Like many others, I have configured Google Voice to ring both my SIP and Cellular numbers. So, I'm getting bitten by the bug described in Issue 1190 . However, I don't consider going back to a two year old version as a viable work around. The current version has much better echo cancellation and audio quality in general.

I've read on the CSIPsimple site that the purpose of switching on "silent" mode during a SIP call is to avoid "exploding" one's ears should a cellular call come through. Could either of the following be implemented instead:

1) Switch on silent mode only after the SIP call is actually answered. Switch off silent mode when the SIP call ends for any reason or when speakerphone is enabled during a SIP call. This should be enough to protect against exploding ear drums without affecting those of us using Google Voice to redirect calls. The key is to leave sound settings unchanged until you really have a SIP call in progress with audio routed out the earpiece.

And/Or...

2) Add (yet another) configuration option to completely disable the switching on or off of silent mode. This would likely be easiest. Bear in mind that many phones actually use a different speaker for ringtones than the earpiece. A ring with the phone to one's ear would not be deafening, only unpleasent.

One thing I still don't understand is why the phone does not at least vibrate on the incoming cellular call after CSIPsimple enables silent mode, even if I have the Android sound option ticked to "vibrate on silent" or "vibrate always". Can anyone explain this?

Comment #7

Posted on May 27, 2012 by Quick Hippo

Several fixes has been done in recent nightly builds.

There is a robust saving of the fact the app changes something in ringer mode and restore it if you restart the app later. Now the app only changes to silent mode if there is an incoming gsm call while you are in sip call (to avoid your ear to explode). During the sip call you'll not see that the state is changed else. There is now a call waiting tone that is produced in audio band to announce about a waiting call. This will work for a waiting gsm call while in sip call but also should work for a waiting sip call while in gsm call. http://nightlies.csipsimple.com/trunk/

Comment #8

Posted on Jun 10, 2012 by Happy Monkey

I just tried the build dated June 10th. You are doing some very nice work with the UI. Congratulations on that. However... The ringer silencing rules you have implemented still are not appropriate for use with Google Voice. You state, "the app only changes to silent mode if there is an incoming gsm call while you are in sip call" But you define "in a sip call" to include the case where the sip call is ringing while a gsm call comes in. Every call to my Google Voice number rings both my phone's GSM (actually CDMA) plus my SIP number. Since my phone has no mobile data plan, it receives SIP calls only when it is connected to the net via WiFi. Google Voice ensures I get the call regardless, by ringing both the Mobile and SIP numbers simultaneously.

But, when I have WiFi, CSIPsimple's silencing causes the phone to ring exactly once, then the phone goes silent. So, if I don't hear that single ring, I miss the call. I think you could correct this by silencing the ringer only after a SIP call is accepted. Either that, or provide a configuration option to disable ringer silencing.

Another issue is that the June 10th version appears to be incompatible with both the Google Voice callback apps in the Android Market. The older GVoice Callback app crashes. The newer Voice+ app places call back, however, after you hang up, a prompt screen appears asking whether you want to use the Mobile or SIP service to place the original call (again). Neither of these problems exist with the released version. (of November 2011)
