Comment #1

Posted on Nov 11, 2010 by Quick Hippo

I don't think that skype has a direct gateway to SIP protocol (they use their own protocol) (to be verified but I don't think so). Some SIP provider such as pbxes.org provide such a feature (but everything is translated by pbxes.org to something that skype network can understand).

In the case skype can support sip protocol, I guess they only use their codec (SILK) (there is an open issue here, I've started to implement it recently but will take a lot of time and is not a priority for me). As consequence, even if they support SIP, there is still a blocking point on csipsimple, but that is already listed in open issues. So I close this one as either duplicate of issue 230 or just invalid (if skype doesn't support direct SIP) or wrong config if you should use another sip provider that translate @skype.com addresses into the proprietary voip protocol of skype (from the open and standard SIP protocol).
