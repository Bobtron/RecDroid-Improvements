csipsimple - issue #1051

Transport type in Contact address varies in case (transport=tcp / transport=TCP).

Posted on Jun 14, 2011 by Happy Camel

In some registrations, the Contact header contains "transport=tcp" and in others it contains "transport=TCP".

Many servers such as FreeSWITCH use the Contact header to determine the registration when in "allow multiple registrations" type modes and they are case-sensitive. The case difference in the transport header causes them to think that there are two registrations for the same user.

What steps will reproduce the problem? 1. Register csipsimple against a service which you can tcpdump on. 2. Run tcpdump for a period of time. 3. Let the phone sleep for a while. 4. Review the REGISTER messages in the tcpdump and see the different Contact: headers.

What is the expected output? What do you see instead?

For all registrations sent, send the Contact header eactly the same in each case, including the case of the transport type.

What version of the product are you using? On what operating system?

CSIPSimple r913 on Desire Bravo.
