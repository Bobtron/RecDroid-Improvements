csipsimple - issue #1022

Using the wrong mic on Samsung Galaxy S 2 (I-9100)

Posted on Jun 8, 2011 by Swift Cat

What steps will reproduce the problem? 1. Place or Receive a call using the phone (no headset) 2. 3.

What is the expected output? What do you see instead? The the person on the other side is able to hear me, but they cannot.

What version of the product are you using? On what operating system? Samsung Galaxy S 2, Android 2.3.3, Baseband I9100XXKDA

Please provide any additional information below. It appears that the app is using the wrong microphone - there are two mics, one on top (near the headset jack) and one on the bottom (where your mouth is). If I speak in to the bottom mic (like normal), the other person doesn't hear anything - however, if I speak into the top mic, they can hear me as expected.

Ok I found why my default setting was not taken into account for SGS2.

Actually the source called DEFAULT was not DEFAULT source but source MIC and you probably observed no selection at all previously cause I was setting DEFAULT automatically (which is in fact not listed).

So fix is done in nightly build : - it will introduce source mic and source default (SGS2 should use mode MIC) - by default SGS2 will use micro source mic

I'll do a manual build on nightlies server so that you can test that it correctly fix that by default.
