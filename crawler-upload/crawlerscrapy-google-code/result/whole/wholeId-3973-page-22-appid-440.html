Comment #1

Posted on Mar 29, 2011 by Quick Hippo

It should be more or less the same settings than the one you use to connect twinkle.

Probably "basic" wizard will be enough for you. If you have special settings, you'll may need the expert wizard, but usually it's not necessary. Once your sip server IP/domain name and your credentials are entered in the wizard it should register. If you have problems, see HowToCollectLogs and send me by mail logs so that I can tell you exactly what is wrong (with trace I'll be able to say if it's cause the network is unreacheable or if it's cause the sip server refuse authentication ...)

The fact that you use an OMAP board connected oevr lan should not impact the behavior of csipsimple. As soon as the connection is properly configured in android, the application will detect route to use to send data packets to the lan. Obviously if for example the android webbrowser can't access a website on your lan, do not expect the sip app to find the route... But if the network is properly configured it should work.
