csipsimple - issue #1774

Zrtp masquerading

Posted on Jun 6, 2012 by Massive Monkey

Would be nice to have support for ZRTP masquerading in csipsimple. I think this may have to come from pjsip, but all the more then in pjsip. There are already few clients supporting ZRTP masquerading support to allow working with many sip providers as current ZRTP is not working with them.

Comment #2

Posted on Jun 6, 2012 by Quick Hippo

Unless I missed something in it, it's a feature request for https://github.com/wernerd/ZRTPCPP It's the ZRTP library used by CSipSimple for ZRTP.

As far as I can tell : * I understand the pragmatic need of that, but it should not be required if ZRTP is really adopted by SIP servers. BTW, same problem would appear with SRTP if not supported by a sip/media server. => It's a hack that encourage lazy sip server implementers. -- and besides it will probably not solve the problem with all sip providers. * the only zrtp masquerading implementation I found is in Zorg implementation of ZRTP by PrivateWave that defined that zrtp extension hack. * it potentially break compatibility with other client supporting zrtp as per RFC 6189 (even if there is some tweak to make it compatible). * AFAIK, it's not defined in a RFC and I absolutely don't know how open/standard it is. It's probably not normalized and referenced as something to adopt by standard clients.

So, I would not be shocked if not introduced in GNU ZRTPCpp because not standard feature that needs to be normalized first.

Anyway, won't fix in csipsimple. Could maybe in ZRTPCPP/ZRTP4PJ (Werner's decision), unless I missed something on how it works. If done, then some new task to add settings in CSipSimple for this will be necessary, but for now as not inside zrtpcpp/zrtp4pj (AFAIK), not yet relevant to open such a task.
