csipsimple - issue #1163

Log to monitor transition from SIP De-Registration (enhancement)

Posted on Jul 20, 2011 by Massive Lion

Problem: SIP VOIP goes to unregistered state and VOIP calls are missed.

Possible Solution:

If there was a small log (array), in CSIPSIMPLE's settings menu, that recorded time pairs of de-registration date\time and the next registration date \ time, then I could calculate how long I was unregistered. The log could have 5 time pairs. Maybe add this 'downtime' to a counter for last hour, 24 hours / 7 days / month / lifetime.

Even though I have ‘Mobile Keep Alive’ set to 30 seconds, sometimes when I unlock the phone, it is not registered and is unavailable for VOIP calls (I call the VOIP number and it does not ring).

Maybe this log exists and I just need to learn more about accessing such a log.

Comment #3

Posted on Jul 21, 2011 by Quick Giraffe

Taking into consideration that CSipSimple has problems with its registration to SIP server, I think it's a good idea.

@raymond: There is no de-registration requests, sent to SIP server. Registration works this simple way - SIP client (CSipSimple in this case) sends REGISTER request to SIP server. In this request there is "Expires: 600" parameter. The latter specifies when SIP server will consider the registration expired. So, if SIP client sends the request now, the registration will expire in 10 min. After that time, if SIP client doesn't sent a new registration request - SIP server will drop any call requests to your SIP client (reason - "unregistered"). In CSipSimple there is special option (in expert wizard, I guess) to set that expiration time - "Register timeout (sec)". You may set it, let say to 600 sec and watch if CSipSimple sends REGISTER requests right on time (every 10 min). If it sends request later - that's the time when your SIP client is not registered with SIP server and, therefore, SIP server will not send new call to your client...

Watch your logs for regularity of REGISTER requests, sent to SIP server. Even better - watch (if you can) logs from SIP server. That log will tell you how reliable CSipSimple is (in terms of registration). And, BTW, I fond it completely unreliable so far - see "Issue 1136 - csipsimple - Registration problems".
