csipsimple - issue #2205

Hardware compatibility issue with Blu Dash 3.5

Posted on Jan 28, 2013 by Happy Monkey

I encountered an unusual hardware compatibility issue with a handset called the Blu Dash, version 3.5.

http://www.bluproducts.com/dash35.php

I installed the release from the Play store around October of 2012. I configured CSipSimple and it functions as expected. I bought a second handset with identical specs this month and tried to install CSipSimple and the Play store says my device isn't compatible with the app. I noticed there was an update release on Jan 15th 2013. I have a third handset (a Samsung Galaxy S3) with the Jan 15th release installed. I checked the version string on both the Blu and the Samsung and it is identical (0.04-04 r1916).

I'm stuck here. I don't understand why I can run CSipSimple on one Blu and not the other with matching specs. I also don't see the difference in the version string between the Jan 15th release (on the Samsung) and the previous release (on the Blu).

Thanks for your help.

Comment #1

Posted on Jan 28, 2013 by Quick Hippo

Hi,

Maybe a problem with the playstore itself (for some device it badly detects the features). CSipSimple normally requires only one feature from the device and it's micro. Apparently the Blu have it so it should be fine.

The first thing to check is that there is no other forked version of csipsimple installed on the Blu. (CSipSimple is opensource and distributed under GPL license terms which allow sip providers and other companies to rebrand it and redistribute it if they keep the GPL license on it). Once this is checked, you can directly download and install one of the nightly build version of csipsimple available here : http://nightlies.csipsimple.com/trunk/ As this is directly the application playstore will not complain about missing compatibility ;).

Note : If you never installed an android application from an apk before you'll have to enabled "unknown sources" applications in android setting (depending on the android version it will automatically propose to activate this option when launching the apk on the phone)

Comment #2

Posted on Jan 28, 2013 by Happy Monkey

Indeed it is an issue with the Play store. I copied the apk from a Samsung device and loaded it on the Blu with adb and it works. These Blu handsets retail US $99 and run Android 2.3.5. Interesting target platform for a low end SIP phone.
