Comment #3

Posted on Mar 31, 2011 by Quick Hippo

Ok, so I think that's a different problem if only on 3G.

In fact there is something that really matter in this case, it is codec used. PCMU/PCMA (aka G711) codecs is the most widely supported but is also the worse in terms of bandwidth consumption.

On mobile data connection just like you I get awful result when using PCMU/a codec. But that's absolutely normal. PCMU codec use 64kb/s bandwidth which is really too much when using over 3G when there is many people connected on the same mobile antenna than you. However, using better codecs such as SILK or GSM or speex 8kHz for example results becomes correct. The problem is that your SIP provider (or the remote SIP party) must support one of these other codecs else codec choose will automatically fallback to PCMU/A. As by default CSipSimple in narrowband try to announce almost all narrowband codec I fear that your sip provider does not support any of these cool codecs. If you want ton continue to use this sip provider, you should probably contact support of your sip provider to ask them for more codec support.
