csipsimple - issue #1638

Separate address book (per account) for Csipsimple

Posted on Mar 25, 2012 by Massive Monkey

This a feature request.

There should be an option to create a separate address book for each account, that has nothing to do with the main address book of android. There are numbers in the main address book of android that I call using the normal phone. Voip is different and there should be therefore (an option) to not have to mix these and make it cumbersome.

Comment #1

Posted on Mar 25, 2012 by Quick Hippo

Address book management is actually not a SIP feature + Android works very very well with their contact provider approach.

... think about that...

Actually I already did ;)... in nightly builds I started something to address your issue, but the way chosen is integrated to android : It will be possible to associate each account to an android contact group. This group can have contacts from every source you can imagine, synchronized in the cloud or anywhere else. + In android, you don't need to display all your groups in stock address book list : some groups can be hidden from contact app listing.

So the feature that will be implemented is : allow to associate a buddy list for a sip account to an android contact group. It already exists and for now is based on the display name of the account. In short future it will be possible to configure it.

This will allow anything you can imagine. Including sync with xcap source or with whatever you want. And besides a sip application can't reach the quality level of stock contact app management. So it will also provide a lower user experience to manage an address book inside csipsimple. And we can take advantage of the fact we are running android where contact API is part of the platform ;).
