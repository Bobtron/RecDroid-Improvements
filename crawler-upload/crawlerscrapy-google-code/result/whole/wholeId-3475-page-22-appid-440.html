Comment #1

Posted on Oct 22, 2011 by Quick Hippo

It's probably due to a change from skype. Actually what is done in the version of csipsimple distributed on the android market is a try to integrate a private api of the propietary software skype. I did reverse engenieering to try to integrate this close application, with a close protocol (BTW, it link all your contact to this service provider).

Now, with nightly builds, the integration of this application is made through a plugin. See my comment here : http://code.google.com/p/csipsimple/issues/detail?id=1228#c1

It will be easier to control and to update if the integration is made through a plugin for skype instead of directly inside the app. I didn't tested recently the integration with skype, but I guess they changed things. Hopefully they started to do things cleanly... was a shame that they didn't support "skype:" protocol while I'm able to do it in csipsimple... So I may have to do some new test with their latest version. But that's not priority at all, cause skype is a proprietary protocol that is not good for users freedom. So that's a nice to have for the skype call handler plugin ;), all the more so as skype can change their private api at any time and break all my efforts to integrate their app cleanly to the rest of the OS.

Can you test with nightly build and the skype plugin and tell me how it goes. Could be a better starting point for me than the test with the market version of csipsimple. But anyway, when I'll get time I'll do more tests with the skype plugin since if you currently get a crash, there is chance they changed things on their side.

Thx for the report.
