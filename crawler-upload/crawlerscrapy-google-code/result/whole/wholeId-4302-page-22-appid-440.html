Comment #14

Posted on Feb 4, 2011 by Massive Lion

I had a look to the third_party SRTP stuff: pjsip uses the original SRTP implementation from David McGrew (David is an author of SRTP RFC 3711). This implementation is around for many years now and servers a reference implementation (sort of).

I would assume that the asterisk/FreeSwitch code is broken somehow. Please keep in min that FreeSwitch reuses some Asterisk code IMHO it is save to assume that SRTP was just taken over. The Asterisk issue 18643 gives an idea what happens: the SRTP implementation does not handle RTP sequence number overflow correctly. The sequence numbers always start at a random value thus it is not really predictable when such an overflow occurs (except if the RTP implementation uses a predictable random number :-) )
