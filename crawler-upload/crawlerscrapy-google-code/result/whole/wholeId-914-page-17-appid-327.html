android-daisy-epub-reader - issue #76

NewStory: evaluate Travis.ci to build our new app

Posted on Jul 19, 2013 by Helpful Panda

We want to have Continuous Integration (CI) for our new codebase, currently hosted on github.com CI should reduce the risk of committing incomplete or broken code.

Travis-ci may be suitable and appropriate. See https://travis-ci.org for an overview.

Our build consists of 2 related projects, daisy-engine and DaisyBookLister. Both are built using ant currently.

Comment #1

Posted on Jul 21, 2013 by Helpful Panda

Here are some of the resources I found helpful: https://github.com/nasamuffin/FridgeMagnet - an example of a .travis.yml file and a buildstuff shell file. Note: this script isn't supported natively in Microsoft Windows. Something like cygwin should enable this script to run on Windows computers.

Our files are: https://github.com/julianharty/new-android-daisy-reader/blob/master/buildstuff.sh and https://github.com/julianharty/new-android-daisy-reader/blob/master/.travis.yml

These run and pass on travis-ci.org however they don't yet build the main app or run any tests. To do these steps we need to extend the build file. Which is the next thing we'll do...
