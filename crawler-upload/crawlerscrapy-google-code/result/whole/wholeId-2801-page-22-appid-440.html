Hi,

The problem could also be something with non routed networks.

The STUN option will not help if your sip server is hosted on same corporate network than you. Because STUN option by default uses STUN servers that are outside (on the public network) and will resolve the "public" IP of your device, which is useless for your sip server if also in your private network. So in this case, STUN option will break things instead of resolving things ;).

In this case (if there is a NAT between client and sip server), a good workaround is to have a STUN server hosted on your corporate network in same subnetwork than the sip sever. Because the STUN server will be able to resolve with correct address (the one seen by the server). Will not solve routing problems however.

To enlight me a little bit more could you follow these steps : http://code.google.com/p/csipsimple/wiki/HowToCollectLogs?wl=en (add in description this issue number : 2024)

It sends an email only to me so it's better for privacy than publishing it here. And if the sip server is able to route to the range of the sip client IP.

Also you tell that you tried with other sip software, is it working? If yes what is the configuration used?
