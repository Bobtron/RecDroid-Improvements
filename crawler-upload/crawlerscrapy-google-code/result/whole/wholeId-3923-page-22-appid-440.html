Comment #5

Posted on Apr 14, 2011 by Quick Hippo

In fact the native lib should have been loaded at least one time to avoid this error.

There is two reason why this lib can not be loaded : * The app is not connected on a network where it should load the lib. I this that in this case I should fix things like I did for codecs and just cache available codecs when lib start once.

  • The library is not compliant with the java code. This appear if you do your own build and did not update the .so file (and this can even appear if you do not refresh in eclipse the root project folder).

Comment #6

Posted on Apr 14, 2011 by Massive Rhino

r3gis,

I did a complete clean build from 792, including the lib. I even refreshed/cleaned the project in Eclipse and had the same result.

If it helps, here is what I found.

Fresh Install of built APK (no Wifi active), accepted default Easy configuration settings. Backed out of the create an account menu and attempted Network settings and it crashed.

Removed APK and re-installed. Activated WiFi and launched the app, accepted default Easy configuration settings. Backed out of the create an account menu and went into Network settings successfully.

So I think your first point is the resolution to the issue.
