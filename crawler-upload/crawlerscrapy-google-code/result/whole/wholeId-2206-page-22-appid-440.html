Comment #1

Posted on Feb 21, 2014 by Grumpy Rabbit

My observation is that the RTP stream has wrong TS with such reduced sample rate. Each 20ms packet increments TS by 320 (as it would for 16kHz sample rate) but according opus RTP draft it should add 960 (20ms at 48kHz) regardless the sampling rate used.
