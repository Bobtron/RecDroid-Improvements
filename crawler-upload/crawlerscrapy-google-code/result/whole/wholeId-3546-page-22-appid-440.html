csipsimple - issue #1262

Issue with phone ringing

Posted on Sep 6, 2011 by Swift Rabbit

What steps will reproduce the problem? 1. Phone will not ring each time when it is called over 3G network 2. Sometimes it work perfectly, but sometimes is not working at all 3. I've tried to call from different SIP phones and always same issue. When it finally appears to work properly, you can repeat calling and it will work for 5 minutes, then if you pause for a while it will not respond anymore. Then you have to call it once, then it might or might not ring on second chance.

I thougth there is some kind of network carrier firewall issue, but then I played with Keep Alive Packet STUN,ICE settings without success. I even set Keep Alive 3G packet very low =1 sec, but again without any success.

I've created accounts on free Betamax provider to test it, but always the same issue.

What is the expected output? What do you see instead? Phone ringing should be reliable in order to use csipsimple otherwise It does not have any sence.

What version of the product are you using? On what operating system? 3G carrier with proxy Samsung Galaxy S2 Gingerbread 2.3.3

Please provide any additional information below.

Please ask for furhter details if you need to resolve this issue.

Comment #1

Posted on Sep 13, 2011 by Swift Rabbit

After I spent so many hours on researching I found out what is causing a trouble.If you have similar problem try this. If you're using Asterix as SIP server you must have qualify=yes option enabled otherwise server will not keep alive mobile on 3G network. Even if you enable Keep Alive packet option in csipsimple it will not resolve a problem. Packet Keep Alive seems to me is not working as it should, it should be working like IP-Peering, but not as sending zero packets. I haven't trace packets to resolve this but it is what I suppose.After all now I can use it on 3G network and is available for call. Thanks!

Comment #2

Posted on Sep 25, 2011 by Quick Hippo

Ok, interesting investigation.

Actually the keep alive feature send 2bytes packets (\r\n). There is indeed to ways to do that (the other one is to send an OPTION). But that consume a lot of bandwidth and battery. (And besides, that's not yet support by pjsip -- the sip stack on which csipsimple rely ;) ). The goal of that is to awake the UDP socket. It usually works but could be not enough on some networks. So if your sip server supports some more agressive keep alive method it is indeed a good solution. Hopefully future updates of pjsip will have this feature. And if so, csipsimple will have this too :).
