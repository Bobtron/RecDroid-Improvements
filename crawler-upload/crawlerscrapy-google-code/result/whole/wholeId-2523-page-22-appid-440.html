Posted on Feb 28, 2013 by Quick Hippo

Thanks for the point.

However, the current Android sdk target of the project is not 17. See : https://code.google.com/p/csipsimple/source/browse/trunk/CSipSimple/project.properties

Understanding the difference between min sdk, max sdk, target sdk and linked sdk is very important in Android. Never change the target sdk of a project if you are not sure of what you are doing and fully aware of what it means.

Currently csipsimple targets the 16. To switch to 17, a full review of what it implies is required. It's not just about changing a value somewhere and fixing some obvious java compilation problems. The android target sdk has a real meaning in android and changing it will change the way the app will behaves on a 17 platform. So it's very important to check what it implies before.

For example if 17 implies one default value somewhere and that you target 17, this default value will apply, if you target 16, an android 17 will see you target 16 and apply default parameter of 16 instead of those of 17.

So, it will indeed some task to move soon (if it bring some new feature or enable some new stuff), but when will be time to do so, we'll review all changes in the sdk and in default behaviors so that it is sure there is no other changes to do (such as forcing some parameters with defaults changed if target is 17). And don't worry, when it will be time to do so, the changes necessary to have java build will be done (and done a safe way so that it remains backward compatible).
