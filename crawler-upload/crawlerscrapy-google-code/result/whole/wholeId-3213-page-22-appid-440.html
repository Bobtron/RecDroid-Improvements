csipsimple - issue #1635

Improvements of backup/restore accounts

Posted on Mar 21, 2012 by Massive Monkey

Its extremely useful to have the feature described below. Regis agrees and am opening this ticket from his suggestion.

thanks.

EXPORT/IMPORT - 2 types Very strongly needed is capability to do the following a) export / import : a single account or muliple accounts (using a list of checkboxes) b) export / import : entire csipsimple c) Make copy of an account : There are many settings in account configuration (some a bit redundant, but I am not sure). Creating another account should be easy by copying. Alternativey is to import, but during import allow it to change the name of the account. There after the user can modify the specific settings only.

If I have two phones and want to export from one and import into another it should work.

Comment #1

Posted on Mar 22, 2012 by Quick Hippo

I would also add that it could be interesting to exclude restore of global settings when restoring. As side note, could also be interesting to see what can be done with issue 957 when treating this enhancement.

Another solution would be a third party application dedicated to accounts management of csipsimple in order to target advanced users that modify their accounts frequently. Since that's not the case of all mainstream users, it could be envisaged to do that as a third party application. Decision on that will comes on how the user interface of mainstream users has to be modified to reach these enhancement. If all can be done using the new design guidelines of ICS (with multi selection of account, and action bar modification, could come on mainstream app). To be checked and ergonomically designed before implementation, I'll do some proposal here.
