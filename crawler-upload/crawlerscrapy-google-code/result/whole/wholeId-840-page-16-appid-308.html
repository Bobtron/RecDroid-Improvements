Ok, apologies for how long this has taken, but v.26 (which will appear in the market in the next day or so) has a fix for this. The fix allows multiple sender strings, separated by commas.

Integrating this matching with Contacts is proving to be a huge hassle, so i have implemented this fix instead.

As for the person who asked about wildcards, the matching uses String.contains(), so any part of the sender's phone number or email should work just fine, no wildcard needed.
