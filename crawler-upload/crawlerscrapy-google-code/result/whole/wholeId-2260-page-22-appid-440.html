csipsimple - issue #2576

Excessive background noise on Nexus 5

Posted on Dec 1, 2013 by Helpful Ox

What steps will reproduce the problem? 1. Install nightly (or release candidate) for CSIPSimple on Nexus 5 2. Enable Echo cancellation, WebRTC Echo mode, noise reduction 3. Make calls to echo back number (I use 909-390-0003) or to another available phone

What is the expected output? What do you see instead?

There is a great deal of background noise audible on the receiving end (doors closing, television audio, toilet flushes, background conversations). Echo cancellation appears to be working (manipulating the setting while the call is active demonstrates a change), but noise reduction appears to make no difference.

What version of the product are you using? On what device / operating system?

CSIPSimple version 1.01.00 r2337. Device: Nexus 5, Android 4.4, Franco Kernel r14 (same issues present on FauxKernel).

Please provide any additional information below.

I had initially thought that this problem was the result of my phone using the speakerphone microphone instead of the normal microphone for calls, but in reality under default settings the correct microphone was being used (same microphone used in normal calls). Regardless, I used Mode Audio API in an attempt to correct the issue, and it certainly did reduce background noise, but it turned out it switched the microphone from the normal microphone to the speakerphone microphone. Background noise was greatly reduced, but my voice was too quiet on the other end.

I'm curious as to whether this is an issue with the Nexus 5 or with Android 4.4 and just hasn't been recognized as an issue yet.

(adding notes from Google group topic here for reference)

I've tried the 8 kHz clock rate, and voice_communication was the option set already, but the problem persists. Echo cancellation under WebRTC mode appears to be working in testing, but enabling the noise cancelation option doesn't appear to do anything. I've tried every mic source and audio mode, some in combination to no avail. However, the IN_CALL audio mode produces the same effect as Mode Audio API.

Reducing microphone gain doesn't seem to help because it doesn't seem to alter what kinds of things are picked up (voice vs background noise).

I'm not certain if this is noteworthy, but every SIP client I've tried has demonstrated this same issue (Android dialer, Sipdroid, Zoiper).
