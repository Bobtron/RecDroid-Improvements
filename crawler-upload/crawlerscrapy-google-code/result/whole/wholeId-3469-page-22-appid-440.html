ok ! So it's more likely about the fact cyanogem is very recent on SE devices :).

They really do a great job, so they should not be blamed, all the more so as they don't have source code of stock audio drivers. I guess they will fix that soon on their side. In the doubt you should report them that there is buggy things with their audio recording implementation. They can ask me if they need more details about what I do, but that's just about using android 3.0 recording audio api and fallback to previous version if I get an error (here sounds there's no error at the right place in their implementation).

So, about settings you should try in priority in this case : * micro source to DEFAULT instead of VOICE_COMMUNICATION (it was introduced in 2.3 source code for voice over IP apps) * audio mode for sip calls to NORMAL instead of COMMUNICATION (same thing). * then play with audio clock rate if the two latest points does not help. (8kHz may be the simplier frequency, but may depend on driver and hardware).

Well, since this problem is reproducible on a custom ROM that is not yet stable for your device, I close the issue on my side (it is not a good idea to change the default setting for this device). However, keep me/ the issue list in touch of your tests. It is very very interesting as a knowledge base to know which are the correct settings for CM7.1 on xperia phones.
