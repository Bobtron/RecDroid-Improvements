Comment #1

Posted on Mar 9, 2013 by Quick Hippo

If the sip server in the middle acts as a media proxy it will break ZRTP path. ZRTP ensure it's end to end media and that nothing in the middle can listen. If the sip server in the middle is a media proxy it could virtually acts as a Man In The Middle. ZRTP doesn't allow that and doesn't start in this case.

So, at choice : - have a sip server in the middle able to transmit zrtp - have the sip server only being sip and no media proxy.
