csipsimple - issue #2211

No audio when talking to Acrobits Softphone via Push

Posted on Feb 1, 2013 by Helpful Monkey

What steps will reproduce the problem? 1. Set Acrobits Softphone to use Push notification 2. Put iPhone into sleep mode (by pressing the hold button) 3. Make a call to iPhone from Android CSipSimple 4. iPhone user accepts the call. 5. Call is successfully established, but neither users can hear audio.

What is the expected output? What do you see instead? Both users should be able to hear each other's voice. But both users hear no audio.

What version of the product are you using? On what operating system? CSipSimple: 0.04-04r1916

Android OS: 4.2.1

Acrobits Softphone: 5.3.2 build 1F56D iOS: 6.1

Please provide any additional information below. When the iPhone user is in Acrobits Softphone app, (namely, when the app does not rely on the Push notification to receive calls) and a call is established, both users can hear audio fine. So it is only when Acrobit Softphone user uses Push notification that both users hear no audio.
