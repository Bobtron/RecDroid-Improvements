csipsimple - issue #1452

Micro source & echo cancellation conflict on Android 4.0.3

Posted on Dec 22, 2011 by Happy Horse

What is the expected output? What do you see instead?

Since the update to 4.0.3 on my Nexus S, the audio quality has been very bad. It was fine before the update. Now there is lots of jitter on Wifi and 3g.

There is also no problem with the stock android SIP stack and the same providers.

Comment #1

Posted on Dec 23, 2011 by Happy Horse

This stops completely when echo cancellation is disabled. Could it be that Android 4 does some echo cancellation internally, which interferes with the one that CSipSimple does?

Comment #2

Posted on Dec 23, 2011 by Quick Hippo

Yep absolutely. On pre-4.0 the micro source was not set to the one that should have stock echo cancellation. Now with 4.0 it should have switched to this new micro source. And in this case indeed the hardware + the software echo cancellation will conflict with each other.

So you can alternatively : * Change the micro source to "default" instead of "in_communication" (see the FAQ in audio troubleshooting section to know how to tweak expert setting mode and get this setting). OR * Disable csipsimple software echo cancellation.

It depends on the result you get with each echo canceller. I guess that the hardware echo canceller is by far better ;).

I keep the issue as an open point cause I'd like to try to detect if the device has a native hardware echo cancellation on micro. Since that's not available on all hardware device, but that the micro mode should be available on all device, it would be better if I could detect and automatically activate or not the good option.

Thx a lot for the report and the feedback.
