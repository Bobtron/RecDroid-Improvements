Comment #1

Posted on May 4, 2011 by Quick Hippo

Can you try with latest nightly build? It should fix this problem. http://nightlies.csipsimple.com/trunk/

If not, using this version, try to activate "Use compact mode" in settings > network.

In fact usually that's due to the fact UDP packets are too big : when csipsimple try to negociate sip media, it sends big packets cause it supports a lot of codecs and besides it use the "verbose mode" (to ensure compatibility with all sip providers).

As usually sip provider does only support UDP, csipsimple by default force transport UDP but unfortunately some routers and network equipment refuse to transmit big UDP packets which can make broken the media negociation.
