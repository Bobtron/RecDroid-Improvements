csipsimple - issue #562

Auto-answer filter leaves Csipsimple running in background

Posted on Jan 3, 2011 by Quick Wombat

What steps will reproduce the problem? 1. Csipsimple app "ongoing" in background configured/registered with sipgate.com SIP account. 2. Csipsimple set to auto-answer incoming call from specified Google Voice number via filter. 3. "Google Voice Callback" used to place call via integrated dialer interface.

What is the expected output? What do you see instead?

The expected result of above said configuration would be that a call placed utilizing Google Voice Callback will be auto-answered by Csipsimple, which is running in the background, and the Csipsimple UI would be displayed to handle the auto-answered call. Though the configured filter does auto-answer the incoming call from GV, Csipsimple does not come to the foreground. Instead Csipsimple remains in the background whilst a notification stating "Call in Progress" is displayed in the notification area, leaving the user no means by which to control the call. In order to display the user interface the Android notification area must be pulled down and the "Call in Progress" notification selected, which then brings Csipsimple to the foreground.

What version of the product are you using? On what operating system?

Csipsimple version: 0.00-16 r410 Android 2.2.1

Please provide any additional information below.

The idea here is to bypass the extra step of the requirement to physically answer an incoming call from a specified Google Voice number and, instead, set Csipsimple to auto-answer the incoming call and present the user the interface to control said call.
