Comment #5

Posted on Feb 24, 2011 by Quick Hippo

pbxes.org advise to use stun.counterpath.com as stun server (as that is what they use by default in sipdroid they develop). It's also the default STUN server used in CSipSimple. So if you activate STUN server it should behaves correctly if your country is not in a private network (china, russia for example).

As for Gtalk client a stun server I have big doubts about that. That make no really sense. A STUN server is a server on the internet that when a client request it provide to the client it's IP seen by the STUN server. It allow a client to resolve its own public IP.

Something, now, since you are using pbxes.org (I guess a free account), I had a lot of issues with their service too. Saving something in their UI solved things. So could simply be something that where wrong on server side, and maybe now, if you re-activate STUN it will works properly. Also default STUN server (counterpath) is instable recently. I guess that's cause it is widely used by many sip clients. You could try other stuns server. Find a list here in Public Stun server section : http://www.voip-info.org/wiki/view/STUN
