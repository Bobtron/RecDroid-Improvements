Comment #15

Posted on Apr 17, 2012 by Quick Hippo

I can add it to the world wide section if you prefer. It depends on where the market is focused. It can be either a list of countries or the world wide section (not both).

In the case of list of countries it appears in the first section for users of this country. But will still be available for ALL users in 'other countries' section. So really depend on your market share/focus. I've no problem to change that, just let me know :)

About the port, does 5061 supports TLS (SIP over TLS)? If so indeed it's very useful and I can add a checkbox to activate TLS (secured transport) for the account. If it's just another UDP port it's probably less useful. (and btw, any user can easily switch the account to basic/advanced wizard once create with messagenet wizard). So never blocking point ;). If many users need the 5061 port instead of the 5060 port we can decide based on this feedback to switch to 5061 always or to add an option in the wizard. But I always resent making things complicate for mainstream users to address few users needs (I prefer to give support on how to change the remote port if necessary)

Don't hesitate to tell me if you'd like me to change improve something in the wizard. It's very welcome when sip provider are involved in the wizard creation since you know exactly features of your sip server. For example, the compact mode option, is often very interesting if supported by the server because reduce network usage. Codec priorities is also interesting to reorder based on assumption that can be done on what is supported by the server.
