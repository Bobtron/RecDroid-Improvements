csipsimple - issue #410

[Enhancement] Add the capability to filter on Wi-Fi spot

Posted on Nov 23, 2010 by Grumpy Elephant

Filter an account to register on selected hotspot. This is a way to use differntly a single SIP account from internal Wi-Fi Hotspot and external ones.

This may allow users to prefix or rewrite some numbers. For example +33610001900 can be used composing the full number from external network and rewrite it to 900 when connected on internal network.

Comment #1

Posted on Nov 23, 2010 by Quick Hippo

Should be done by a third party application. Read my comments on issue 267.

I've this kind of application in mind. But should be another project. I already all API in csipsimple to allow such a feature.

But the good design is not to try to implement it directly into CSipSimple but to do a powerful other app that contextually change settings (such as Locale, but in opensource ;) ), with a plugin for CSipSimple.

This is just a matter of design. It's not the job of a SIP application to be aware about context. And even better, a real application designed to completely achieve some contextual settings of the global android system would be really really more powerful than something integrated in CSipSimple.

Just imagine... An application that just handles plugins : with a plugin to be aware of wifi hotspot with another plugin that can activate / deactivate sip accounts You can perform what you want ! And better => you can imagine a plugin that is aware of your location as an input plugin... and just use the already defined SIP plugin to also be able to activate / deactivate account Or another output plugin that just change your ringing mode... Possibilities are infinite with this design pattern ! While integrating something in CSipSimple would make configuration much more complex, will not be useful for all users, will not be as powerful as the other app !

So I've this kind of application development in mind. Unfortunately my time is not infinite ;)... If some great developers decide to make such an opensource app, I will contribute with my CSipSimple plugins... In the future if I get time and nobody had the courage to develop such an app, I'll do it as an opensource app, just like csipsimple... But for this problem, directly inside csipsimple, response is no.
