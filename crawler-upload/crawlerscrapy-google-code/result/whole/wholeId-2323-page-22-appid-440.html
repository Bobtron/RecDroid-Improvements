Comment #1

Posted on Aug 19, 2013 by Quick Hippo

Je ne crois pas que freephonie authorise 2 sip client à être connecté sur le même compte simultanément.

Cependant, l'erreur 'request timeout' est généralement liée à un problème différent. En général cette erreur est le signe que le réseau refuse de transmettre les packets SIP vers les serveurs de Free. Il peut y avoir plusieures raison à cela. Le plus souvent c'est tout simplement le réseau qui a un "firewall" qui bloque purement et simplement les packets SIP. C'est le cas sur certains réseau/offres 3G; mais cela peut aussi se rencontrer sur certains réseaux wifi où il y a un firewall.

Il peut également arriver que le réseau supporte mal les packets UDP un peu gros. A priori la config a défault + un compte freephonie ne devrait pas produire des packets SIP trop gros et ça devrait passer avec tous les équipements réseau. Cependant avec une configuration modifiée, possible que ça arrive.

Pour déterminer la cause du problème, il peut être interessant de tester les deux appareils sur le même réseau pour s'assurer tout d'abord que ce n'est pas un problème de réseau sur l'appareil (par example une route vpn qui serait toujours présente). Il faut veiller pendant ce test à désactiver le compte sur l'appareil non testé afin de s'assurer qu'il ne se ré-enregistre pas pendant que l'autre s'enregistre.
