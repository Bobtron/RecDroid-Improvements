csipsimple - issue #598

Access sip call from contact list via additional button - enhencement

Posted on Jan 17, 2011 by Massive Cat

What steps will reproduce the problem? 1. Start contacts or dialer 2. Tap on the picture of the contact 3. Options (icons) show up for call, video call, sms, contact edit

What is the expected output? What do you see instead? There shoul be an own icon here for cspsimple. By hitting this icon the contact should be called via csipsimple. Comment #1

Posted on Jan 17, 2011 by Quick Hippo

It will be possible the way you think it in android 2.3 with the stock SIP support of the platform.

On android 2.2, the android platform does not support "SIP" calls directly (or other apps to register here if not an app that manage mail/sms/im protocol). -- technical limitation of the platform (but not sure that's a bad idea). However, on android 2.2 and lower it's possible using nightly builds (http://nightlies.csipsimple.com/trunk/) and filling IM of custom type "SIP" for a contact. BTW, CSipSimple will appear in list of quick action popup of a contact.

You can also disable android integration if you don't want the csipsimple outgoing call chooser, but I do not recommand cause using this way make it accessible to all apps (not only the stock contact app). You can also play with filters/rewriting rules feature of csipsimple which is a powerful way to improve your user experience and do not ask each time for choosing SIP or GSM (you add rules and CSipSimple choose automatically between GSM or one SIP account -- must call / can't call filters).

So I mark this request as wont fix cause : 1 - technical limitation of android the way you are thinking it 2 - will be automatically available on 2.3 when SIP protocol will be managed by android OS 3 - Already possible to workaround it several ways even in 2.2 and lower

Comment #3

Posted on Jan 17, 2011 by Quick Hippo

I don't really like this way cause it prevent from sending SMS or always ask you whether you want to choose between SIP or SMS and then as CSipSimple support multiple accounts CSipSimple will ask you what to do....

So, I think that the way Sipdroid integrate using sms is really bad. I much more prefer using Filters/rewriting rules and be integrated to standard phone calls. As a user you want to make a call not send a SMS.

The good and clean way is to integrate using SIP IM fields of your contacts, or to wait for android 2.3 that will automatically manage that. But integrating to SMS make no sense I think. It introduce bad habits for users and finally in my case with multiple accounts support introduce more steps for users than what I have currently.

The only reason I'll integrate one day to SMS will be to provide actual SMS feature (the already existing "Messages" feature).
