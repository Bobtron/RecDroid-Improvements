csipsimple - issue #1817

CSipSimple calling over Open Garden mesh network

Posted on Jul 5, 2012 by Massive Hippo

Open Garden lets you connect Android devices directly into a mesh network, and they have an add-on for Firefox which sets necessary proxy settings so one device can fetch files over multi-hop. It would be grand if CSipSimple could do this too, because then people could call each other at no extra cost. See https://play.google.com/store/apps/details?id=com.opengarden.android.MeshClient

Comment #1

Posted on Jul 8, 2012 by Quick Hippo

If I correctly understand the goal of this it's to just create some local network shared by different device.

In this case, you probably want to make direct SIP to SIP calls.

This is absolutely possible with CSipSimple with the existing version (and with nightly builds too http://nightlies.csipsimple.com/trunk/ that gives in screen indication on local IP).

Just configure a "Local" account (in wizard generic section), on each device. Then switch to text dialing mode (the little TXT menu button while in dialer) and try to dial : test@xxx.xxx.xxx.xxx where xxx.xxx.xxx.xxx is the IP of the other device. (the "test" username doesn't really matter you can put anything you'd like instead). (alternatively you can also dial sip:xxx.xxx.xxx.xxx)

You will find the ip in the local account interface so that it's not too hard to share.

Alternatively, you can have on sip server on any of the node of your network and register all sip clients to this node. This may be easier for users because no need to know IP of individial clients but only the one of the sip server node of your network. This is obviously free and you just need to install a sip registrar on the network. You can have a look to freeswitch project for example, it's pretty easy to install.
