csipsimple - issue #2101

The dialer part is shown even after uninstall

Posted on Dec 3, 2012 by Grumpy Rabbit

What steps will reproduce the problem? 1. Install csipsimple on Huawei Ascend G300 with Android 4.0.3 2. Choose integradte with native dialer 3. When a call is done with the native dialer in the the last step the csipsimple screen with the red banner appears where you can stop the call 4. Uninstall csipsimple 5. When a call is done with the native dialer the csipsimple screen with the red banner still appears instead of the native one 6. Reset the phone to factory defaults 7. When a call is done with the native dialer the csipsimple screen with the red banner still appears instead of the native one

What is the expected output? What do you see instead? When uninstalling the dialer should be completely removed and the native dialer screen with the bluetooth button should appear.

What version of the product are you using? On what operating system? csipsimple 0.04-04r1916 Huawei Ascend G300 with Android 4.0.3

Please provide any additional information below. I upgraded the phone's OS to 4.0.3 and installed the csipsimple from a local copy not from the play store. But then I tried many installations and uninstallations with various combinations without success. In the last step I need the native screen since I want to choose bluetooth handset sometimes and csipsimple doesn't have this option.

Comment #8

Posted on Dec 3, 2012 by Quick Hippo

Yes the new interface of 4.x is very similar to csipsimple one. There is some difference you can observe to distinguish each other.

For example the dialer : the stock one : http://cdn-static.zdnet.com/i/story/40/10/025057/android-ics-3.jpg csipsimple one : http://www2.r3gis.fr/CSSPublic/screen4.png

CSipSimple can support tablets, has more tabs, has an icon to switch sip account etc. For the rest graphics are identical. It's all taken from android stock source code. That's the reason why it's similar but actually not the same and completly rewritten inside csipsimple (in csipsimple it also run on all android versions)

For the call control the stock one should look like : http://reviews.cnet.com/8301-19736_7-57325419-251/android-4.0-ice-cream-sandwich-review-cool-confident-chaotic/ (see at 2:03 of the video) And csipsimple one is like the link you posted. Again, it's all rewritten. CSipSimple doesn't reuse/ add something to stock call screen application. The call screen you see in csipsimple is deeply inspired from stock 4.x one and reuse some graphics from android opensource code; but it's a different code. There is some differences : like the possibility to manage more calls, an additional toolbar on the top of the hangup buttons for action on the call itself and the bottom toolbar is only dedicated to media actions that are not related to a particular call.

So there is no way csipsimple (and any other app) could modify the stock call screen. So it couldn't be responsible for the presence or not of the BT button in the stock call screen. Maybe the rom you have is buggy with BT for stock call screen. But anyway not a problem with csipsimple (maybe even, csipsimple can use BT while the stock app cannot ;) ).
