Comment #6

Posted on Apr 2, 2011 by Quick Hippo

The fact you can't end call and you can hear the other side is probably linked.

Most of the time it means that something goes wrong at some state of the SIP invite.

If you say that it worked and suddently does not work anymore, my think is about length of UDP packets. CSipSimple support many codecs. The bad side of that is that it make UDP packets bigger. Some routers does not allow big packets in UDP.

Fortunately, in nightly builds there is a new feature to reduce packets size. (It's in network settings > compact mode). You can also disable some codecs. I'm almost sure that your sip provider does not support SILK for example, so useless to announce it in SIP Invites :). You can just disable it in codecs list.

Quick note about nightly builds, as their name suggest, it's rebuilt each night, so "latest" always evolves ;).

Also, I'd suggest you to activate STUN (I fixed things in nightly builds so that stun does not fail after some usage). Sipdroid activate it by default. I do not cause in some country it provide a very bad user experience.

Comment #17

Posted on Oct 6, 2011 by Swift Hippo

To bring these comments full circle, I have the same issues as the original poster but was able to resolve it through changes to the available codecs. I also use pbxes.org to and Google Voice, but on an LG Vortex. My understanding is that anyone using pbxes.org likely has encountered this problem. Outgoing calls work as expected, but with incoming calls the other party can not hear me.

The change required was removing some of the available codecs, as suggested by r3gis. I just kept removing a few until it started working. I also tried to enable the "SIP Compact" option, but that prohibited CSipSimple from registering with pbxes.org so I left it disabled.

Here are the codecs I currently have enabled:

Wideband: G722 16 kHz, PCMU 8kHz, PCMA 8kHz Narrowband: GSM 8kHz, PCMU 8kHz, PCMA 8kHz

I can have incoming and outgoing audio for both calls I make and calls I receive.
