Comment #1

Posted on Jan 8, 2011 by Quick Hippo

my@login@voiprovider.com is not a valid sip contact address (regarding SIP RFCs).

It's possible as authentication name but not as username.

I think that your username and your authentication name are different. If it's the case you have to use expert account wizard. The only way to have "@" char in it's sip contact address is to encode it properly, which is (I have to check but quietly sure) done by csipsimple... but I think that it's not the case.

What is your sip provider? Maybe I could add a wizard to ease things cause using expert wizard require some SIP skills.

Yes in you authentication name it's possible but in your username not. I mean if somebody wants to dial you, dialing "my@login@voiprovider.com" will not work at all.

my%40login@voiprovider.com is valid.

In authentication name it can be something absolutely different from username and here @ is valid (even if not encoded).

What you can try : Transform you account into an expert account (long press on the row, choose wizard, select expert). Then edit the account (you'll have a lot of new values :) ). Edit Sip ID field and replace by .

I'll fix the fact basic wizard do not automatic char encoding cause it should.
