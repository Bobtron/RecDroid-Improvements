csipsimple - issue #1805

outbound proxy RFC5626 may cause delay in registration and call set up

Posted on Jul 1, 2012 by Swift Ox

What steps will reproduce the problem?

1.enabling RFC5626 with certain providers 2. 3.

What is the expected output? What do you see instead?

Immediate registration; no delay in setting up call. Long delays in both.

What version of the product are you using? On what operating system?

r1660 on Samsung Galaxy Nexus running Android 4.0.4

Please provide any additional information below.

Latest nightly r1660 appears to have enabled RFC5626 by default. Pbxes.org seems not to work well with that. I got long delays in registration; in addition, when a call is placed, it takes long to connect and sounds a bit garbled. Disabling RFC5626 seems to cure the problem.

Comment #5

Posted on Jul 2, 2012 by Quick Hippo

Oh, ok :) No problem thanks for the update. So I'll revert the default value for next build but keep in mind it's a possible root cause for issues. If you reproduce don't hesitate to tell me. I've no strong position on the fact this option should or shouldn't be enabled by default. Normally it's harmless (unless buggy server), and could help a lot to traverse NAT topology if the server support it. So I guess it's good. BTW, in the underlying sip stack (pjsip) it's enabled by default. But I've no idea on how many potential buggy server there is regarding this feature. Usually in this kind of situation I rely on feedback on see if it helps for mainstream sip providers. As I can also selectively enable/disable features automatically thanks to wizard another option is to disable it only when the sip provider is known to have the problem. Here I disabled it because was the second time a server were known to have weird behavior with this option. The other one was some hand written server running windows... so I did the exclusion only on the sip provider, but here as pbxes.org is mainstream, I thought it was better to disable it completely by default.

But if in this case was just a pbxes.org server temporary problem... quiet normal. It happens very often for free accounts. Sometimes users has to go in the web interface to re-validate account (even if no mail were sent).

So don't hesitate to ping here (even if the issue is closed), if you encounter problems again.
