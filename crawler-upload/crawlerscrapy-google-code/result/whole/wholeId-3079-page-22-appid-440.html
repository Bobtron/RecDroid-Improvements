csipsimple - issue #1703

ENUM services support required

Posted on Apr 25, 2012 by Happy Giraffe

It would be nice if you'll add ENUM support, so we can use e164.arpa and OTHER enum services (with service list edit capability) in our android phones.

Comment #1

Posted on Apr 25, 2012 by Quick Hippo

Well, for e164.arpa, since it's not sip specific, it would be better if done by another application that integrates cleanly to android system. If this application exists, it will not only works with CSipSimple but with all SIP clients and could be integrated a better way in Android.

Android OS is a good piece of software design that allow apps to communicate with each other. So apps should take advantage of it and do one thing and only one thing but do it good. And then allow other apps to integrate to it using apis and intent system.

So even if I personally do the work, it will not be inside csipsimple since could benefit users that doesn't use CSipSimple but other opensource (or not) SIP applications (and maybe also other protocol).

When you mention "other" enum services. Have you a complete list (with related RFC if possible)? Could it be done in third party app ?

Usually if it's not directly linked to sip protocol changes or extension, it can be done by third party app. CSipSimple is designed to allow third party apps to manage accounts and make sip calls (if this other app has rights to do so of course ;) ).
