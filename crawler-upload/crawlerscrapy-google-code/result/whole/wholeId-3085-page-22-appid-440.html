Comment #7

Posted on May 1, 2012 by Helpful Cat

I think I found the problem. I have two installations of Ubuntu 12.04 (64-bit), one with ia32-libs installed and one without ia32-libs installed.

On the system with ia32-libs I run:

ldd ~/android-ndk-r7c/toolchains/x86-4.4.3/prebuilt/linux-x86/bin/i686-android-linux-gcc

linux-gate.so.1 =>  (0xf7795000)
libc.so.6 => /lib/i386-linux-gnu/libc.so.6 (0xf75d3000)
/lib/ld-linux.so.2 (0xf7796000)

Running ldd on the system without ia32-libs shows:

    not a dynamic executable

/lib/i386-linux-gnu/libc.so.6 and /lib/ld-linux.so.2 are both provided by package libc6:i386

So obviously trying to build ffmpeg without ia32-libs installed is going to fail. I assume that this is true for all of the other native libraries as well.

Regards, Paul
