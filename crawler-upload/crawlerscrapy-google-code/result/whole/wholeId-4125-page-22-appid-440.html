Comment #3

Posted on Feb 1, 2011 by Quick Hippo

Thanks for logs.

As for the "force close" in fact was an ANR (application non responding). I'll try to debug that.

Regarding other logs what's strange is that invite is sent but there is no replies from the sip server. That's really weird. As consequence the SIP stack waits for a reply from the server and timeout long time after to be sure there is no replies and provide a consistent state to the other side.

The strange point is that it seems to work on the second try you did while what is sent by CSipSimple is exactly the same. This kind of problem are really hard to debug. Sometimes it's due to something that happens wrong on the other side but could be due to something in the pjsip stack configuration too. As it comes after the upgrade I think more about something on CSipSimple side... but I don't know really what could be the faulty setting I changed recently :S
