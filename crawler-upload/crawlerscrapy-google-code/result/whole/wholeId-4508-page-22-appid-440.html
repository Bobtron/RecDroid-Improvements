csipsimple - issue #329

Optional on-demand account registration

Posted on Oct 30, 2010 by Grumpy Giraffe

It should be possible to select whether account should: a) register automatically on csipsimple start and listen to incoming calls b) register just before placing outgoing call and unregister when call finished

I only need to use sip for cheap outgoing calls so no need to drain battery and waste network bandwidth in background all day.

Now I have to manually exit from csipsimple after each call.

Another case is when someone has more accounts and he wants to share one/some for outgoing calls (and concurrent registration is not permitted).
