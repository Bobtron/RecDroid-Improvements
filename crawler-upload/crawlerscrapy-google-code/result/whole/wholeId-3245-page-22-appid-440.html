Comment #2

Posted on Feb 26, 2012 by Quick Hippo

CSipSimple already has a rewrite number feature. As far as I understand, it overlaps features of Right-number application. That's why both are not compatible, because they do same thing actually :).

Also csipsimple allow some plugins to register the rewriting engine to allow for example web callback, or call card features.

All rewriting and filtering rules can apply to gsm, sip and plugins. That's why you normally not need Right-number if you use CSipSimple.

In fact, both apps technically can't work together due to the fact both register the same broadcast of android system and only one can catch the broadcast. So it's something that can't be fixed both in Right-number nor in CSipSimple.

Comment #4

Posted on Mar 8, 2012 by Quick Hippo

Indeed it's another way to integrate. However I didn't choose this one for a very good reasons.

With this kind of integration, first it's not seamless for users. It requires to show a chooser that always displays. And secondly and very important, it doesn't allow to do clever filtering rules.

With csipsimple installed and the normal dialer always selected as call mean, the application will automatically be able to choose for you the mean you want to use (a sip account, gsm or any other apps through plugins). It can be based on the current network connection and on the number you are dialing. Viber can't do that, and will prompt user each time you make an outgoing call from your contacts.

In CSipSimple my approach is to tell users to choose "Use dialer" and remember this choice. This will raise CSipSimple outgoing call chooser that can manage filtering/rewriting rules. CSipSimple outgoing call dialer is also integrate other apps. For example there is already a plugin for Skype. I think that it would be easy to do the same for Viber and for any other application. And even better, with that approach, even the integration to plugins such the skype plugin will benefit of rewriting/filtering rules of CSipSimple !!! So if you want to tell csipsimple to always use skype for phone numbers starting by +33 for example, you can do ! And if some plugin is done for Viber (as the plugin I did for skype integration), you will be able to also add a rule to always call using viber if it starts by 06 for example, or add a rewriting rule, or anything you can have in your mind that you'd like to automate.
