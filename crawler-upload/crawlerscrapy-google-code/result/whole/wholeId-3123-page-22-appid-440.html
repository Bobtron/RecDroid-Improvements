Comment #1

Posted on Apr 23, 2012 by Quick Hippo

The goal of CSipSimple is to encourage people to use open protocol to communicate. Supporting OCS server or any other proprietary or self normalized extension of SIP is not the goal of the project.

It would be possible however through extension but I leave microsoft engineers or addicts developing that. I consider that contributing this kind of extension will benefit Microsoft first before users (and their freedom). As I do the project on my own free time, I focus on features that could benefit all users. Besides, AFAIK, some of the OCS extensions to the SIP protocol have to be licensed in the client (or assume final user already own the license).

I hope you understand my position ;). If you want to contribute this kind of extension you would be welcome however, and can join the developer group. I'll help there just like I help everybody else trying to build on the top of csipsimple respecting the GPLv3 license of the project ;).

For reference, the discution about OCS on the pjsip (the sip stack used by CSipSimple) mailing list : http://lists.pjsip.org/pipermail/pjsip_lists.pjsip.org/2007-December/000941.html
