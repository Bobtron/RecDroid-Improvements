Posted on Feb 11, 2011 by Quick Hippo

There is something I'm thinking about....

You have CM7 on a desire HD. I suspect CM7 to not be compliant with the full android 2.3 cause of the bad of google.

There is something called PSP behavior that link the wifi card speed of packet reception with the fact the screen is on or off. However in android 2.3 there is a programatic way to fix this problem. Previously there where a workaround (keep awake while in call), that forced the screen to remain on for some devices (HTC desire hd included). But with android 2.3 assuming all ROM will implement the new option to lock wifi even if screen goes off, I disabled the workaround.

I think that CM7 does not support it fully yet. Cause of the fact : * Google has not announced that officially while it's actually in the source code * HTC has not released drivers for the wifi chipset to allow to manage this problem.

Maybe worth to give the information to CM7 developpers. You should link this android issue : http://code.google.com/p/android/issues/detail?id=9781

My last comment explain exactly the change made in gingerbread and that I early adopt cause it worked great on Nexus S.

If that's what you experiment... I understand that you think that CSipSimple call quality is awful, and that it is unstable... but if you try with screen always on, or if you can get a CM7 build with PSP mode disabled or with the support of the new wifi lock (or a CM6 build that is in android < 2.3, and so where I apply the keep awake while in call workaround), you'll see that it has nothing to do with what you are currently experimenting. :)

Comment #3

Posted on Feb 11, 2011 by Happy Rhino

CM7 does support it. There's no difference between CM7 and stock Android 2.3 in this respect. The new lock option is implemented by passing a generic power mode option to the driver via wpa_supplicant. The effect this has depends on the driver and chipset. So far the only driver/chipset that I know for certain has the PSP problem is bcm4329, and for bcm4329 in CM7, this power mode option does work and does solve the problem. The Desire HD uses bcm4329. Verifying that it works as expected by trying with and without "Keep awake while on call" is still a good idea, of course.
