Comment #1

Posted on Sep 10, 2012 by Quick Hippo

Is stun option enabled?

STUN will resolve your "public" IP and use it for media so that you are reachable from outside. Obviously in case of a sip server on a local network it's a problem.

Several solutions are possible to solve that. The easiest is to disable STUN. However, if you have other sip accounts on public network, you can configure your account (with expert wizard mode -- you can convert the account to use expert wizard by long pressing account row). And disabling STUN option for this account. Another solution is to host a stun server on your sip server so that it always give to the client the good ip (if from outside or from inside).

Comment #6

Posted on Sep 10, 2012 by Quick Hippo

Indeed. That's unfortunately by design of pjsip (the opensource sip stack csipsimple rely on) : http://trac.pjsip.org/repos/wiki/FAQ#multihomed

What you could try (not sure it will work -- depends on the remote side) is to activate ICE option (without stun). If your remote side supports ICE as well, the SDP should contains candidates with each interface of the device. The remote side will then be able to try each candidate and find the good one.
