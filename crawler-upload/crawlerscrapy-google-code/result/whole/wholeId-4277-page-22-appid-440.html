csipsimple - issue #499

"My Phone Number" field in settings gets blanked.

Posted on Dec 14, 2010 by Swift Kangaroo

What steps will reproduce the problem? 1. Install and use CSipSimple (I am registered to pbxes.org and also sip2sip.info) 2. Reboot the phone (this step is required)

What is the expected output? What do you see instead?

I expect no change to my SMS/Messages application.

What I see is that it blanks out my own icon on existing message threads, though does not break any of the sending or receiving functionality.

If I then go into Settings -> Call Settings -> Additional settings -> My phone number. I've repeated my test sequence a number of times - sometimes the field is blank, sometimes it's filled with my regular number. In either case, if I either hit "OK" or enter my number and hit "OK" then the Messages app re-associated my own contact record with the sms thread and includes my icon.

What version of the product are you using? On what operating system?

CSipSimple is 0.00-16 r410 Android 2.2.1 (Cyanogenmod 6.1) on HTC Aria

Please provide any additional information below.

I have also seen this, or something like it, happen on SipDroid which I have since uninstalled.

I suspect it had something to do with setting of caller-id on outbound calls that overrides the cell phone's own configuration.

I have seen this happen with all SIP providers marked inactive. So, boot, confirm problem, fix problem in settings, open csipsimple (but leave all providers inactive and without making or receiving any calls), reboot.

Comment #1

Posted on Dec 15, 2010 by Quick Hippo

Mmmm, sounds really not linked to the SIP app you are using nor to the SIP app, but more to the ROM / hardware

The Caller ID feature in SIP has really no thing to do with the GSM caller ID. It's transmitted in SIP packets while your phone number is something that is get back from your SIM card. So in dev terms there is no link between this two things. For example a SIP application can run on a Tablet were there is no GSM feature (nor phone number), so there is nowhere in my code where I read nor write this value. Besides, CSipSimple doesn't ask the right to modify that (there is a security feature list) and even if CSipSimple ask for a lot of feature, it doesn't ask to modify nor SIM infos nor your base GSM info (all the more so as I think that this one, no apps can modify it unless it has a root right).

Besides, you guess that I (and many other CSipSimple and SipDroid users) reboot their phones and does not have this problem ;)...

The first thing that you can try is uninstall CSipSimple and SipDroid and reboot and see if you have not the same problem. I think that you'll reproduce exactly the same.

So it's maybe your SIM card that is not correctly plugged or something in the ROM, (but for that you may ask on Cyanogen's forums) or something with the hardware that should read the infos in the SIM card at boot time.
