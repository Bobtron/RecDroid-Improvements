Comment #6

Posted on Jan 29, 2011 by Quick Hippo

For ippi it's a simple GET on an URL. They compute a hash based on password, and current date time. ( http://code.google.com/p/csipsimple/source/browse/trunk/CSipSimple/src/com/csipsimple/wizards/impl/Ippi.java )

Something based on http is fine for me. This kind of thing is easy to do, so if you have already something working not a problem to adapt. Regarding source code of ConnectorBetamax, seems to be a simple way to do it. There is a https get to a given url with params username=$username&password=$password and the https server returns the balance. As it is https it's fine sending directly username and password like that.

I think that the return of the webpage could be directly displayed on the top of the wizard. So that sip provider could add information he'd like to add. Current balance but why not other info related to the current account.
