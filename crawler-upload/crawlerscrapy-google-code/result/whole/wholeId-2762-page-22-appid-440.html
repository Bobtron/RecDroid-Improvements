Comment #3

Posted on Oct 31, 2012 by Quick Hippo

Ok about tcp, tls I'll not change default value as it's not used in your case ;).

For the icon, I'm not able to access the google doc file. Can you send it directly by mail (r3gis.3r at gmail.com); or use the share option of google docs (seems the link is the link for editing the file and I guess that only the owner can edit it). As I guess it's currently time for sleeping in your timezone I'll use the icon from your website for tonight nightly build, but as soon I have the logo you'd like to see, I'll do the update :)

For 3G, in my opinion, if enabled by default you'll have other more angry users crying about the fact the app drain battery over 3G. I had that kind of feedback previously when 3G was enabled by default. And now I also have the same requests than you about 3G not activated. I guess that it's impossible to have a setting that satisfy all :).

But it's less a problem for users with option disabled by default and the first start setting view that allow to quickly setup this behavior. Most of the time users providing feedback about lack of connection on 3G, remind that they didn't read very carefully the very first page where it was proposed to them. It's affordable for them to go again in settings and press the checkbox they missed the first time they launched the app ;).

So if the decision is based on some user feedback, from my experience on users feedback, you'll get contrary requests. On the other side, if the decision is from the fact your 3G network coverage and quality is excellent in Japan and that all mobile carrier allow users to call over VoIP, it's a good reason. But if not, it's better to be sure user acknowledge he wants to try over 3G even if quality can be worse or could not work if blocked by the mobile carrier.

It's always better to have users to do a manual action to activate a mode that can possibly not work rather than assuming it will work and get users for who it doesn't work get very frustrated without understanding why it doesn't work.

You can also do some pedagogy when you have users asking for using your service over 3G which could be not so bad to inform about quality issues, battery drain faster, extra data usage (if limited in japan). One user informed is usually less frustrated than one that just get bad quality without knowing why; or than someone that get fast battery drain without knowing why.
