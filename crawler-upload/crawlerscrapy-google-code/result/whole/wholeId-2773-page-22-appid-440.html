csipsimple - issue #2057

Cannot Record Phone Calls (ICS 4.0.3)

Posted on Nov 5, 2012 by Helpful Rabbit

What steps will reproduce the problem? 1. Switch on Automatic Call Record OR 2. Switch on Manual Call Record

What is the expected output? Recording should appear in CSipSimple Recordings folder at end of call.

What do you see instead? No recordings are seen in CSipSimple Recordings folder

What version of the product are you using? On what operating system? CSipSimple 0.04-04 r1916 Android 4.0.3 Huawei Ascend G300

Please provide any additional information below. Recording worked perfectly with Android 2.3 on the same phone immediately before upgrade to ICS 4.0.3

Ok, good, so it explains the problem :)

I basically use this method : http://developer.android.com/reference/android/os/Environment.html#getExternalStorageDirectory%28%29 to retrieve the folder where to store.

I guess that the Huawei update to 4.0 changed the return of this method to the "internal" sdcard. Which is maybe not a good idea from user experience point of view. (as described by google doc the folder returned here should be : <>)

So I would say that if it's hard to access this folder for you on the Huawai, I think it's a problem on the new Huawai ROM.

On my side the way to address this issue would be to have another application dedicated to "call record" management. There is already the API for that in csipsimple and any developer can already start developing a dedicated app for managing "call records". But sounds I'll have to do that too ;)

Ideas of such an app would be to allow to compress (using better format than wav), rename, and organize call records by different ways than the default app and play these files (and maybe edit these files and send too).

Inside CSipSimple, I want to keep the feature simple and allow one (/several) third party application to manage these wav files. I want to provide an extensible API for dedicated apps and have just SIP features (or features that couldn't be done outside of the app).

Comment #5

Posted on Nov 5, 2012 by Quick Hippo

Great; thanks for the update. Very interesting for me to know some ROM has such a setting :). I didn't knew.

About having csipsimple on sdcard, you're right better to keep it on internal SD card. Some users reported problems when moving it on SDcard. (Often with the right of the app to use network). I have explicitly allowed the app to be installed on sdcard, but I usually advise to not move it. It suitable only in case you use it very rarely and don't care about reliability. That's the case of some users and that's why I leave the choice :).

BTW, about this topic, some more information you might be interested in: You can probably contact developers of apps that don't move to sdcard. It particularly makes sense if games. So you can ask them to add : android:installLocation="auto" in their manifest. By default, if this is not added, the application is forced to be installed to sdcard (and only way, as you said, is to root the phone). I don't think it's a change in ICS however; was probably also the case previously ;). But, some developers may have real reason for not allowing that : There is a list of case provided by google that tell us when we should or not allow moving to sdcard : http://developer.android.com/guide/topics/data/install-location.html#ShouldNot It's developers webpage, but as user can be interesting to read too :)
