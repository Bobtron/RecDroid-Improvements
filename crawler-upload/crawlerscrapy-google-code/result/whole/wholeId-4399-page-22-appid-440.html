csipsimple - issue #426

add boolean operator to rule/filter expression

Posted on Nov 27, 2010 by Massive Ox

suggestion : it would be nice to have an OR (an AND) operator for writing rules in the Filter menu.

I have : rule#1 : phone number starts with 01 rule#2 : phone number starts with +331 rule#3 : phone number starts with 00331 rule#4 : phone number starts with 02 ...

I'd like to have : rule1 : phone number starts with 01 OR +331 OR 00331 OR 02 OR ...

it would be easier to read and manage

Comment #1

Posted on Nov 27, 2010 by Quick Hippo

You can already do that with a regular expression on a single rule. something like :

^(01|+331|00331|02).*$

I prefer to leave things like that cause indeed it's more complicated to create a regular expression, but introducing something able to manage AND/OR merges would not be user friendly and the app would become something for only power users. So my approach : If you want to make and+or, you are probably a power user, so you can use regular expressions ;). For mainstream users I want to keep things as simple as possible.

Comment #3

Posted on Nov 28, 2010 by Quick Hippo

Tu as choisi que c'était un type 'expression régulière' dans la drop box au dessus? Si tu choisi begin with en fait ça ne fonctionne pas les caractères de regexp en effet, mais si c'est une regexp ça devrait fonctionner. Par contre après reflexion ça devrait plutot être : ^(?:01|+331|00331|02)(.*)$

(car le premier groupe capturé est celui gardé pour la récriture, il ne faut donc pas capturer le premier groupe d'où le ?:)
