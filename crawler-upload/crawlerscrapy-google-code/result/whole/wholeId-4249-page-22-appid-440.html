Comment #1

Posted on Dec 26, 2010 by Quick Hippo

My first thought is that as it's an HTC you are maybe affected by the PSP behavior. (When screen goes off wifi data packets are not received as frequently as it should).

To workaround that there is an option in CSipSimple : Activate ExpertSettingMode (wiki page => for global settings), and in User interface > activate Keep awake while in call.

However, I'm not sure that it will help cause in this case sound become really choppy but you get sound. So there is maybe another reason.

Another thing I can advise to you is to force UDP. (in market version by default it is set to Auto : UDP if short packets TCP if big packets, but your server doesn't support TCP so you are probably experiment some latency while initiating call). To do so, you can either : * install latest trunk version (http://nightlies.csipsimple.com/trunk/) delete and re-create your account with the wizard you prefer (basic wizard?). /OR/ * Switch to expert settings for account (ExpertSettingMode => for account settings), and in Transport choose UDP instead of auto. It will probably not solve the initial issue but will be better.

And last point that can more probably help : Some wifi network doesn't like my default RTP port. You can change it in Settings > Network > RTP port. Change from 4000 to something higher (for example 8000)

Let me know how it goes.

Comment #3

Posted on Dec 26, 2010 by Quick Hippo

What solution worked ? (maybe I can change default settings if it's the first or the last point ;) ).

For audio quality several things can be tried : * Increasing audio frequency if you have a high CPU or reducing it if low CPU. * Forcing a better codec (a codec that match more your bandwidth, for example on low bandwidth prefer GSM on high bandwidth (very high) you can try G722 (HD codec). Some other codecs has good compromise (iLBC or G729 - this one you need to pay the license of the codec)) * Disabling echo cancellation if the CPU is overloaded. * Increasing/decreasing media quality setting in media settings (expert setting mode). * And testing the last trunk version (I've increased the buffer size which may produce better results).
