Comment #14

Posted on Nov 7, 2013 by Quick Hippo

@maxim : might be linked to packet size. Using UDP as transport there is a common problem for everybody: udp packets sizes are limited by some network. The size limit is not predictable and in case a network equipement in the middle decide to not transmit a packet the app is not informed. In this case it will timeout and never get the confirmation from the server that the call request has reached the sip server (and so the answer button does not disappear as not taken into account by the server yet). Using TCP as transport, there is a problem on official nexus 4 rom (was the case for 4.0.x to 4.2.. I didn't tested for 4.3 yet). The effect was the same than the previous (with UDP) but the root cause was slightly different (a bug in nexus 4 rom)

For these reasons, in order to solve this kind of problem a common fix is to try to reduce sip packet sizes. To do so : - try to reduce the list of enabled codecs to the minimum you need (in settings > media > codecs) - try to enable 'compact sip mode' (in settings > network)
