Comment #9

Posted on Sep 18, 2013 by Quick Hippo

I think when you add this feature it would be a good idea for the user to be able to choose whether the callerid name and number they provide should be placed in the P-Asserted-Identity or the P-Preferred-Identity header.

For us we would prefer it be put in the P-Preferred-Identity header as thats most accurate place for it. However I know Asterisk for example doesnt support P-Preferred-Identity and if you tel it to trust the pai headers it will pull the callerid from the P-Asserted-Identity header.

Just a bit of background of how these headers work in the telecomms world in the UK (NICC standard ND1017) you can read on or just ignore if you are not interested :)

In the UK systems have two different telephone numbers. There is the Billing number which can also be known as the line number or ANI number. If you have an ISDN30 line for example and multiple ddi numbers then the ISDN30 line has the first number and you can present that number or any other DDI number that you own. When a call is sent to another telco all this information is in the SS7 UK-ISUP packets. The ND1017 states what should happen when the ISUP call gets translated to SIP. The callerid that is to be presented is put in the P-Preferred-Identity header and also the From header with the Privacy header containing the presentation value. The Billing number is placed in the P-Asserted-Identity unless this is marked as to be withheld (via a parameter called the CBI) in which case that header is omitted. Its all a bit of a bodge and means systems designed to work in the UK often need different rules when talking to other systems. Also the vast majority of telcos in the UK dont support the standard anyway! Just included incase anyone is interested.
