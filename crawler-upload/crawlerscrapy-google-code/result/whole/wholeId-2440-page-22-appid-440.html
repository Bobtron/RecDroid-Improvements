csipsimple - issue #2382

Opus codec starts calls with a buzz/cracking sound

Posted on Jun 2, 2013 by Grumpy Camel

What steps will reproduce the problem? 1. Any call where opus codec is used starts off with an annoying buzzing sound before going away. 2. After that initital loud buzz/cracking sound, the call sounds fine as would be expected. 3. I've tried other SIP clients like Jitsi and Groundwire with opus and the problem only appears with CS. This seems to rule out a PBX implementation issue (latest FS build).

What is the expected output? What do you see instead? I expect there will not be any loud buzzing sounds when call is first established. What version of the product are you using? On what device / operating system?

Latest nightly builds with opus codeec enabled.

Please provide any additional information below.

I imagine some simple timing issue with CS's opus implementation is causing that annoying buzz/cracking sound but I'm no codec expert.

All I know is that this only seems to happen with CS, not with Jitsi or Groundwire when using opus.

Comment #2

Posted on Jun 25, 2013 by Grumpy Giraffe

I also had the same issue. I use FS version 1.2.10 which seems to be latest stable. When i upgraded CSipSimple opus codec to version 1.0.2 (latest stable from opus-codec.org) all the buzz disappeared, and opus codec now works perfect. Here is the patch for CSipSimple:

--- CSipSimple/jni/opus/Makefile 2013-06-13 11:50:02.450825224 +0300 +++ CSipSimple-trunk/CSipSimple/jni/opus/Makefile 2013-06-25 11:35:21.123799121 +0300 @@ -1,5 +1,5 @@ # This makefile aim is only for now to address source checkout and patch -opus_tag := v1.0.1 +opus_tag := v1.0.2

all : # For now we do nothing to build - it's made by android_toolchain
