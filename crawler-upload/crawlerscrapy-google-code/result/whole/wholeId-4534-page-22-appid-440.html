Comment #18

Posted on Aug 31, 2012 by Swift Monkey

Here is my observations. It looks like if for some reason the sipstack service is dead, or not functioning correctly, the main program is somehow unaware of the situation. The accounts will remain inactive or respond with "service unavailable". And "disconnect" option in the main program does no good. You have to kill and restart the program (sometimes multiple times) in order for the main program to restart or revive the sipstack service. Network interruption can be one of the reasons the sip stack goes dead. If the sip stack service is not self proof to recover from errors, the main program needs a more aggressive mechanism to detect and recover or revive it.
