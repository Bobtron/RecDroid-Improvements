csipsimple - issue #2476

question re: re-registration vs keep alive

Posted on Aug 29, 2013 by Quick Lion

Hello there,

With respect to battery life, I am using TCP, and I have set my keep alive value values to match my router and cell provider for wifi and 3G/4G, respectively.

Correct me if I'm wrong, but a registration requires a wake up just like a keep alive does, so it makes no sense to have the re-registration value any lower than the keep alive, correct?

Thus should the only consideration for re-registration time be how long my SIP provider keeps the registration for, and set it to just below that?

Thanks, Shiraz

Comment #1

Posted on Aug 30, 2013 by Quick Hippo

You are absolutely right. Keep alive should always be lower than re-registration interval (else it made keep alive useless as the re-registration can be considered as a keep alive).

So, you can set the re-registration interval as high as you wish. Normally an higher value than the one of the SIP server will result in server fixing the bound automatically (and client app will be informed of this limit and re-register when asked by server).

The only reason why you could want a lower re-registration interval is to avoid the following case : - you loss brutally your network connection consequence --> app is obviously unable to send un-registration as no network available - the network connection is not recovered consequence --> sip server keep the registration and thinks your as still reachable

consequence --> in case of incoming call on your account a caller might wait for tcp socket timeout of server before being redirect to voice mail. Reducing re-registration interval will limit the delay of the situation above.

For reference usual values for re-registration interval on desktop are around 15minutes (because network connection is supposed very reliable). Keep alive is more something like 30sec-4minutes (this one is more about configuration of the network routers between the sip client and the sip server).
