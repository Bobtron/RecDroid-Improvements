csipsimple - issue #1204

Show SIP code response (or reason pharse)

Posted on Aug 11, 2011 by Happy Cat

CSipSimple makes a call and receives 486 "Busy Here". But it shows nothing in the screen, just "call terminated".

Please, render the call reject status in some cool way, if not it seems that a 20 years old legacy analogic phone XD

For example:

  • 486 receives: show some icon meaning "Busy", or show "Busy" text or whatever.
  • Same for 480.
  • For unknown response codes, the original code number and the reason phrase could be displayed (and it's much better than showing nothing IMHO).

Comment #1

The main problem here is to find where/how to display the info. It could be on the red screen (but would be too fast I think). Or with an android toaster. But I don't know if that's very user friendly and what should be the time to display that on screen.

Comment #2

Posted on Aug 12, 2011 by Happy Cat

IMHO the red screen would be the best place (of course by making it to appear for time enough so the user can read it).

In a GSM call such "Busy"/"Not Available"/etc notification just takes a few seconds in the screen.

BTW a feature I miss in any GSM mobile or SIP (soft)phone is the indication of the rejection code in the call registry. This is: I would like to inspect the calls registry and see a "Busy" icon if the call was rejected with "486 Busy", or any other icon if the callee did not answer the call so I canceled it after N seconds.
