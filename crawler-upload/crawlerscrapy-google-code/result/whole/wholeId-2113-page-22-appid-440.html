csipsimple - issue #2736

XNet wizard - SIP server address has changed

Posted on Jul 1, 2014 by Happy Horse

What steps will reproduce the problem? 1. Use XNet wizard (in New Zealand) to setup a SIP connection 2. Connect to SIP server 3. CSipSimple uses an old IP address

What is the expected output? What do you see instead? The XNet wizard should use 182.154.16.15

What version of the product are you using? On what device / operating system? 1.02.00 r2330

CSipSimple has a wizard for a SIP provider called "XNet" in New Zealand. I have been using CSipSimple with this provider (on two Android phones) configured using the XNet wizard, and it works fine.

I received the following email from XNet (they're also known as WorldXChange, WXC Communications). It appears they have changed their server IP address and the CSipSimple wizard is using the old address, so it will stop working in a few days when they shut down the old server.

Are you able to update the CSipSimple build with the new IP address for XNet (182.154.16.15) ?

Regards, Russell Bell

-------- Original Message -------- Subject: WXC SIP Changes - pan.wxnz.net Date: Mon, 30 Jun 2014 22:28:36 +0000 From: Lucky Jaiswal <ljaiswal@wxc.co.nz> To: rb@fastbase.co.nz <rb@fastbase.co.nz>

sip:48311531@pan.wxnz.net

We are still currently seeing SIP traffic from your location on 58.28.20.150, this server is due to come offline shortly (approx end of the week). If you device is still using the old address at this time it will interupt inbound and outbound calling. It's extremely important that your device is looked at prior this SIP server coming offline.

There are few things which may be causing your device to use the old address, it could be ;

A Proxy, or SIP registar address still has the 58.28.20.150 address set somewhere. We've seen cases where some settings were updated but others needed to be corrected in the local config.

The other possibility is a DNS based issue, either the HOSTs file is set locally directing pan.wxnz.net to 58.28.20.150 instead of 182.154.16.150 or 58.28.20.150 may be cached and the device may need a restart.

The address your device should be using is ; pan.wxnz.net (182.154.16.15)

If you have a managed firewall you will want to ensure you allow traffic from 182.154.16.15 to reach your PABX. We would recommend contacting your dealer/IT support urgently to ensure this is updated ASAP.
