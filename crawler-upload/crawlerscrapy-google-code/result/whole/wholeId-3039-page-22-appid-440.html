csipsimple - issue #1751

Filters

Posted on May 22, 2012 by Happy Bear

Just a suggestion.

I've a filter that uses automatically my voip account which is: "Starts with 2". So, when I dial for example 245445678 it connects automatically by voip.

However, when I dial 200 (my voicemail) I want that it goes directly to a regular phone call. The problem is, even I create a filter for that (for call directly by regular phone call, via operator), it continues trying to connect via voip, which don't works!

Can you please rearrange the filters section?

My opinion is to haven't the filters divided by voip services and operator, but only a single page for all of them.

Then, when we create a new filter, we'd have four options:

1st - Select voip/operator (on this one, we choose the voip service/or operator to filter9 2nd - The filter (call directly, stop process, etc) 3rd - The rule (starts in, ends in, etc..) 4th - The digits.

After that, we can choose the priority, has is working now. The difference is that the priorities works for all voip/operator calls and not only for voip and only for operator.

I hope you understand me!

Best regards

Comment #1

Posted on May 23, 2012 by Quick Hippo

Normally, current approach will allow to deal with your expected behaviour.

I see two ways to do that, the most simple is to use the "Stop process" filter : Create one new filter in your sip account to "stop process" if number is exactly 200. By this way the rule "Force call" will not match. And the force call of your gsm account will be taken into account.

A second way to do that, is to refine the force call rule by using a regexp instead of a simple start with rule. The regexp would be something like "^2[0-9]{8,}$" (to match starting by 2 with 8 or more digits behind.

Actually your proposition is almost what is implemented but with a merged list which has the big disadvantage to make it very complicate to deal with the case where you activate/deactivate account. Also it may in the future make complicate to import/export only one account. And I think that it will make unclear rewriting rules, even if in case of force call/can't call rules, as you say it's probably a better presentation.
