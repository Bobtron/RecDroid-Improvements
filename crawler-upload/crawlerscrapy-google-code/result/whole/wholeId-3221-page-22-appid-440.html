Comment #1

Posted on Feb 11, 2012 by Quick Hippo

Hi Dani,

Yes I saw your logs, I was in hudge refactoring of the app and the nightly builds were not very stable ;).

Well I guess that you still have the problem with nightly and stable version right now which means that there is a real problem.

Regarding your logs, I can see that you try to make a call to +xxxxx number. The plus sign is very very very often not correctly understood by sip server. They usually require you to transform that into a 00 for example. I don't know how it goes with 12VoIP, but you should check if it's not something required on their side.

If so, you have to dial without the + inside the csipsimple dialer (because the app considers that you know what you are dialing within the app), but for numbers that comes from contact application (or anything else outside from csipsimple), you can use the 'rewriting/filtering' rule feature (available in settings) in order to automatically rewrite the numbers starting with + to 00 (or anything that 12voIP requires for international calls).

Comment #7

Posted on Feb 12, 2012 by Quick Hippo

Regarding logs, it sounds about packet size.

  • 12VoIP only support UDP
  • CSipSimple force the use of UDP
  • When doing the authorized invite with SRTP (that increase packet size), it makes a big UDP packet. You can probably get exactly the same by increasing activated codecs number.
  • CSipSimple try to send a big packet, but your routers on the middle consider it's a too big UDP packet and drop it.
  • Your sip server never receive the authorized invite, csipsimple timeout out because never receive any hack from server
  • The call fails.

To solve that, you can * Try to activate the "use compact mode" option (in network settings). Unfortunately not all sip servers support that. I don't know about 12VoIP. * Reduce the number of codecs announce by the application disabling all codecs but the one you want to actually use. Usually you pass through sip server so codecs are always the same. A working call should allow you to get infos about the codec in use. Then in settings / media / codecs, you can long press on a codec to disable it * switch to a provider that support TCP (tcp allows to fragment packets ;) -- so the packet size is not a problem)
