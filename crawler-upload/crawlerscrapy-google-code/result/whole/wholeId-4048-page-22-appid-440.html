Comment #3

Posted on Mar 15, 2011 by Quick Hippo

Sounds you have not activated TLS transport in settings.

Go in CSipSimple settings > Network settings > Secure transport.

Here activate TLS :)

That's it, it should register properly. Actually, it should be done automatically for you, but for now the design of the SIP stack and the first naive implementation led me to put settings like that : * A way to activate / deactivate a transport in global settings * A way to configure the transport to use in each account setting.

But when TLS builds will be merged to trunk and the true integration of security (tls, srtp, zrtp and so on), fully done, the user will be warned if one of his accounts can not be started cause the transport is not activated in network settings.

Comment #8

Posted on Mar 15, 2011 by Quick Hippo

Oh ok, there is something I should have said before.

In fact market versions and "trunk" versions does not ship the TLS support.

There is a very good reason : TLS builds must contain the entire openssl library. As consequence, I decided to not ship TLS support on the market build cause few users actually use this feature and it increase a lot the application size.

So only TLS builds available in TLS folder allow the TLS and ZRTP features that require openssl.

In the future I plan to make this as a module that can be get separately. (Maybe from the application itself or from another package available on the market/other distrib channels). It will be the case for extra modules such as codecs, tls, the gingerbread dialer and so on. For now features are distributed as different apk available on the nightly build server that build for different targets : http://nightlies.csipsimple.com/
