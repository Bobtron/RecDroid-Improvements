csipsimple - issue #539

Contact from CSipSimple app

Posted on Dec 25, 2010 by Quick Hippo

Some missing features in this implementation (written down here to not forget)

  • Add contacts skype uris to proposed numbers
  • Add contact emails to proposed numbers
  • Apply rewriting rules (this could be tricky to do if we want something nice cause here rules must apply dynamically since we have the account chooser button) (Note : no filters rules here cause could be confusing).

Maybe also we should consider GUI design : put contact and text access in the same place. (for example little button on the right of the digit dialer text field) -> And when press switch to text dialer enriched by contacts picker. Maybe even we could reuse the entire contact picker widget, so that if enriched by new features (pickup from call logs for example), it will benefit all places at the same time. BTW, if this is done, on the bottom we could have the voice mail just like in android dialer - as consequence we can add voice mail in accounts settings and allow automatic enrichment of voicemails thanks to the new voice feature and what is announced by sip voice mail NOTIFY messages.

Comment #4

Posted on Dec 26, 2010 by Happy Horse

Idea in Comment 1 integrated into branch dc3-539. If a contact has a custom IM address labeled 'sip', it can be used to place a SIP call from SipHome screen 'people' icon. It will show in the phone number chooser as sip:xxx@yyy.zzz. Address without the domain are supported and will use the domain of the account chosen. However, a domain should be supplied, and if it is, there are two benefits:

  1. A filter can be set on Dialer to prevent it from showing in the OutgoingCallChooser (Can't call : Custom RegExp : ^[^@]+@[^@]+$ -> contains '@'). For SIP numbers, then Dialer doesn't show.

  2. The form sip:xxx@yyy.zzz carries into PjSipService.sanitizeSipUri() and passes/matches the 'integrity' check, needing only to be reassembled from the matcher groups.

(1) leads me to a request for another 'simple' filter type, 'contains', and I will file the issue.

Final decision was to keep "sip:" (from custom protocol), "csip:" (from custom protocol), sip: from internet call of 2.3 and upper, and phone numbers.

If users needs wants to use email address as sip address it's advised to use a third party tool to automatically convert email address to "csip:" custom protocol. BTW, somehow this tool will be included in the buddy list feature when configuring the buddy from an existing android account.
