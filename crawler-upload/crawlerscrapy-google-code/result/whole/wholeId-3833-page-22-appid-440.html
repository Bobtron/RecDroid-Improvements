csipsimple - issue #997

No outgoing calls after recent update - PBXes

Posted on May 26, 2011 by Happy Bear

After recent update I have no outgoing calls on Nexus S

It used to work just fine but now I cannot even make a test call to PBXes echo server. After dialing a number I get nothing for 30 seconds and then the call ends. Checked everything I could and all seems ok.

Incoming calls work fine

CSipSimple ver. 0.02-01 r841 Nexus S running Android 2.3.4

Thank you Tom

Comment #1

Posted on May 29, 2011 by Quick Hippo

Maybe worth to try to reduce number of codecs allowed. Pbxes.org only supports correctly UDP transport (TCP is buggy on their side), so packets size are limited. In this cases "Use compact mode" option of network settings may help but I heard that it's not supported correctly by pbxes.org. If so the only thing you can try is to reduce the number of codecs activated (anyway pbxes.org does not support all of them).

Comment #2

Posted on May 29, 2011 by Happy Bear

Thank you for your help. As per your suggestions I've tried all the options but it wasn't any of those. It turned out to be the STUN server setting. Once STUN is disabled it works just fine. Regardless of the server used when I enable STUN the csip registers and is able to receive call but I'm unable to make calls. Since it works fine without it I ended up disabling it.

Thank you Tom
