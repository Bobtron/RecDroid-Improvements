csipsimple - issue #1484

r1177 sip stack process dies

Posted on Jan 9, 2012 by Helpful Cat

What steps will reproduce the problem?

  1. Install r1177 nightly build
  2. Send or receive a phone call

What version of the product are you using? On what operating system?

HTC Eris / CyanogenMod 6 (Android 2.2.1) / no camera hardware

Please provide any additional information below.

The calls are never completely set up because the sip process dies before the final ACK is transmitted. Attached are two logs. The "inbound" log shows what happens when receiving a call. The "outbound" log shows what happens when sending a call.

Posted on Jan 10, 2012 by Quick Hippo

MMmmhhh weird Is it a force close or a ANR or a direct crash? Force close popup something that tell you that something went wrong. ANR tells you that the app is too long to reply and give you the choice to wait or to kill the application And direct crash is usually due to a problem in native library.

The weird point in your logs is that I see : I/Process ( 137): Sending signal. PID: 3629 SIG: 9 So normally there is something that send this signal to csipsimple. Usually it's ANR that does that.

But if ANR problem, what happens if you choose wait instead of kill?

The video processing uses a lot of CPU process for now and probably it pledge your CPU on this phone. So in this case, it would be more about performance than real crash.
