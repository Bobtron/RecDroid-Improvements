csipsimple - issue #257

Can't pick up/deny calls by hardware button

Posted on Oct 1, 2010 by Happy Hippo

What steps will reproduce the problem? 1. Call yourself from another computer to the phone. 2. The phone rings. 3. Try to pick up the phone by pressing the green answer hardware button.

What is the expected output? What do you see instead? It goes to the phone application, instead of picking the phone, very frustrating.

Same for the red hardware button, will turn screen off, instead of denying the call.

Since it's a magnitude easier to use the hardware buttons instead of sliding or pressing the on-screen buttons, I think it's helpful to catch the hardware buttons if that's possible.

My phone is an Eris, running on Froyo, if that matters.

Comment #4

Posted on Oct 14, 2010 by Quick Hippo

Yes for red button your right : I added the code for what the platform call the END_CALL button however, on officials ROMs it's not an event that can be intercepted by an application. In fact, the end call and the home button could not be intercepted while in the application.

On some alternatives ROMs (such Cyanogen's one that run on my HTC Magic), applications are granted this right and so my code will works. But that's because Cyanogen has removed this restriction from the android system. For other official ROM, I didn't found any workaround (but that's reassuring since the system prevent application to use this button so, it's a good thing if application could not actually change its behavior :) ).
