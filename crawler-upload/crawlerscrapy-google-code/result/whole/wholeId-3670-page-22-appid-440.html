csipsimple - issue #1162

Annoying aliasing/ringing on incoming audio in 8kHz uLaw

Posted on Jul 20, 2011 by Helpful Lion
  • What steps will reproduce the problem?
    1. Disable all codecs but uLaw 8kHz.
    2. Register CSipSimple with Asterisk. Make sure the extension allows uLaw.
    3. Call anything. In my case, I am calling the internal IVR in FreePBX, for the best quality of sound.

Steps 2 and 3 can perhaps be done with other PBXs. Just make sure it plays a high quality non-compressed voice.

  • What is the expected output? What do you see instead?

Expected is clear audio, just like with a desk phone or Linphone on the same smartphone.

Received a high-frequency ringing over audio, typical of aliasing in a digital system. Are you doing SRC correctly?

What version of the product are you using? On what operating system?

CSipSimple 0.02-03.r944 Android 2.3.3 Kernel 2.6.35.10. HTC Sensation branded by T-Mobile.

  • Please provide any additional information below.

I am not sure if other 8 kHz codecs are affected. I think they are. Did not do any testing with 16 kHz.

Feel free to contact me for additional information. I am a VoIP software professional, but have no Android background.

Comment #2

Posted on Jul 20, 2011 by Quick Hippo

Can you try to record a conversation (use menu > record while in call) and attach it to the issue (or send by mail)?

I never had this problem. - For info, CSipSimple rely on pjsip sip stack which is a widely used and reliable sip stack. So this sounds is probably produced by csipsimple on the top of pjsip or by the way it open the android audio device (there is hacks available to try to support most devices). For example ringback is produced by csipsimple with simple tones - it maybe simply does not stop these tones in your case for some reason.

You can also try to play with the way csipsimple open the audio. By default, CSipSimple open your sound device @16kHz (cause on most devices it produce a better quality), and it use the new android audio api of android 2.3 that should improve things for 2.3 device (but HTC is known to have badly implemented this new api). Linphone for example open @8kHz by default and does not use the new api. You can try to get the same behavior in csipsimple by using audio hacks (see audio troubleshooting section of FAQ).

Comment #3

Posted on Jul 20, 2011 by Helpful Lion

Thank you very much for the clarification! The problem is probably in HTC audio implementation. Enabling "Mode API" checkbox fixed it completely. Just to be sure, I did not change any other settings: the FAQ suggested to begin with that option, I checked it, and magically the problem is fixed. Perhaps, this setting should be made the default for this phone model.

The problem repeated with all other SIP phones I tried, except Linphone, so CSipSimple is not in fault per se.

I am referring to "ringing" not as in ringing telephony tone, but rather a common name for a sound distortion. It is like there is a high-pitched noise in background, proportional in loudness to the signal, and changing spectrally with it.

To illustrate the problem, I fixed a microphone on phone's headset earpiece and recorded the same sample twice via that mike: first with the default setting (garbled), and next with the Mode API enabled (clean). I am attaching the files labeled so.

Please let me know if you want me to try other things on my phone in relation to this problem or any other, if you need to figure out some default hacks for it.
