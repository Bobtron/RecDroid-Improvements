csipsimple - issue #657

cannot hear call, call won't end unless force stop in the latest version.

Posted on Jan 31, 2011 by Swift Kangaroo

What steps will reproduce the problem? 1. Use CSIPSIMPLE on wifi 2. Make a call 3. Call will connect, but then can't hear anything, and it will hang-up, but the "call on-going" notification is still there. Need to force-stop.

What is the expected output? What do you see instead? In previous version, the call will come through and connect perfectly. In the new release, the above happened. I am using SIPGATE and every setting I used is the same as the old version.

What version of the product are you using? On what operating system? This is on My Touch 3G, running Android 2.2 (Cyanogenmod 6.1.1 stable, rooted and S-OFF).

Please provide any additional information below.

Again, the previous version works just fine, so that should rule out any Tmobile setting and SIPgate account reliability. In fact, I back up the old version and once I install the old version back, it works again.
