csipsimple - issue #1759

Exclude stock phone mean from outgoing chooser when placing a sip/csip call

Posted on May 24, 2012 by Quick Hippo

I'd like to express the following:

Sip account 1: direct call: if number is of type "Private" and begins with +49

rest: gsm

Why? Because by convention I set "private" for landline numbers and "mobile" for mobile numbers, and as it's cheaper in Germany to call landline by landline (~SIP) (if you don't have a mobile flatrate) I'd like to have that filter.

At the moment I have if number begins with +49xxx, where xxx is my local area code. Theoretically I would have to create a filter for every landline area code in Germany to have that automatic behaviour.

Does that make sense?

Comment #1

Posted on May 24, 2012 by Quick Hippo

It make sense, but unfortunately it's not possible.

Let me explain :

in fact when you make a call from the contact app in android, it's exactly the same than if you dial the number from the dialer, or dial from anywhere else in android. The only thing that is broadcasted to other apps is the phone number. The number "type" is lost. It's a property of the contact app only and is not linked to telephony layer.

So unfortunately this is not possible.

About the per code area problem, I think that you could fix that using RegExp rule. It's a very geeky way to express that but the power of regular expression is very very important.

To come back to your initial problem, there is also another solution you could use to get that working. (it works cleanly only with nightly builds). But it's instead to use a phone number of kind "private", to use the internet call field of the contact (if using android 2.3+) or an im field with custom protocol "csip" (or "sip") with the phone number of your contact.

My english is far to be perfect :). I want to say that if the third party app - in this case the contact app - launch some sip:xxxxxx request, CSipSimple will not present choice to use mobile but only sip accounts. BTW, in your case, it will normally only display one sip account, and so will automatically use it to place the call.

The point is that technically its not directly forcing sip accounts because there is existing call handlers that could manage call as well. So it's just about excluding stock telephony choice in the popup.
