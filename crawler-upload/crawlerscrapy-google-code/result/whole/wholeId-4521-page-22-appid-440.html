csipsimple - issue #265

IPV6 Support

Posted on Oct 8, 2010 by Swift Hippo

Does the current version of CSipSimple have an IPV6 support or not? I can't find IP configuration interface in it. Android 2.1 already supports IPV6 now, I wonder if the develop group has a plan for IPV6 support in lator versions?

Comment #1

Posted on Oct 8, 2010 by Quick Hippo

Well the sip stack on which csipsimple rely allow ipv6. I didn't know that android 2.1 has ipv6 support so I didn't build it with IPv6 support.

As the native stack support it (http://trac.pjsip.org/repos/wiki/IPv6), not so dev effort to make it available on CSipSimple. Only a precompilation variable to change I guess. I can try to turn it on in next version. The biggest deal is to see if it doesn't increase too much the native lib size.

For example for TLS support, I will have to release an alternate native library since it make the size of the lib twice bigger than the normal one. But my distribution mode quietly already allow this kind of diverts for the native library source. So CSipSimple is already designed to do that kind of extra support.

So if you have a way to test ipv6 support I'll try to build next version with ipv6 so that you can test it.

Comment #3

Posted on Oct 8, 2010 by Quick Hippo

Quietly everything is done by pjsip : getting IP addresses, sockets etc... That's the advantage of relying on a full featured sip stack :). The only thing CSipSimple do is using & configuring the UserAgent provided by the native library. Then everything is done on the native library. (That's the reason why we get good perfs results).

I've just done a build with ipv6 enabled, there is not a so big extra size, so could be included in main version.

I've already done the switch to create the good transport type (checkbox added in preferences - I guess that's useless to enable both ipv6 and ipv4 at the same time...). That's quietly easy to do... Now hardest part, I have to do the media transport creation. That's differ in ipv6 according to the pjsua_app sample. But it doesn't look so complicated.

Comment #12

Posted on Oct 13, 2010 by Quick Hippo

For reference (for other users that arrive on this page) : 13-07 allow to use IPv6, but you have to : * Disable STUN (or change the default one a ipv6 stun server) * Use IP in registrar and proxy uri (since IPv6 DNS resolution is not supported by pjsua (supported by parts of pjsip but not the all stack)

The corresponding ticket on pjsip : http://trac.pjsip.org/repos/ticket/419

If I get time, I'll dive into pjsip to have a closer look.
