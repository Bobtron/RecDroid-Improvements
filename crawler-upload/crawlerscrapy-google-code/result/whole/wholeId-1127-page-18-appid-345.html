geform - issue #66

Melhorar layout do aplicativo

Posted on Feb 8, 2014 by Quick Ox

A disposicao dos elementos e funcionalidades deve ser melhorada, por exemplo: - O botao de relatorio so pode existir depois de escolhido um formulario. - Apenas na coleta de dados os inputs devem funcionar. Ainda, se nao esta sendo realizada coleta botoes de envio e o campo para inserir o coletor nao precisam aparecer.

Comment #1

Posted on Feb 9, 2014 by Quick Ox

This issue was updated by revision a4aeef468f26.

Fiz algumas alteracoes de acordo com os pontos listados anteriormente. Agora o relatorio so pode ser acessado apos se definir um formulario. Com um switch é possivel acionar o modo de coleta de dados, onde é possivel enviar respostas para o servidor ou apenas visualizar os items do formulario (sem poder inserir valores). Coloquei um link para possibilitar baixar os dados coletados do formulario em formato tsv (que pode ser importado em um gerenciador de planilhas). Ainda é necessario melhorar o layout para o relatorio e tambem a forma como é mostrado, ja que visualizar-lo so permite visualizar o formulario com uma nova busca (o que nao deveria acontece). Outro ponto a adicionar é um forma de mostrar tarefas assincronas sendo realizadas, com um elemento de loading.

Comment #2

Posted on Feb 9, 2014 by Quick Ox

This issue was updated by revision 07cc6e482381.

Agora é possivel alterar entre a visualizacao dos items e o relatorio dos dados coletados. Depois de aberto um formulario, um icone pode ser clicado para trocar a visualizacao.
