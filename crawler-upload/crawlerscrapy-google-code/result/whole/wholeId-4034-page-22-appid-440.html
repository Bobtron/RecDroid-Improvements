csipsimple - issue #786

Use "Display name" instead of <username@...> in Chat GUI

Posted on Mar 10, 2011 by Helpful Monkey

When incoming MESSAGE arrives with the From header as below:

From: "John Smith" <sip:j.smith.323@sip.example.com>;tag=veoiz

CSipSimple will display "j.smith.323" in system tray notification and Chat GUI panel.

It should use "John Smith" instead.

Please see attached PDF for more details.

Comment #6

Posted on Mar 29, 2011 by Quick Hippo

Backport done in trunk. Thanks a lot for contribution.

As for chat vs message translation, I didn't backport it. In fact we have to keep in mind that this feature can be used by many ways. Some may use it as SMS others as chat. It will depends on the server on the other side and of billing of the service. As consequence I think it's better to keep message which is 1/2 SMS (Short Message Service) and 1/2 Instant Messaging (chat). Besides it's the name given in the protocol SIP Message. So I think that's not so much hurting to keep message as label.
