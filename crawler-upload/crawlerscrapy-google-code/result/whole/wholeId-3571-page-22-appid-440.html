csipsimple - issue #1247

Low audio level - HTC 2.3 - BT activated without BT device connected

Posted on Aug 31, 2011 by Helpful Dog

What steps will reproduce the problem? 1.Anytime I make or get a call 2.Must click the "bluetooth" button and click off of it to be able to control the volume of the ear piece 3.Either using application volume or phone volume, no difference is made until I click the "bluetooth" button on and off

What is the expected output? What do you see instead? The expected output of the earpiece is desired after having to click the "bluetooth" button on and off. I can then manually change the earpiece level while in a call after this is done.

What version of the product are you using? On what operating system? HTC Thunderbolt model ADR6400L android 2.2.1 baseband version 1.39.00.0627r, 0.01.69.0625r kernel version 2.6.32.21-g0606a3a htc-kernel@and18-2 #1 thur apr 14, build number 1.70.605.0 CL343953 software number 1.70.605.0

Please provide any additional information below. I cannot use speaker option until I click the "bluetooth" option on and then off. Once this operation is done all functions and features of the application are apparent.

Comment #4

Posted on Nov 13, 2011 by Quick Hippo

Ok very interesting point. In fact, the app tries to enable bluetooth routing if bluetooth is activated on the phone. (Unfortunately there's no way to know if there is actually a device connected)

On android devices that are correctly implemented this should not change a lot of things since the device should be aware enough about the fact a BT device is connected and to not change routing in this case. Sounds there's something wrong on the implementation made by HTC on the Thunderbolt. I will try to see if there is not something else I could test on to avoid trying to route to BT if it's unecessary.

Thx for reporting.

Normally BT is not tried as audio output by default anymore. In fact it was trying to use bluetooth if bluetooth was detected (and it detects when active even if not connected). And apparently some manufacturer in this case, even with no BT connected tried to route most things to use the BT even if nothing connected which was resulting a low audio.
