Comment #1

Posted on Sep 29, 2011 by Quick Hippo

Did you configured some rewriting rules in settings > rewriting rules > "Mobile" entry?

The application rewrite the phone number if filters/rewriting rules are added. Indeed this occurs only when you dial from stock dialer or when picking up a contact within the app. Both are considered as external input. So rewriting rules apply. In case you manually enter the number, this is considered as internal input and the app considers you know what you want to call and so does not rewrite.

So what you describe correspond to the use case where some rewriting rules are defined for "Mobile" provider. In this case would be just a problem with the configuration. Could also be linked to a third party application that also rewrite phone numbers.

On my side I do not reproduce this problem and that's the first time such an issue is raised, so it's probably about conf or external program. But anyway let me know your findings :)
