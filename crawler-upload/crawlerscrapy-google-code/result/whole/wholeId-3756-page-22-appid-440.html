Comment #1

Posted on Jun 13, 2011 by Quick Hippo

So several points you can try to play with :

  • There is one micro source option in csipsimple settings (this is hidden setting - follow this instructions to releval it in media settings : ExpertSettingMode wiki page). You can try to play with the micro source and the mode for sip calls. Samsung has some profile features that could lead to some weird behaviors. Maybe there is better settings than the default one. (I'll recieve to test a SGS2 soon and will be able to tell the best settings for this one then, but for now that's a little bit blind empiric results from users tests ;) ).

  • If this does not help, don't worry, CSipSimple has a micro amplification feature (it was already used in SGS 1 to reduce micro volume cause it was very very loud and produced a lot of echo). You can change it simply by pressing the menu key while in call and choose "media" option. Here you can adjust the micro amplification (up to *10 ;) ). (you can also adjust this values from media settings when expert setting mode is activated).

If you find better settings than the current default one, do not hesitate to share. Normally the approach is to automatically set up best values for a device when application startup (cause android audio does not behaves the same way on all device -- that's also why csipsimple is really tweakable on this point ;) ). So if you provide me better values for SGS II I'll integrate it to trunk to have it benefit all users :)

Comment #4

Posted on Jul 7, 2011 by Swift Monkey

The issue is: csipsimple is using the top microphone, not the button microphone that normally capture audio for calls. Solution: Go ExpertSettingMode, MediaSettings, Micro Source, choose default.

Comment #5

Posted on Aug 29, 2011 by Quick Elephant

I had mic low sensitivity problem with my SGS2. It turned out this cut off threshold was set by echo cancellation feature. After switching "Echo mode" to "Speex" or "Auto" or turning it off completely the mic sensitivity boosted dramatically.

"The issue is: csipsimple is using the top microphone, not the button microphone that normally capture audio for calls. Solution: Go ExpertSettingMode, MediaSettings, Micro Source, choose default." Can't confirm. Fore me the default Mic source ("Mic") matches the bottom mic and setting it to "Default" sets the upper mic active.

Comment #6

Posted on Jan 6, 2012 by Happy Elephant

Had the same "low mic input sensitivity" problem on a Samsung Galaxy phone and could solve it by selecting the option "Use Samsung Galaxy Hack" in the "Media" options. Sound is load an clear now - before it was nearly unusable.
