Comment #1

Posted on Feb 9, 2012 by Quick Hippo

Well, very first point, the market version does not contains TLS feature.

Previously it was build as a separate branch since it required to include openssl library. But I've found a clean way to reuse openssl embedded in each android device so that the binary size is not increased too much. As consequence, TLS is now part of the trunk version ! It will be included in next release on android market.

So, if you want to use TLS, you should install a nightly build. http://nightlies.csipsimple.com/trunk/

Once installed. * First step is to activate TLS transport in csipsimple globally (by default it's off). To do so, go in csipsimple settings, press menu key of the phone and select expert mode. The go in network > secure transport and activate TLS transport (normally nothing else is necessary). Maybe you also want to use SRTP, the default value is also here, but it can be configured per account as well.

  • The second step is to configure the account; I think that you are almost there with your expert wizard configuration :) The "sips:" is not strictly required in the proxy field, you can use "sip:" instead. But should be good as well. I've not yet finalized the UI stuff about differentiating schemes but the sip stack handle it correctly. So if the "sips:" is a problem it will not be for registration (only could be for my stuff on the display).

Also, you should uncheck the "Allow contact rewrite" option. (It corresponds to the "discover global IP" of your generic settings). Also ensure that there is not stun activated (it should not be the case by default but could get activated if you configured another account with a wizard that consider stun as mandatory).

Don't hesitate to feedback here if you get problems using the nightly build. It could be a little bit more instable than the market one, because it reflects current dev status... and there is a lot of stuff ongoing (video, sip presence...). But I try to keep it usable.
