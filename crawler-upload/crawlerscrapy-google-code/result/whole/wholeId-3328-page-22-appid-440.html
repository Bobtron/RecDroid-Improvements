I'm adding these notes to an existing Issue since it seems to be describing the same problem. If I've misinterpreted and this is a different issue, I apologize and perhaps then it should be a distinct issue?

However, it looks like the code is already written and should work, but an if statement is branching the wrong way.

Setup consists of two CSipSimple clients, both registered to the same OSTN-compliant SIP server (e.g. TLS only).

I tested it with both the Android Market version (R1841) and a nightly release (R1883).

Encryption settings were:

Network -> Secure transport -> SRTP Mode = "Mandatory" Network -> Secure transport -> ZRTP Mode = "No ZRTP"

or

Network -> Secure transport -> SRTP Mode = "Optional" Network -> Secure transport -> ZRTP Mode = "No ZRTP"

During the call, the status information revealed by the "i" button indicates:

SRTP status: Active Crypto-suite: AES_CM_128_HMAC_SHA1_80

but there are no overt User Interface signs that the call is secure.

This is in contrast to when the settings are:

Network -> Secure transport -> SRTP Mode = "Disabled" Network -> Secure transport -> ZRTP Mode = "Create ZRTP"

With this configuration, there is a clear yellow banner with a lock symbol.

The problem may be revealed in ./jni/csipsimple-wrapper/src/pjsua_jni_addons.c

call_secure_info() has an if statement that branches based on the state of "tp_info.spc_info[j].type".

I've enabled log level 5 for both test scenarios.

In the first scenario above (SRTP), I get:

In the second scenario above (ZRTP), I get:

It looks like the if state is branching based on "tp_info.spc_info[j].type" always being PJMEDIA_TRANSPORT_TYPE_ZRTP, not PJMEDIA_TRANSPORT_TYPE_SRTP.

Comment #5

Posted on Sep 9, 2012 by Quick Hippo

I think that I got it : if you use OSTN wizard, it will force the account setting ZRTP to "Create ZRTP". As SRTP probably remains "default" (so global setting), you are probably in the situation where there is both Create ZRTP and SRTP activated at the same time. In this situation strange things can happens.

Can you try to convert your ostn wizard to "Expert wizard" (long press the row, choose wizard). Then disable ZRTP in the account settings.
