csipsimple - issue #2411

csipsimple w/o video generates invalid SDP message

Posted on Jun 21, 2013 by Swift Bear

Running Version 1.0 on sony xperia tablet SPGT12 with Android 4.1.1.

Trying to establish a call from jitsi via SIP (Sipgate) to CSIPSimple (without Video configured). It sends the following SDP message:

SIP/2.0 200 OK Via: SIP/2.0/UDP 192.168.3.2:5060;branch=z9hG4bK-323330-a4dbfebd946736e4c495ce0ff8d14280 Record-Route: <sip:217.10.79.9;lr;ftag=8da6be98>,<sip:172.20.40.1;lr>,<sip:217.10.79.9;lr;ftag=8da6be98> Call-ID: 1fb7a6b6fb8b93c141abc703677d73af@0:0:0:0:0:0:0:0 From: "1709641" <sip:1709641@sipgate.de>;tag=8da6be98 To: <sip:1649539@sipgate.de>;tag=qrT.Q23oSLGB1BC6svZ1w.b4gGa-cZu- CSeq: 2 INVITE Allow: PRACK,INVITE,ACK,BYE,CANCEL,UPDATE,INFO,SUBSCRIBE,NOTIFY,REFER,MESSAGE,OPTIONS Contact: <sip:1649539@79.243.192.80:33585;ob> Supported: replaces,100rel,timer,norefersub Content-Type: application/sdp Content-Length: 338

v=0 o=- 3580747049 3580747051 IN IP4 79.243.192.80 s=pjmedia c=IN IP4 79.243.192.80 t=0 0 m=audio 4020 RTP/AVP 97 101 c=IN IP4 79.243.192.80 a=rtcp:4005 IN IP4 192.168.3.2 a=sendrecv a=rtpmap:101 telephone-event/8000 a=fmtp:101 0-15 a=rtpmap:97 SILK/24000 a=fmtp:97 useinbandfec=0 m=video 16 RTP/AVP c=IN IP4 79.243.192.80

Jitsi rejects the call with an error message "There was an error parsing the SDP description of ...".

The problem is that the media description "m=video 16 RTP/AVP" does not have a media format description. RFC 4566 says:

  If the &lt;proto&gt; sub-field is &quot;RTP/AVP&quot; or &quot;RTP/SAVP&quot; the &lt;fmt&gt;
  sub-fields contain RTP payload type numbers.  When a list of
  payload type numbers is given, this implies that all of these
  payload formats MAY be used in the session, but the first of these
  formats SHOULD be used as the default format for the session. (...)

After I discussion with the jitsi people it became clear, that the sentence "When a list of payload type numbers is given (...)" is not to be understood in the sense that the payload type numbers are optional. This becomes clearer when one looks at the BNF grammer also contained in RFC 4566:

media-field = %x6d "=" media SP port ["/" integer] SP proto 1*(SP fmt) CRLF

http://tools.ietf.org/html/rfc4566#page-40

So, lack of formats is not allowed.

If I install the video plugin and configure everything for video chats, a proper SDP message is sent. Video does still not work properly for me, but this a different story.

Suggested Fix: CSipSimple should not send a "m=video 16 RTP/AVP" media descriptor if video is not configured.
