csipsimple - issue #643

filters - dialer

Posted on Jan 29, 2011 by Grumpy Bear

I'm trying to implement 'calling card' functionality with filters.

First I tell my SIP profile never to call my calling card access number.

Then I go to settings - filters - dialer.

There I enter some rules: - if a number is 7 digits, prefix it with +access_num,1,8495 - if a number starts with +7, replace it with +access_num,1,8 - if a number starts with +, replace it with +access_num,1,810

Then I disable my SIP profile and start testing.

First, I enter a 7-digit number in system dialer and press dial. I get a dialog, which lists my 7-digit number entered and tells me to select a profile - but there's only one available, the dialer - so what's the point? Ok, I select dialer. Then another dialog appears. It lists the number after it was processed by a filter and again makes me choose from a list of one entry. I do that and get an error from my calling card service - their log shows that there was a second filter substitution done by csipsimple. Looks like a bug.

Comment #8

Posted on Jun 11, 2011 by Quick Hippo

Hi geejay,

The relevant parts of the code are probably there : http://code.google.com/p/csipsimple/source/browse/trunk/CSipSimple/src/com/csipsimple/service/OutgoingCall.java

and

http://code.google.com/p/csipsimple/source/browse/trunk/CSipSimple/src/com/csipsimple/ui/OutgoingCallChooser.java

I think that the problem is about the fact pause in numbers are dropped from the place I get the outgoing number in the intent. Problem is that the other place is probably uri encoded and it make things complicated. Besides rules for dialer option are managed differently and are probably buggy.

If you have a patch, do not hesitate to send it to me direclty (or in this thread). Or if you prefer I can grant you write access on the svn.

However, another problem is the fact it may mix with the dialer option and I think a better solution would be to have a csipsimple plugin that provide a way to add some entries in this chooser.

So to address this issue (and also all other issues about webcallback requests and access number requests), it would be better to make the outgoing chooser extensible in CSipSimple and provide a way to plugins to register add some entries in the chooser and to process different intents with what is done when clicked. It would be much more easier, extensible and user friendly way than adding this into csipsimple code.
