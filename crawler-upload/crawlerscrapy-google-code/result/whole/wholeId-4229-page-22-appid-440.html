csipsimple - issue #586

[Plugin idea] Time schedule for filters

Posted on Jan 13, 2011 by Happy Giraffe

What steps will reproduce the problem? Filters are active 24h, would it be possible to create filters which are time controlled as to say the filter allows me to call all local number e.g. +34 from 18-8 with the cell phone carrier and between 8-18 with csipsimple?

Comment #1

Posted on Jan 13, 2011 by Quick Hippo

Good idea,

Well very very low priority of course ;), but I'm wondering if it would not be better to have that as a third party application.

There is many cool feature that could be done by some "context aware" application. For example activating an account if you are on wifi a special wifi access point or things like that. Unfortuanetly for now AFAIK there is no opensource "context aware" application such as Locale or Tasker. If you know such an app I'd be pleased to develop a plugin for csipsimple that do that.

If not, I've in my mind to start such a context aware opensource application that accept plugins. And provide an open API for developpers so that developping a plugin will not benefit a application in particular and require my users to pay for another app. And also I'm not pleased with the api provided by locale and tasker. I think that something much more powerful can be done. But for now... I've not enough time to start this "context aware" app (would be a different app and absolutely not related to SIP feature in particular - even if one of the first plugin will probably be CSipSimple plugin if I do).. ;).
