Comment #3

Posted on Mar 17, 2011 by Quick Hippo

If so simply disable the automatic NAT detection that is done by pjsip. That's a cool feature of pjsip that allow to automatically detect what is your public IP based on your provider reply.

However if your provider does not like public IP (weird... but why not), you can configure CSipSimple to not have this feature :

Read this FAQ entry : http://code.google.com/p/csipsimple/wiki/FAQ?tm=6#I_receive_calls_twice_/_Registration_is_done_on_the_sip_server_t

And disable Allow contact rewrite. It will then disable the auto detection of your public IP.

Then if it works, I'll add a wizard for your weird provider so that other users of this provider will not have to bother with the expert configuration ;).

To do so I'll need the name of the provider, the sip domain, the countries where it is available (and if you have a 48x48px icon for this provider).

Thx in advance.
