Comment #1

Posted on Oct 1, 2010 by Quick Hippo

Hello, I appreciate the effort, however, read my comment on issue 148.

The best way to have contacts, is not to integrate it into CSipSimple. It should be done in a third party application that just support sip: protocol. Just have a look on android sdk about how intents works. CSipSimple just like sipdroid and probably linphone too, supports sip: protocol.

Doing an app that add sip: protocol support to contact is the best idea to have : the app will not be linked to CSipSimple in particular and work will all other sip application. Besides, CSipSimple already provide a way to use your existing contacts... just using the native contact application. So there is no real need to add something in CSipSimple to support contacts : You can use native android dialer, you can use contacts from the contact application, you can use shortcuts on home screen and so on... CSipSimple is already integrated to the whole android system. And that's the good way to do the things. Repeating yet another implementation of a phonebook in the sip application is a dead end approach IMHO : * Your approach need to be maintained * You'll need to bring all features from the native contact application (showing information from facebook, twitter and so on). * Make it compatible with the api from 1.5 to 2.2 and to next releases ! * It will increase a lot the size of the app. * You'll not be compatible with other sip applications * CSipSimple goal is not to be a contact app, but a SIP application, contact application can be delegate to another app! (and for now, delegating it to native contact app works very well !)

So it's not a good idea to introduce that into the app directly. It can be another project in itself. CSipSimple is by design open to this kind of third party projects that want to rely on CSipSimple SIP stuff and bring other features. So you should consider this solution instead of directly integrate yourself in the CSipSimple code : not only it has advantages it also avoid a lot of problem in CSipSimple.
