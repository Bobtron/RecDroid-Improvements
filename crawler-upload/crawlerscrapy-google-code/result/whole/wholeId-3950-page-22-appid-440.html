Comment #1

Posted on Apr 2, 2011 by Quick Hippo

Unfortunately, using 3g the problem is that there is no real way to ensure the data stream is kept alive by the core network of your carrier. Usually to do so, it's necessary to reduce keep alive interval (for 3G) to low values. Re-registration timeout can be left high however.

The time routers from your carrier close the socket may depend a lot of the carrier.

Anyway, I'd be interested by logs from your device. Can you send me logs (see HowToCollectLogs wiki page).

Also, if the asterisk server is yours, you should probably try to use it using TCP. TCP sockets are not closed like UDP sockets are by routers. So could be interesting to try to switch to TCP (usually it allow to increase a lot keep alive and re-registration timeout).
