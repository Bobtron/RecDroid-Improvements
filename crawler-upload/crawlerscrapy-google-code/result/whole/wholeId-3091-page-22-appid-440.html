csipsimple - issue #1715

GoGear_Connect : Missed Call Notification leads to contacts, but no call log

Posted on May 2, 2012 by Happy Rhino

Issue 1546 and Issue 1517 are close, but not exactly my issue.

Some inbound call that came through CSipSimple did not get picked up, and something triggered a Missed call notification.

Fine, that part is OK. However there seems to be no way to get rid of the notification.

I have visited the CSipSimple call logs, and cleared them. The notification persists. When I press it in the notifications list, I am taken to two tabs, Contacts and Favorites, with the Favorites tab selected. I think that if this was a "real phone" there would be a third tab somewhere around here with the Dialer's missed calls log.

So, where does this go? I finally figured out that I can get rid of it by finding the Dialer app in Manage Applications and clearing the cached storage. I would like for there to be a better way!

Should I file a defect with my tablet manufacturer? They will surely respond "Sorry, not a phone, it's an MP3 player. CSipWho?" Could this be handled any better by CSipSimple?

Posted on May 3, 2012 by Quick Hippo

Mmm, interesting problem. Well, normally the app should only raise a missed call notification that it can manage itself. The other one -- integrated to android call log stock system, is normally mark as read when inserted and so should not produce a stock notification. For some reason, on your device, the stock contact app seems to raise this notification. I guess they didn't really care about that and did some fast hack in stock contact app which produce this result.

So probably a bug on their side about the management of the stock call log list. But good news... :) there is a way to disable this stock call log integration in csipsimple. With the "Stock call log integration" option disabled, CSipSimple will not anymore write your call logs in the stock application (in your case anyway it's useless since you have no ways to access it since manufacturer disabled that). And normally the notification that leads to nowhere will not be shown anymore by system.

So, do disable "call log integration", first go in csipsimple settings, press menu key. And select "Expert setting mode". Then, enter the "User interface/integration" section (I'm not sure of the english label, but the icon is a little droid). Here you'll find the option to disable "call log integration".

Let me know how it goes with this disabled. Also I'd be interested - for the record - on the device / model you own. It could allow me to automatically disable this option when I detect the application is running on this device. To get the exact device/model, you can follow instructions of HowToCollectLogs wiki page (it sends me a mail with information on the model).

Also, as it's a tablet you may be interested in the nightly builds. If you have time to test it and wants to participate beta phase : http://nightlies.csipsimple.com/trunk/
