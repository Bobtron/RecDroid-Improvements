Comment #1

Posted on Jan 4, 2012 by Quick Hippo

What do you mean by "notification screen"?

Is that the notification bar? If so it's not really a good idea to do that because notification bar info is raised as soon as the call arrives and introducing a contact resolution is not good to be done here (it could be updated later, but I'm not sure that's really useful).

If it's the actual call screen. It should already be the case. If the sip uri has display name or username that looks like a phone number ("display name" ), then it will try to find the contact in your android contacts. It will also try to display the photo if exists. However, the resolution is made on Exact phone number. So if your contact does not contains the +1 for example... it will obviously not match and not be solved by csipsimple.

To be sure the phone number is properly resolved, you can simply switch to text dialer and start searching +1212 , if the autocomplete list doesn't show your contact, it means that your contact has not the correct full number in its infos.
