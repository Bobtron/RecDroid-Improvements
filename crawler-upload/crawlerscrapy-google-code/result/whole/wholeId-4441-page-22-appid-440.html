Comment #4

Posted on Nov 23, 2010 by Quick Hippo

The rules are implemented to allow almost everything your imagination can produce ;).

The important thing to remember is that rules applies only to the stock android dialer (not the csipsimple one... we assume that on csipsimple you are dialing exactly what you want to send to the server).

Some quick example of what could be done with filtering/rewriting rules : * rewrite entirely for example the voicemail number you can say that for example 123 is directly changed in **1 (if 123 is the number of your GSM voicemail and 123 the one of this sip provider). * Ignore numbers that starts with 06 (in france that's mobile phone prefix), so if no sip account manage phones starting by 06 (all are configured with filters to ignore phones starting by 06) => 06xxxxx calls will direclty use GSM (and don't ask you for choice). * Force numbers that starts with 04 (in france landline phones), so you'll not have the popup and it will directly call using this SIP account. * Rewrite "+" sign into "00" many sip providers doesn't support "+" sign and prefer the standard prefix "00".

Of course, that's just examples and you can configure as you wish ;)

Comment #8

Posted on Nov 23, 2010 by Quick Hippo

There is no way to disable the account chooser button !

I'll try to explain the two points :

  • As for integration with android : -> This enable the native dialer integration. The rewriting and filtering rules applies only when you dial from the native dialer. And not from the CSipSimple dialer that already have an "account chooser button". Trying to explain in other words (my english is far to be perfect so maybe something with my previous explaination :) ) : If you are in CSipSimple, as you have the account chooser button, I assume that you know exactly what you are doing and that you want to make a SIP call using the selecteed account. If you are in the android native dialer (and android integration is activated in settings). You just want to make a call to somebody... Here filtering and rewriting rules applies ! That's logic, you never tell the system which SIP account it should choose ! You are just trying to make a call, and here the SIP application must be intelligent (and use rewriting and filtering rules) to guess what you really want to do! The system will show you (or not if configured to force call or ignore call) a popup with account lines to choose.

To sum up : if you use CSipSimple dialer, you have the intelligence and dial properly and choose the outgoing account using the account button. If you use the native dialer (or click on make call on the contact), CSipSimple try to be intelligent and guess what you want to dial out.

  • For incoming / outgoing mode : I guess that when you say I restart the app, you clic the "Disconnect button" in menu. If you are configured in "available for incoming" mode; there is a popup .... read it... it explains you that by disconnecting the app you turn off the incoming mode !!!! Again that's logic ! If I still left the incoming mode here, what will happen... the app will try to restart it itself as soon as you network status changed. And besides how could you be in incoming mode if the app is actually stopped? So I had two solutions :
    • Hide the disconnect button.... well, a lot of users that doesn't understand that incoming mode and disconnect the app is not compatible complains cause they think there is no way to stop the app... (indeed, they configured it to receive calls...)
    • The actual method where I show the button, but show a popup if you are configured in available mode saying that I will remove the available for incoming mode.... Well if here user doesn't read the popup... I don't know now what I can do to do something more understandable by users !

Hope that make things more clear for you. If you have any suggestion to make things more clear for users, do not hesitate I'm open to suggestions.
