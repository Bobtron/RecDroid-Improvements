android-daisy-epub-reader - issue #57

Calculations of timings need to be reliable

Posted on Mar 26, 2012 by Helpful Panda

We currently use floating point values to represent the timings read from SMIL files. Comparing floating point values can be problematic as some values are impossible to represent accurately e.g. 0.1 seconds. This means our current code might not be able to compare accurate values, which may lead to complications in the code (where we try to code around inaccuracies) and bugs where despite our best efforts the calculations remain inaccurate, e.g. where the audio player may continue playing rather than finish at the correct offset.

To improve the accuracy and simplify the code we can consider using BigDecimal representations http://docs.oracle.com/javase/1.5.0/docs/api/java/math/BigDecimal.html

I've not used these before so I need to experiment first.
