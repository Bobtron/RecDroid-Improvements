Comment #3

Posted on Sep 17, 2010 by Quick Hippo

Well, I've just created an account on sipgate.

To configure properly csipsimple (Use a dev version of csipsimple - see wiki HowToInstallDevVersion to find out how to install it - it's by far better than the market's one).

Go on sipgate website. Login and click on settings (top right). On the overview tab there is all needed information. Now open csipsimple and create a Basic account. Display name : Anything you want (for example sipgate) User name : fill with the value of SIP-ID displayed on sipgate setting website Password : fill withe the value of SIP password displayed on sipgate setting website Server : fill what is displayed in both register and proxy (sipgate.co.uk for example for me). Save. It should register right now.

To complete the setup, you should go back in the main screen of csipsimple (the dialer), press menu, settings. Go in Media Settings and scroll down to STUN. Tick the checkbox and click on STUN server bellow. Fill with the value of STUN available on sipgate website (for example stun.sipgate.net:10000) . (STUN is useful to ensure media path is ok - and can be useful for registration step too).

It works properly for me. I think that you confuse your login/password on the website and your sip login and sip password. (In sipgate they are different). I'll add a wizard dedicated to SipGate in order to clarify things.

Comment #9

Posted on Jun 11, 2011 by Quick Hippo

Are you using the SIP-ID and the SIP-password? Be really really careful, that's DIFFERENT from the login on the webpage of sipgate !

To get it go on sipgate web page, in your settings (top right of the webpage), you'll find a "SIP account data" with SIP-ID, SIP-Password ans Server. Use these settings to register.

(for my account for example it tells server : sipgate.co.uk - I'm in france). But I know that they change the server depending of your country.

I'm currently trying to work with sipgate api to allow users to login using their web credentials (credentials that you use to login on the webpage). It will be less confusing for mainstream users that usually don't understand why sip credentials and webpage credentials are different (BTW, this is particular to sipgate provider).
