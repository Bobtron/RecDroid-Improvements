csipsimple - issue #2384

DNS-SRV TCP transport - Possible bug on outbound call DNS lookup

Posted on Jun 3, 2013 by Swift Dog

What steps will reproduce the problem? 1.Register to a DNS-SRV address that only gives TCP service responses(server does allow UDP transport as well) 2.Make an outbound call 3.Call will fail on DNS lookup, prior to contacting the server to initiate call.

What is the expected output? What do you see instead?

Expected: Successful call via tcp transport on the primary A-record lookup

Actual: 502 - DNS "Name Error" (PJLIB_UTIL_EDNS_NXDOMAIN)

What version of the product are you using? On what device / operating system?

Android 4.0.4 -- Kyocera Rise Android 2.3.7 -- Motorola Triumph

csipsimple version 1.00.00, with codec pack(not in use)

Please provide any additional information below.

It appears that the client will attempt to run a UDP DNS-SRV lookup prior to making the call, despite being set to TCP transport. Our TCP DNS-SRV record does not supply any UDP connection information, and the call fails. Inbound calls work fine with TCP, as well as UDP DNS-SRV records.
