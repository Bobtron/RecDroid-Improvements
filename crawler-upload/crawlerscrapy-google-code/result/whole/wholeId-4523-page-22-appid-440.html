csipsimple - issue #267

Action intent to activate/de-activate an account?

Posted on Oct 8, 2010 by Massive Hippo

What steps will reproduce the problem? 1. When I'm at home, my cell-phone registers and rings for incoming VOIP calls instead of the phone(s) connected to my Linksys VOIP adapter (which is hooked up to my router).

What is the expected output? What do you see instead?

I realize that this is really an issue with my VOIP provider not supporting multiple end-points, but I would like to configure my phone so I can activate/de-activate my VOIP account in CSipSimple using an action intent. I have a context-aware application called Tasker which allows for settings to changed or tasks to be run based on different contexts. I have different contexts set up for when I am at home, at school, etc. Tasker allows for action intents to be triggered based on context. Is it currently possible to activate/de-activate a CSipSimple account using an action intent? If not, would it be difficult to add?

Comment #2

Posted on Oct 8, 2010 by Quick Hippo

Indeed, such API is planned. For now there is already a complete API but not accessible through intents. It use the binding mechanism (which much more rich, but also more complex). For what you want to do ! First of all, I can advise you to consider using "Only For outgoing" mode of csipsimple. Can improve things a little bit. To do something really user friendly that respond to your use case, we should write another program that make the glue between csipsimple and another program that send only intents. For example in the other program if you want to choose for which account you want to activate / deactivate things it would be more user friendly if you give a list instead of letting the user guess which is the key for this account.

However, in the absolute that's possible to add an intent to control this stuff... The only thing is that it will not be user friendly and that there is already an API in csipsimple that allow this feature. So I'm wondering if it's really worth to add a intent, and if it would not be better to create a plugin for tasker / or a generic plugin that receive intents and use the csipsimple API.

Is there an opensource alternative to "Tasker"/"Locale"/"Setting profiles"?

I'm not really pleased with making free work for a close source app... I consider that if one of these apps want a csipsimple plugin, they should do it their self. Moreover, choosing one will incite CSipSimple users to choose this app instead of another. I'm pleased with incite users using opensource/GNU free software, but not really with choosing a closed source app and support it.

What would be more ethic for me would be either : * To do one for each closed source alternative if there is no opensource alternative. * Or if all these apps has developed a common API.... but I guess that's not the case since their benefits are to put users in jails with plugins....

However, I'm not against the development of such plugin on each of those contextual settings apps, but I'll don't do it myself. If needed I'll provide help to enrich CSipSimple API but as for me I can only consider making this work to promote another project under a nice license ;)

Yes I know I'm a little bit fundamentalist but that's in accordance with my own point of view of what should be the freedom provided to software users. As freedom is matter of choice, I want CSipSimple users to be free to choose the "contextual setting" app they want !

Comment #7

Posted on Nov 13, 2010 by Quick Hippo

An intent now exists :

com.csipsimple.service.ACCOUNT_ACTIVE_CHANGED

An extra "activate" boolean whether you want to activate or deactivate the account. An extra "acc_id" Integer : the account id (account in database are incremented by creation date).

You can sniff what is sent by account widgets if needed.
