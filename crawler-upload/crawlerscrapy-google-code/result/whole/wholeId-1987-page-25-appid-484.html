bthfpowersave - issue #6

Usability: Handling states before / after phone calls

Posted on Nov 13, 2011 by Grumpy Kangaroo

As of 1.0.20 the user has the option to say that bthfps should turn bluetooth off after the call has been ended. I don't think this option is really satisfying for most users.

My suggestion would be to return bluetooth to the state it was before the call came in. Actually I see several possible scenarios:

Bluetooth was previously turned off:

1.) The user does not have access to his handsfree device when he receives the phone call. He does not need bluetooth at all and thus wants bthfps to turn bluetooth off (or even not turn it on at all, but how should bthfps know that...).

2.) The user has access to his handsfree device when he receives the phone call. bthfps activates bluetooth and the user answers the call using his handsfree device, but after the call ended he does not need bluetooth anymore thus wants bthfps to turn bluetooth off again.

Bluetooth was previously turned on:

3.) The user turned on bluetooth himself because he started a journey where the handsfree device was available all the time. He does not want bthfps to turn bluetooth off just because the first phone call he received was ended.

Of course, there could also be more "dynamic" sitations where the user started the phone call when being at home and continued the call in his car, thus he wants the handsfree to take the call over. So in the end I would opt for the following options:

  • return Bluetooth to the previous state (perfect for situation 2 and 3)
  • (additionally) return to the previous state before the end of the phone call, if no pairing did take place for <configurable amount of seconds> (perfect for situation 1)
  • manually specify the behavior on an ended call(the currently implemented solution)

I don't think bthfps should handle those more dynamic scenarios because most phones allow to dynamically switch bluetooth on/off easily during an ongoing call. bthfps's main task should be to inititially turn bluetooth on in case the user is i.e. driving in his car and has the phone in his pocket.
