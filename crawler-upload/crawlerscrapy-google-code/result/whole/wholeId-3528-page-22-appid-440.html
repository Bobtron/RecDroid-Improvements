Posted on Sep 27, 2011 by Quick Hippo

It's better to ask this kind of question on the developper group. (Many questions about that has already been replied). (http://groups.google.com/group/csipsimple-dev/)

The java trace is the obvious consequence of the fact you have not the .so files built.

If you don't want to modify the .so file, I really advise you to extract the .so files from a nightly build. All the more so as you seems to want to use it under windows, which I do not advise (windows is such a bad OS for developpers ;) ).

However, as per indication of the android NDK, you should define a NDK_PROJECT_PATH to CSipSimple app if you want to use ndk-build. It is possible to use ndk-build, but one that want to use it, have to know what he is doing ;). If you don't know how to define a variable... you should switch to Linux ;) You'll learn so many things and improve your devs skills.

Anway, for now, the best solution remains to use the standard make toolchain. What is your problem? It is maybe some of those already replied on the google group?...
