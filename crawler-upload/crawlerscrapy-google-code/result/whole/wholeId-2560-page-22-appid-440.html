csipsimple - issue #2244

Enhancement: Auto answer contact(s)

Posted on Feb 13, 2013 by Helpful Rabbit

This is sort of an addition to http://code.google.com/p/csipsimple/issues/detail?id=1726 .

To be honest, what I really wished for with the last request was a way to block a certain contact, and this issue was "fixed" by adding the ability to autoanswer a call with a custom SIP code.

The current fix does however not really reach the desired functionality as it does not seem to be able to match anything else than the number itself. (And it's simply not reasonable to add every single known number to an individual filter; I've got +50 numbers and counting..)

So, please add the ability to match the contact name (& etc?) for the auto answering filters.

Comment #1

Posted on Feb 13, 2013 by Quick Hippo

Mmm, well, first point is that it can be done with a single very complex regexp rule to match all your individual contacts (not needed to have as many rules as contacts).

But, anyway, I think the best solution to address that with a plugin to csipsimple that just generates/maintain this rule with your contacts. CSipSimple allows third party apps (with right permissions) to extends features. And this kind of feature could be done outside. It will definitely allow more creativity if done outside. For example, setting rules depending on your current availability im status or on your calendar meetings etc. I think a dedicated app could do much more than the app for mainstream and there is room for more than just one app depending on users needs. So I rather encourage these kind of apps than making more complex the core app and addressing only parts of the needs. Auto-answering with a sip code was a feature of the core app (as was not possible to do outside), that the reason why it was added in the core engine.

But writing a rule based on contact app or writing a rule base on your favorites contacts and if you are in a meeting or not, or should be done outside (as a plugin) to encourage creativity on these kind of features. The basic stuff in csipsimple API is ready to allow that. There is just the need of some developer to take the point and develop such an app using the open programming interfaces of csipsimple.
