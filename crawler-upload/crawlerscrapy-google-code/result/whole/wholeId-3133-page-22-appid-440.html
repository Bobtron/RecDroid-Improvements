csipsimple - issue #1660

Bug (not seeing invite after proxy auth on server, but there in csipsimple log)

Posted on Apr 4, 2012 by Massive Monkey

I see that the server doesn't see an INVITE after Proxy Auth required message. It sees the ACK after it, but not the INVITE. The csipsimple logs here show that it does seem to send the INVITE also but I don't see it in the server.

Other clients work perfectly where I do see the INVITE (above) fine.

I think Csipsimple should resend the INVITE after a small time out.

Another issue is that in the GUI, when I hit the end button it seems to not respond. I think the user hitting an end button should have at least some impact on Csipsimple even though that screen is showing the state of the call or something. But it just doesn't respond. A long later based on the sip state I think it closes the window it appears.

Note that sometimes the calls go through.

Android version is Froyo

Comment #1

Posted on Apr 4, 2012 by Quick Hippo

It's typical problem due to the fact INVITE packet is too big for UDP, and your routers on the middle may just drop this packet because it's a too big UDP packet.

This is not a bug in csipsimple, but about something that can't be done (and can't be done with any sip client configured the same way csipsimple is).

You should : * 1) check if your sip server supports TCP, if so just switch to tcp or transport mode auto (auto mode will use TCP if packets seems too big). * 2) if not, you have to try to reduce packet size by : - using compact mode if supported by sip server - reducing list of available codecs.

Other sip client may succeed because they don't have as many codecs enabled or because compact mode is activated by default (which I don't do because many sip servers doesn't supports that correctly).
