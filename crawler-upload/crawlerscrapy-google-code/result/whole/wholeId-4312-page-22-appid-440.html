Ok, Nexus S is probably affected by the PSP problem (just like HTC devices and Dell streak)

There is a simple workaround for that : Activate ExpertSettingMode for global settings (see wiki) Then in Settings > User interface > Activate Keep Awake while in call.

Everything will go fine then.

To allow me to activate that by default could you send me logs with my builtin tool (see HowToCollectLogs wiki). It will provide me the device identifier so that I can autodectect this phone and add it to the list of device which need the PSP workaround.

Another things I'd be extremely interested in : in 2.3.1, there is a stock SIP application. Is this application also affected by the same problem? Cause if not they maybe do something special (probably a private API) but as source of this stack is open I can have a close look and see how google solve the problem. (which is a problem all sip application has for a long time - it goes the same on nexus one and many htc devices )

Comment #4

Posted on Dec 26, 2010 by Quick Hippo

For testing version : http://nightlies.csipsimple.com/trunk/CSipSimple-r507-trunk.apk

If you can say me how it goes if you disable "keep awake while in call" setting and when proximity sensor activates. Would be amazing if working :) (and I'll cry to google why they do not open this part of the API ;) ).

The stock SIP application is in fact compound of 2 things : there is a "service" that any application can call and use to make SIP calls : it's the SIP stack. And there is the stock application that allow you to make stock sip calls (that have rights to use the SIP stack).

On android apps can work together (that's a really cool and extensible approach). And most of the times stock application are well done to allow other apps to use services apps. For example for standard mobile, the "telephony service" is distinct from the "dialer/contact app" (which allow manufacturer to code their own contact/dialer app without changing the stock telephony service.

It goes the same way for SIP, the stock stack can be used by other apps. And I've planned in CSipSimple to allow users to choose between the service I already use (based on pjsip stack that I've ported to android) and the stock SIP stack (if available on the phone ;) ).
