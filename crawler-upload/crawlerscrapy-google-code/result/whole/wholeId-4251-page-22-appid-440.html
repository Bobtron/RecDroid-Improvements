Comment #7

Posted on Dec 26, 2010 by Quick Hippo

For iLBC => Normal your device doesn't support it (in theory was deactivated by default). Actually your device has armv6 or lower CPU which doesn't support floating point CPU primitive. As iLBC implementation is a floating point one your CPU keep converting floating/fixed point which is pledge CPU. So do not activate it for your phone.

For craks maybe worth to try to deactivate Echo cancellation. And also try the latest trunk : http://nightlies.csipsimple.com/trunk/ (I've changed buffer size maybe it can help on cheap devices).

For TCP/UDP, you really should switch to UDP (in logs I see TCP, and again pbxes.org doesn't support it properly). It will not improve media but for SIP signaling may help a lot. Pbxes.org develop sipdroid and their implementation of TCP is not standard (not follow SIP RFC). As consequence we have to fallback to UDP (and force it) with this provider. In logs you sent to me was still UDP.

I'll add zte to list of devices that need Audio mode API. Thx for logs.
