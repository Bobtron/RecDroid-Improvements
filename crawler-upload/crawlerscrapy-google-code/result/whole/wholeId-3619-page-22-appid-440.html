Comment #1

Posted on Jul 26, 2011 by Quick Hippo

This kind of issue is linked to the way the manufacturer implements the android audio api. So it's really really really important to say the android device you own.

I really guess that you own a Samsung with a ROM they probably didn't update to fix this problem. Samsung did this kind of thing on a couple of devices. Usually using the "samsung galaxy hack" helped (but that's a awful hack), but on some other devices, even this hack is useless, and there is absolutely no ways (at least as far as I can tell for now) to workaround things on these device).

About the hacks you should read my comment here : http://code.google.com/p/csipsimple/issues/detail?id=119#c38 (issue 119 comment 38). I deeply explain what are hacks and what they do. For example it explains why using "audio mode api for sip calls" and using IN_CALL mode is the same (while you do not activate speaker mode)
