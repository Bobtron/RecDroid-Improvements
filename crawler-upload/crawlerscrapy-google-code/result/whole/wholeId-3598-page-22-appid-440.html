Comment #1

Posted on Aug 21, 2011 by Quick Hippo

I guess you are using sip over 3G?

What codec are you using? If you use PCMU (aka G711) you have to be aware of the fact his codec use 64kbit/s which is not very suitable for 3G (and any mobile data network). As consequence a little restriction in bandwidth of the antenna you are connected to (for example another people that place a call on the same mobile antenna than you), and the bandwidth will be limited for your connection.

As consequence if using a codec such as G711 that does not compress audio, will obviously result overload of network.

To check what codec is selected, tap the little arrow under the remote party card and select 'Info' button. It will popup infos about the call and you'll see the codec in use.

CSipSimple support a lot of codecs, so many are suitable for mobile calls (iLBC -- that have been improved in nightly builds, GSM codec, speex 8kHz, g729 ). However, to use a codec the remote party or the sip service you are using must also support this codec. So maybe csipsimple select pcmu/a because your sip provider does not support codecs optimized for mobile calls. If so I'd advise to ask them for support of iLBC.
