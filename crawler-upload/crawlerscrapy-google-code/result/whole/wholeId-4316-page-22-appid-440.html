Comment #2

Posted on Dec 25, 2010 by Quick Hippo

My fist thought reading title of issue is to ask you to try to activate DNS_SRV in settings. (That's not enabled by default cause impact perfs but some sip provider need that activated). If helps I can add a wizard for vono.net.br not automatically activate this setting.

Then reading your first logs : 404 is not a problem with DNS. DNS is properly solved (201.86.87.5 seems to be ok). Most of the time 404 mean that the user is not found on the server. So probably something with configuration. How do you configure the account and what are settings advised by your sip provider (any tutorial somewhere I could read and say you how to configure csipsimple to have good settings?).

Reading your second post make me think about another thing. Some sip server doesn't fully support SIP standard RFC and we have to change settings of csipsimple to workaround with these servers. If it's the case you should try to follow in FAQ the "double registration" issue workaround.

Comment #3

Posted on Dec 25, 2010 by Massive Bird

Setting the DNS_SRV doesn't solve the problem. It looks like using an ip address for the proxy just makes csipsimple ignore the error.

What worked in the end is to disable renaming as you suggested and keep the proxy config empty.

The sip provider has instructions on how to setup some atas: http://www.falevono.com.br/portal/ajuda/instalar_atas.html

They are in Portuguese, but the screen shots should make it easy. For example, on the "bps 5100", they suggest

"Display Name": respindola "User Name": respindola "Register Name": respindola "Register Password": password "Domain Server": vono.net.br "Proxy Server": vono.net.br "Outbound Proxy": vono.net.br:1571
