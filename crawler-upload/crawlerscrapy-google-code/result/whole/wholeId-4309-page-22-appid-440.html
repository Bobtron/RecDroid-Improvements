csipsimple - issue #531

Lessons to learn from Gingerbread native SIP client

Posted on Dec 23, 2010 by Happy Monkey

A few things that I noticed running the native SIP client:

  • there is no "vibration mode" indicator while talking, though I did not check how it deals with incoming mobile call. Perhaps it deafens the user like csipdimple did before this problem had been taken care of.

  • While I was talking, the screen blanked out after timeout, and it did not have any detrimental effect on the sound quality! Maybe some new classes of lock are introduced in 2.3? I think it's worth looking into.

  • Battery life seems much worse with this ROM (Oxygen 2.0 RC2), though I do not yet know if this is related to the SIP agent or not.

Comment #1

Posted on Dec 23, 2010 by Quick Hippo

And I know that cyanogen's ROM has already a fix for this PSP bug, so it's maybe included in your Oxygen ROM.

But I don't think that could be the cause of a loss of battery life (the lock CPU option if you didn't use it in CSipSimple, could be a good candidate however).

Posted on Dec 23, 2010 by Happy Monkey

re: the "PSP bug", for all I know this is not a "bug" but a "misfeature" and the only way to "fix" it as it is is to just never use PSP mode while the adapter is switched on. See http://code.google.com/p/android/issues/detail?id=9781#c21 for discussion. If indeed PSP mode is not engaged in this ROM it might account for higher battery usage.

I will check this and the rest of the points when I can, and report.

Posted on Dec 24, 2010 by Quick Hippo

However... somer interesting finding about PSP mode : In 2.3 code I've noticed this code : WifiManager.WIFI_MODE_FULL_HIGH_PERF

For the wifi lock. It's not in the API (not documented at all)... but as in another thread some user with nexus S report me that Nexus S also have PSP mode, google has maybe workaround it with WifiManager.WIFI_MODE_FULL_HIGH_PERF ....

Maybe could be an excellent idea to ask custom ROM makers if they'll have that implemented to deactivate PSP mode.

In previous API logic, would have been to deactivate PSP mode on simple WIFI_MODE_FULL mode (reason why I said there where a bug for PSP mode). But maybe this new constant (undocumented but existing and used by the stock SIP app), can match the good one for disabling PSP mode !
