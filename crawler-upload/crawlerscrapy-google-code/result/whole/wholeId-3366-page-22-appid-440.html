Motorola defy+ proximity sensor does not work

Posted on Jan 1, 2012 by Swift Camel

The dial screen does not lock on the defy+ I use exDialer for GSM and that one works fine.

Comment #1

Posted on Jan 1, 2012 by Quick Hippo

For GSM, call screen probably remains the stock one.

There is 2 possible cases :

  • Your device is maybe considered as one that does not support PSP mode (I don't remember the exact list). To learn about that read : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#When_screen_goes_off_sound_quality_is_bad.

If it's the case, you can try to disable the option that prevent screen from going off. But I'm sure that finally you'll not choose to keep that because quality will be awful... And unfortunately, I (and nobody) can't do anything against that. This is about wifi card data transfer and screen that is hard-linked by some manufacturer. And the fact it does not goes off is very very very good for you ;). And if not you'd have cried that the quality is bad ... ;). This is a very known problem and a solution was theorically introduced by google engineers in android 2.3 after many guys (including me) cried about that weird behavior (that could be explained by battery consideration however).

  • It could be something about the sensor itself. After a quick search I found : http://motoroladefy.com/index.php?/topic/225-proximity-sensor-problem/ In this thread, sounds that there is a feature in the device that allow "pocket detection" that may mess up proximity sensor.

Let me know how goes your search. I'm sure that many other users would be interested in your result. There is two expert options about that in csipsimple expert setting you can play with : the "prevent screen from going off" and the "invert proximity sensor" (AFAIK this one is only required on a particular samsung device).
