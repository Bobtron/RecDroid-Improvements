Comment #4

Posted on Mar 14, 2014 by Quick Hippo

MMmmmmh, actually it's about target sdk... this one is currently not yet changed in csipsimple !

The migration to target sdk 19 is not yet done. Only migration to api 19 was done. It's very different, please have a look to android documentation about versions and keep the source code as it's delivered on the svn without modifying when raising issues here. (or actually use the nightly build version which is provided on nightly build website : http://nightlies.csipsimple.com/trunk/)

If you want to talk about something you modified and does not work after your modification, please join the csipsimple-dev google group. It's the right place to post. Your feedback is still very interesting and will allow to anticipate target sdk migration. But it's way less critical than if were a real csipsimple issue. I might fix that right now as started to try to reproduce. But will still not migrate the target sdk. It usually has hudge impact on many things and the changes in the sdk has to be reviewed very very very carefully. There is often changes about default behavior for example on power management etc etc, and changing the target sdk may completely break the core feature of the app even if it does not crash.
