Comment #1

Posted on Sep 1, 2011 by Quick Hippo

Probably linked to the fact the Galaxy tab usually require a hack to have micro working properly on calls.

Actually what is important to know is that samsung do really weird audio drivers which made necessary to use hacks in order to workaround their implementation and have things working in normal case.

However, it's highly possible that in this kind of case these workaround break things. What you could try is to use audio hacks (there is csipsimple mode where you can access tweaks). By default on the galaxy tab I apply some special settings in order to workaround their driver. I was only able to test for 2hours on a galaxy tab so I focused on the main use case (and I'm already lucky to have found something to make it working ;) ).

What you can try is described here :

http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting

You can focus on "Audio mode for sip calls" and "Micro source" which are the main settings that may impact the galaxy tab.

However, even if you find something working, in this case there is few chance I can get something working in all cases (unless you find a settings that work in all cases). This is because, actually the headphones are not managed by the application at all. Unlike bluetooth where the app should do something, the headphones should be completely managed by android system. It should be transparent for the application. Problem is that, as I said, samsung do really weird audio driver and they have probably broken things with the mode that solve problems when no headphones are plugged.

Do not hesitate to share your findings here. Probably other users have same issue and if you find some expert setting that help, I think that many other guys will be pleased to find the info here.
