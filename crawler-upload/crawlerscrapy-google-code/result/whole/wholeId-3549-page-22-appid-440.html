Posted on Sep 25, 2011 by Quick Hippo

It's maybe about the codec in use. Under 3G codec used really matters. Default usually used by providers is pcmu/a (aka g711) which is very bad for low bandwidth (because does not compress at all). If g711 is used, your call quality under 3G will be obviously very bad. (This does not depend on the client you'll use).

Maybe could be worth to try with another set of codecs. Your Nokia maybe used another set of codecs. CSipSimple already support a lot of codecs, but sometimes negociation from remote server does not comply with all SIP RFC and it is necessary to manually reduce the number of codecs (by disabling all but the one you want to force).

Comment #4

Posted on Mar 11, 2012 by Quick Hippo

The codec of skype (SILK) is supported by CSipSimple. So if you want to get the same quality than skype you can use SILK 8kHz. The problem is often that the sip server does not support it ;). So you should first ask your sip provider what are their supported sip codecs, and then choose among possible codec of csipsimple (you can search on wikipedia for codec bandwidth)
