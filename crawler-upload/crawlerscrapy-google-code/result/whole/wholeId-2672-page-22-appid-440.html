csipsimple - issue #2145

SIP registered, I can receive calls but I can't place any calls, gives 408 Request Timeout

Posted on Dec 24, 2012 by Swift Monkey

What steps will reproduce the problem? Dialing a number from CSipSimple

What is the expected output? What do you see instead? To get a call

What version of the product are you using? On what operating system? Android Gingerbread

Please provide any additional information below. I am able to register to my SIP account and I am able to receive calls, but when I try to place a call I get a 408 Request Timeout. I already tried the TLS options, but as soon as I put TLS, my SIP account does not even register, when I disable it it registers again and I am able to receive calls but times out when trying to send a call. I already sent some logs to the developers following the Logs guide.

I tried sipdroid and that one works correctly, but I like a lot more CSipSimple

Comment #2

Posted on Apr 5, 2013 by Grumpy Rhino

Hi! I decided to start over from scratch, so I removed CSipSimple from my Android, deleted everything I could from my phone's folders, and downloaded again, installed. I generated the account (my SIP provider, Axtel in MX, suggests Basic) and I filled in the required info (user, server:port, password) and the service registered as usual in seconds, and again the same problem, the 408 Timeout. So I went to settings, and avoiding the Expert Mode, I configured my CSipSimple as follows: In Easy Config I decided not to integrate w/android, always available, plus I'm allowed to use mobile. In network I checkmarked: Resolve DNS SRV, Use comppact SIP, then entered Secure transport to checkmark TLS (to Enable TLS transport), went back to Network menu, and left Enable ICE uncheckmarked, then checkmarked Enable STUN, and for both Incoming and Outgoing calls I checkmarked everything except Use other networks. In Media, I uncheckmarked Echo cancellation, and Voice audio detection, did nothing to any other parameter within Audio. Went back. User interface: per my personal needs and taste, I disabled User integration, also disabled Call logs integration, and did not modify any other parameters, went back. Within Call Options, I just setup my Caller ID, didn't enable Auto record nor Support for multiple calls. Filters: did nothing to it.

Hope this works out for anyone else having this 408 Timeout message.

Cheers! A very happy and satisfied CSipSimple user (Thanks R3gis!!!!!)

Comment #3

Posted on Apr 7, 2013 by Quick Hippo

Yep I think you did things the right way.

Basically 408 timeout occurs when packets generated by the app are too big to be transmitted over the network. This happens over UDP when the routers filters big packets and also occurs on TCP remote port 5060 with recent android (because of a big bug in 4.2.x kernel introduced by some manufacturers that didn't care about instructions from some google engineers.. and actually this bug is also on nexus 4 so even manufacturers that works closely with google did the mistake)

So, in this situation the solution is to try to reduce packet size. To do so : - enable "sip compact" mode. - reduce number of codec announced (for example select only one you know supported by the server) - disable ice if possible (because this announce each possible ip of the device) will help to reduce packet size and so pass the restrictive routers(or tcp port 5060 bug).
