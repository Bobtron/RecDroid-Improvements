Comment #1

Posted on Dec 27, 2010 by Happy Horse

I just learned about ISN numbers as a response to your post :-) For now, you can set up a filter for each ITAD you use. See our new wiki article http://code.google.com/p/csipsimple/wiki/UsingFilters. You will need to get the latest nightly build (at least http://nightlies.csipsimple.com/trunk/CSipSimple-r522-trunk.apk) to make full use of filters per the wiki.

For each ITAD, make a rewrite rule "ends with *123" replace matched text "@sipdomain.com", which will transform 98765*123 to 98765@sipdomain.com.

You might also want to put a "Can't Call" filter on the Dialer which matches "contains *", so any number in your contacts with a * in them will not allow the mobile dialer to be used. If you have numbers that start with * that you need to use, then before the above filter, put a "Stop Processing" filter that matches "starts with *". That will allow you to dial numbers that start with * using the native dialer, but if the * is anywhere else in the number the native dialer won't show.

I'm sure with some imagination you can set up the application to use the ISN numbers for the (few) ITADs you need.

Comment #2

Posted on Dec 27, 2010 by Happy Horse

For reference, without any filters, the application will send the ISN number as is to the chosen account's SIP switch. From my 3CX log (107 is my CSipSimple phone)

16:52:17.170 [CM503001]: Call(97): Incoming call from Ext.107 to

So you could set up rules/dialplan on your switch also, avoiding duplicating the ITAD rules on potentially many end-user CSipSimples.

I understand that you are asking for CSipSimple to resolve the ISN into its equivalent SIP URI, then call. A new ISN filter type would be a logical place to do this (except for the possible time lag involved in resolving it). As ISN is a bleeding edge/experimental technology, I've set this to low priority.
