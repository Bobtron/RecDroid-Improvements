Comment #3

Posted on Feb 1, 2011 by Quick Hippo

Current implementation should be already almost functional with current codec2 code. However, AFAIK, for now no sip server support this version of the code of codec2. (Freeswitch does but that is on an older version of the codec so that it will not work with csipsimple unless you update the source of codec2 in freeswitch branch).

So for now, not yet stable enough to be shipped with csipsimple. Will be when codec2 will be released for VoIP apps.

As for amr-nb... well, the biggest problem here again (like for g729) is license of the codec. That is really really really annoying for me. I know that technically it's absolutely possible and I already almost know how to do it. However, problem is license. And that sucks. An alternative could be to re-use the existing implementation available on android system which could get rid of license. (Cause if available on the android platform ,which should be the case, it mean that the manufacturer has paid the license fee for you). But using this method mean using the java interface for encoding/decoding. And I fear that it will result very poor results. If the choice is native implementation it mean reusing amr port available in android source code. It's opensource... however distributing it probably require to pay for fees. And as an alone developer (doing csipsimple just for my pleasure) I can't manage paying for the codec and I don't want to bother with distributing a paid version.

So for now amr is really low in my priority list for codecs. If I do I have to dive into the codec license and that's really borrrrriiinnnggg ... I'm a coder not a lawyer and spending time to study what I can do to be compliant with law is far to be what I prefer.

Comment #7

Posted on Feb 15, 2011 by Quick Hippo

As the sip provider does not pay for you to use the codec, it's up to you to buy the g729 license to use it. CSipSimple does not grant rights to use it. I don't know what is the policy of Asterisk when retailing their g729. You should ask them directly if buying a license for their server also grant you right to use it in a sip softphone.

g729 is included in csipsimple only cause of the pressure of guys with providers that already pay for them and that only provide support of g729 codec (and no other codecs).

If you have choice, do not use g729. Use iLBC or use Silk. That's much better and you'll not be annoyed by this polluting license.
