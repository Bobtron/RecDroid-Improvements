csipsimple - issue #2560

How can I enable g729 codec after compiling csipsimple from scratch?

Posted on Nov 21, 2013 by Happy Hippo

I have successfully compiled csipsimple project and g729 codec module on debian 7.2 using cli(not from inside eclipse). CSipSimple application starts and works on my samsung s3 mini phone. I also see ./CSipSimple/obj/local/armeabi-v7a/libpj_g729_codec.so has been compiled.

But I couldn't see the g729 codec in the codec list of csipsimple application which I have built and run. How should I enable and allow application to utilize libpj_g729_codec.so shared library? I will test the application and codec by this way.

What version of the product are you using? On what device / operating system? Samsung S3 Mini - Android 4.1.2 original rom sdk android-16 android-ndk-r9b

Kind regards, Arda

Comment #1

Posted on Nov 21, 2013 by Quick Hippo

You have to deploy the CSipSimpleG729Codec project. Normally if you didn't modified the build toolchain, the libpj_g729_codec.so file should have been moved (during dispatch step) to the correct folder of the G729Codec project instead of where you see it currently. So maybe something went wrong during build and logs from your build could be interesting. If you manually changed something in toolchain scripts, don't ;). Just revert to original code. Rebuild. And you only have to deploy the G729Codec project app (apk) on your phone (no need to rebuild and change the csipsimple BTW, you can keep official or nightly version and benefit updates and you'll keep g729 codec).

Also would be appreciated if you join and ask this kind of question on the csipsimple-dev google group as related to a custom build. Thanks !

P.S. : obviously I have to remind you that g729 is patented codec that you should not use for freedom of people you are calling and that you must not use if you are not in a case allowed by g729 usage license ;)
