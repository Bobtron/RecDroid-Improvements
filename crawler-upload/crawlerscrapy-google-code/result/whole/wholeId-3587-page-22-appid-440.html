Comment #1

Posted on Aug 14, 2011 by Quick Hippo

Ok. Several points you probably missed in the software.

You can select when you want the app to run. In your case sounds you are not interested in having it running to be available for incoming calls.

The first screen is the fast setting screen that you can also get again by pressing menu key and settings > fast settings. Here in your case I'd advise to select "Only for outgoing" option. The application will automatically be shut down when you finish to use it. (Remember that in android when you press home button you suspend an application and when you press back button you quit it).

Besides, if your use case is to be available at times and be able to quit the app... there is ALREADY this option. Press menu and "Disconnect". It will shut down the app and it will come back to live only if you re-enter it. Else, and obviously, if you have configured the app to be always available for SIP call, the application will try to run in background to receive calls. But that's because you have selected this option !!!

About speed dialing keys, you should have a look to csipsimple feature called filters and rewriting rules. And also to another option of the first setting screen : "integrate with android".

About integrate with android, by disabling it, will completly disable the integration to all other android system. Android is a very cool OS that allow that. By default it is enabled in CSipSimple, but you can disable it. But a better idea is to have a look to the filters option. You can define rules to tell which kind of phone numbers should be treaten with sip and which ones should use your mobile connection.

Please, have a look first to settings, and then to the wiki and the already raised issues of this website.

You'll find a lot of information, and you'll see that you are not the first person to think about this, and that the software is very customizable if you take time to have a real look to what it can do ;).
