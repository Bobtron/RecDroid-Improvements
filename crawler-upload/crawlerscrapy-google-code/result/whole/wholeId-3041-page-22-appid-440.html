csipsimple - issue #1753

Easy way to do IP to IP

Posted on May 22, 2012 by Massive Monkey

I think there should be an easy way to do IP to IP dialing directly without sip server. I am not sure if this is an issue, but please respond with any useful information towards being able to do this.

Comment #1

Posted on May 23, 2012 by Quick Hippo

You can : 1 - Create a local account (choose the "Local" wizard in the generic section). 2 - Switch to text dialing mode, enter the full ip/address of remote side for example test@192.168.1.1 (or sip:192.168.1.1 if calling a machine instead of a user). 3 - make the call :)

I often use this when testing video so it's known as fully working and reliable. However, take big care to not have STUN activated in this case cause you may announce to your remote side a public IP that it can't resolve.

Comment #2

Posted on Jun 6, 2012 by Massive Monkey

If the IPs are WAN ips behind NATs, will this method work (with STUN enabled) ? I tried to do it but wasn't lucky.

Comment #3

Posted on Jun 6, 2012 by Quick Hippo

No it will not work for an obvious reason : there is no way to route your packets from an IP to another in your case. So even with STUN it's absolutely impossible. STUN is to detect your public IP address and transmit it to a known public third party.

But here you want to reach another SIP client directly, with knowing only his private IP address..... Routers in the middle are not able to route your IP packet to the other side if it's a private IP address. Routers in that have to transmit the packet to the remote side just don't know where to send his packet. (actually they always know but that's more likely not what you expect).

Else it would mean that from the WAN to reach 192.168.1.1 for example, there is only one route... that reach your device !!!

If you want to understand how to make IP to IP calls, you should try to have a look on documentation on basic networking and how IPv4 packet routing is done. It's not so hard to understand. Else you'll try to do things that are obviously impossible.
