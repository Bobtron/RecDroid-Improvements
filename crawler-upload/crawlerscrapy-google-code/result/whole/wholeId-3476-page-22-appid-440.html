csipsimple - issue #1339

Adjusting "Session-Expires" value

Posted on Oct 21, 2011 by Quick Giraffe

What steps will reproduce the problem? 1. Prerequisites: a) SIP and RTP channels are separate (UA use bypass media mode) b) There is NAT between CSS and SIP server (UAS) 2. Make call, talk for several minutes, then hang up remote phone 3. CSS doesn't receive final BYE message. It's because NAT hole between UAS and CSS is closed during that call. As a result, CSS keeps that actually terminated call as it's alive for a long time after that...

What is the expected output? What do you see instead? Expected: CSS gets the final BYE message and terminates its call immediately

What version of the product are you using? On what operating system? r1060 Android 2.1

Please provide any additional information below. CSS in header field specifies this default value: Session-Expires: 1800 1800 sec is way too long time to keep SIP control channel alive in case if NAT is in the middle and if UA use direct RTP channel (control channel, separated from RTP channel). As a result CSS doesn't receive the final BYE message if call takes more then several minutes...

Is there a way to change that default value? (in UA or via config file)

I need to change "Session-Expires" value to e.g. 120 sec to keep NAT hole alive during the long call (taking several minutes or more).
