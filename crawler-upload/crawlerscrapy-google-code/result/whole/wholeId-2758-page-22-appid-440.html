csipsimple - issue #2042

OPUS codec quality is inferior - CSipSimple-latest-trunk.apk (19-Sep-2012 15:36)

Posted on Oct 29, 2012 by Quick Bird

Connecting two Android phones (Samsung Galaxy Ace Plus and HTC Desire S) resulted in quite disappointing audio quality (very harsh metallic, a bit jumpy, sound mostly in higher frequencies) using a stable version: CSipSimple-latest-trunk.apk (19-Sep-2012 15:36). Two connection types were tested with no difference in audio quality: 1. WiFi to WiFi 2. 3G to WiFi

OPUS project site presented much smoother voice even at 8kHz. In this case, the algorithm selected opus@16kHz. The quality can be compared to a worser cases of iLBC.

At the same time, tests with SILK@16kHz at the same environment gives smooth and stable voice.

QUESTION: Is there something wrong with the hardware, software or environment?

P.S. October nightly builds make no sound on Opus at all.

Comment #1

Posted on Oct 29, 2012 by Quick Hippo

Is there a media gateway in the middle? (like a freeswitch?).

Also did you update to latest codec pack version : http://nightlies.csipsimple.com/plugins/CSipSimpleCodecPack.apk

In fact the initial version used 10ms ptime for frames. Most of other implementations (and the RFC) chose a 20ms frame ptime by default. The first implementation were done before RFC finalized that's why I choose 10ms instead of 20ms. Now it uses 20ms. Current implementation will not necessarily play well with other remote frame ptime and this has still to be fixed. However, for now all remote side I tested works correctly with 20ms.

Also, there is also a known bug on Freeswitch implementation some user on the user group of csipsimple found a fix for that in freeswitch code and if you have a freeswitch you have to fix it first.

Also, on current implementation the opus codec is not told to use the audio device clock rate and the bitrate is not adjusted dynamically. So probably not perfect yet, but regarding your description it's more something either with not up to date codec pack version or something with buggy freeswitch on remote side.
