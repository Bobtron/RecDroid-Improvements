Comment #1

Posted on Jan 18, 2012 by Quick Hippo

In fact all accounts wizards relies on the same model behind. The wizard that allow to take control on all arguments of the account config is the expert one. But it's not advised to modify if you are not sip expert.

So in your case, my first advise is about voipbuster. This one is in fact a betamax clone. So you should use the betamax wizard to configure it. (You'll be able to choose the name of the betamax clone inside the wizard).

However, I think that you'll not get things working with the wizard.

Usually when you get some request timeout error it means that there is something on the network you are connected to that block sip traffic. Sapo account has the particularity to use a proxy on a different port than 5060. If you have a firewall on your wifi access point (or 3G data) that block traffic to port 5060 you could get this kind of error. In this case it's not about the device, nor about the application but about the network you are connected to.

If you still have the error and if you are sure that your network allow sip transmition, you should try to collect logs (see HowToCollectLogs wiki page). Maybe I'll find a clue on what's going wrong :)

I found the problem,thanks to r3gis, my adsl provider router in fact blocks trafic in 5060 port. here is the solution(for who lives in portugal and have same problem, with Meo or Sapo internet providers):

telnet 192.168.1.254

User: microuser Password: !C0nf16,M30

(or your root acess)

connection unbind application=SIP port=5060
