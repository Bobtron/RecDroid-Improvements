Comment #5

Posted on Nov 24, 2010 by Quick Hippo

Not sure that I've well understood whether it actually works now or not but guess yes :)

Actually that's really tricky to solve NAT issues.

Some applications activate STUN and ICE by default. However, I didn't choose to do so, cause in some countries (where the public national network is separated to the common public network - in those countries or if using a private SIP server the STUN server must be in the same network than the sip server), it can even fail to register the account. Which is worse in term of user experience (but that's just my approach and I think that both approach could be discussed). Besides for users that doesn't need it activating by default means a higher battery consumption.

So I prefer to provide the explanation in the FAQ about that... and besides if user can learn something I'm always pleased with that (goal of opensource GPL apps is also to enrich users knowledge ;) ).
