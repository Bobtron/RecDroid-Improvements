csipsimple - issue #2180

DNS SRV issue

Posted on Jan 17, 2013 by Helpful Kangaroo

Hello, we have system cluster with two servers with two public IP addresses so we use DNS SRV for failover purpose (only one server is active at the time). As soon as I enable the client it will send DNS_SRV request to translate the hostname. DNS server reply back with two server hostnames. The client then send another requests to translate these hostnames to IP addresses. Again DNS server replies back with the correct IP addresses. So until now everything is correct.

Then the CSS tries to register to first server (first IP address) but there is no reply ( because the first server is down). But instead of trying the second server (second IP address) the CSS keep sending register to the first server until it times out. For some reason CSS does not even try to register to the second server. I can provide packet capture where you can see the DNS SRV requests and replies and the SIP REGISTER packets. Is there anything that can be done to solve this issue? thank you Peter
