csipsimple - issue #1509

Homescreen Toggle Widget for switching CSipSimple on/off

Posted on Jan 17, 2012 by Massive Elephant

FEATURE REQUEST -- WISHLIST

What version of the product are you using? On what operating system? 0.03-01 r1108, Android 2.3.3

Please please please create a gorgeous toggle widget for the homescreen which allows to easily switch between the different settings -- always available [-- available in WLAN environments] -- only for outgoing calls

I am referring to this FAQ issue: http://code.google.com/p/csipsimple/wiki/FAQ?tm=6#Is_there_a_way_to_close_the_application?

It would be much more convenient to be able to turn on/off CSipSimple by simply having to tap on the screen -- rather than opening the app, going to the settings and change the configuration (which I have to do several times a day, the WLAN-option is not enough for me, at home I do not want to use it in my WLAN-environment, at work, at my parents' etc. I do want to use it in WLAN-environments).

Such a "simple" widget would be marvellous.

Thanks for this great app!

Comment #3

Posted on Jan 17, 2012 by Quick Hippo

Yes, in fact I often talked about the automated thing of the new app, but it can also be used as a basic switcher for settings : In fact one of the possible condition is a force on/off widget on home screen. So possibility are really endless with this approach. While doing single features in csipsimple is also endless but for only one feature on the other side (sip). So better to have something more generic I think.

In fact I already use this app to do so. There is some automatic feature (that switch between my company and my public personal sip account). This is done based on network. And there is also another one that I use to switch to development mode and that force activate a group of account and deactive another one.

I know that what you ask is not so difficult to implement, but if I do that inside csipsimple, it will just be a little part of what could be done and there is a lot of little things that could be done easily. My point is that : * It make more complex the sip app, that should be just about sip * It should be done by a dedicated app, that will allows to do much more things. For example activating/deactivating more complex settings. Activating/deactivating the always available or only for outgoing feature and so on.

About your workaround 1; indeed it's a good solution. Besides keep in mind that as many account you have, as fast you'll drain the battery of your phone. Each account try to keep a connection with its own server. And some server are very verbose. Even if pbxes.org is sometimes very buggy, they tried to optimize things for mobile.

About point 2 : to get the disconnect button. Go in dialer screen of csipsimple, press menu key of your phone (it's a physical button usually or sensitive one). The last entry is the disconnect option (the icon should be enough to find it :) ).
