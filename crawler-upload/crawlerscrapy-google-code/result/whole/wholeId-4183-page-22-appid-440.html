csipsimple - issue #615

loop of "downloading" and "Next" when first time use

Posted on Jan 23, 2011 by Massive Cat

What steps will reproduce the problem? 1.first time use 2.the program begins with downloading another part of the software, then press "Next" when the downloading finished 3. it performs the downloading again as before, and presents the "Next" button. the procedure above will repeat after you press "Next"

What is the expected output? What do you see instead? it's expected that the program will ask me to enter my sip account, but it loops at the first downloading step.

What version of the product are you using? On what operating system? I check out the code by "svn checkout http://csipsimple.googlecode.com/svn/trunk/ csipsimple-read-only", compile in Eclipse and run the project in an 2.2 android emulator. The OS I use is Ubuntu 10.10.

Please provide any additional information below. hi, I want add some extensions to CSIPSimple, such as Instant Messaging, Buddy List, etc., your work really does me a great favor.

//NativeLibManager.java if (standardOut.exists() && !isDebuggableApp(context)) { return standardOut; }

Please read the wiki page : HowToBuild you'll find that what you observe is not a bug, but due to the fact there is two ways to distribute the application for now (the production mode when debuggable is false and the development mode when debuggable is true). And that you forgot to build the native library part and bundle it in the libs/ directory.

For your second point : yes indeed :). But if you want to contribute you'll be welcome ;). (Specially for XCAP support that is planned but not yet implemented).

You can also play with the latest nightly build to have a good view of what is currently supported. http://nightlies.csipsimple.com/trunk/ .
