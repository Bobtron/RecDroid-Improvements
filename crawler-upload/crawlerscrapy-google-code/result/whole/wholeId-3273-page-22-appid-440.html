Comment #4

Posted on Aug 1, 2012 by Massive Bird

Hello. I'm trying to setup MegaFon (it's the provider's name, multifon is the service name) on CSipSimple.

First, some mentioned setting are not required. TCP is not required - works fine over UDP. 60 sec registration timeout is not required, furthermore, server returns Contact: ;expires=180 if timeout is less than 180 secs.

According to official website, required settings are:

account id: sip:79261234567@multifon.ru registration server: sip:sbc.megafon.ru username: 79261234567 password: myverysecretpassword proxy server: sip:sbc.megafon.ru

I have to use "Expert" wizard, because my sip id is @multifon.ru while registration and proxy server is sbc.megafon.ru.

Also, there is a problem.

If I set proxy server to sip:sbc.megafon.ru, CSipSimple sends REGISTER with Route: and server replies:

    SIP/2.0 500 Internal Server Error
    <skipped>
    Reason: Q.850;cause=47;text="Call Terminated"

Addind Route: header causes server to reply with Internal Server Error. If I delete proxy server settings, CSipSimple sends REGISTER without Route: header and registration is ok.

I've found that if "Allow DNS SRV" is checked in CSipSimple settings, I can use "Simple" wizard and just enter: User: mynumber Server: multifon.ru Password: myverysecretpass

multifon.ru has SIP SRV records pointing to sbc.megafon.ru and everything works fine.

However if "Allow DNS SRV" is not checked, there's something weird.

account id: sip:mynumber@multifon.ru registration server: sip:sbc.megafon.ru username: mynumber password: myverysecretpassword

If I leave the proxy server blank, all requests are routed through provider's server even if "Allow DNS SRV" is not checked. It seems to be a bug in CSipSimple.

Another proxy server settings:

  • sip:multifon.ru: I cannot register if "Allow DNS SRV" is not checked
  • sip:sbc.megafon.ru: I cannot register at all (SIP/2.0 500 Internal Server Error)
  • sip:193.201.229.35 (IP address of sbc.megafon.ru): everything works
