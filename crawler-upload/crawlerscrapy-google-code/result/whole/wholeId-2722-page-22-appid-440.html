Comment #1

Posted on Nov 8, 2012 by Quick Hippo

Does the VPN you use makes the connection to your VPN server the default route?

Actually, depending on the network route and where your sip server is you have to take big care of the configuration you are doing.

Here sounds not a bug in the application but a problem with your configuration. As when VPN comes there is often some big misunderstood from users that have no knowledge in network topologies and on how works SIP.

If your remote side (or server if it's a media gateway) supports ICE; turning on ICE option in settings could automatically solves the problems as ICE allows to announce more than one interface to other side. If the remote side doesn't supports ICE, you'll have to configure properly your account in expert mode so that it announce the interface you'd like it to announce to your remote side. Another more simple solution is to host a STUN server on the same network that your media gw/sip server and to enable STUN and configure stun server to be your stun server instead of publics ones.

If you have no way to add stun server in your network, I'll need more accurate details on your topology : * what does the vpn ? Does it becomes default gateway? Or just access to a LAN? * Where is your sip server? Is it on the LAN accessed through the VPN? Is it public? * Is your sip topology have a media gateway? If so where? On the LAN accessed through the VPN? Elsewhere?

You can also find more detailed technical explanation here : https://groups.google.com/d/msg/csipsimple-users/Rj8yw35DMgw/6DiUjzfasl4J
