csipsimple - issue #1366

SMS Character Count

Posted on Nov 6, 2011 by Happy Wombat

It would be better to display a SMS Character count when typing SMS because there is no indication of such on current version.

Comment #1

Posted on Nov 6, 2011 by Quick Hippo

Is there sip providers that limits size of sip messages?

Actually the feature provided by csipsimple is SIP MESSAGE protocol (it's different from SMS). So this does not really introduce a limit of size. It's a little bit more like a chat protocol. Then it's up to the gateway to the SMS world to translate into an SMS message. This is done by your SIP provider (if they have the feature).

So it would be risky to try to count caracters inside the app since : 1- nothing ensure that the remote side is an SMS gateway (so maybe it's totally useless to count characters). 2- nothing ensure that it will count the right way. For example, special chars are not counted the same ways by all mobile carriers. So it becomes even more complicated to tell from a SIP app that does not know how the SMS gateway will count characters...

Normally, for mobile carriers, SMS cost is nul. So I thought that most sip providers would not take care about the size of the message when translating over SMS network. But if your SIP providers that make you pay per "SMS" sent, I guess the fact the app does not show that is annoying indeed...

I'm not too pleased with the fact to link the SIP message view with a completely independant protocol. And I was more about to change the UI to match an instant messaging app with SIP presence, than to go in the very restrictive use case where you communicate through an SMS gateway. So, maybe this issue could be envisaged treaten as plugin that would be dedicated to the SMS use case.

Anyway I'll keep the point in mind when I'll change the messaging UI. But I can't tell you that I'll fix that in a future release cause actually that's not really a bug and that's not really a feature that should go in the scope of a "SIP" application. But If I find a fancy way to display the info as an option, in the future interface, I'll do :). Thx for the report
