Comment #6

Posted on Mar 5, 2012 by Quick Hippo

I pushed something that will fix the existing option "Mobile integration" in ExpertSettingMode to prevent gsm integration from within the app.

It doesn't apply to stock dialer/contact integration however. Because I think that it's two different scenario and both should be managed a different way.

So if you don't want to dial using gsm from within csipsimple the "Mobile integration" is the good one to change.

If in addition to that, you even don't want that the integration to contact/stock dialer allows to dial through gsm mean, you have to create a filtering rule for "GSM" virtual account telling "Don't call" for "All" numbers. (BTW, you can refine the rule here if you want to prevent calling numbers that starts with some numbers instead).

The r1300 (or upper) will be built automatically tonight. Don't hesitate if you have feedback or doubt on the way it works.
