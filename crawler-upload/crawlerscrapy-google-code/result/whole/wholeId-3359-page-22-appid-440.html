Comment #6

Posted on Mar 2, 2012 by Quick Hippo

The problem is the way you import the project in Eclipse. Since I don't commit the eclipse project files (because it's IDE files and should not be put under version control), the project can't be imported as an Eclipse project. Instead you should Create a new android project in eclipse and specify "From existing source" Take care that when you will do that you have to follow the android project property that tells which version on android it depends on (it's in project.properties file).

In fact, what you see as import proposal comes from webRTC unit test project. I don't know why they commit their eclipse props files but that's not a good idea :)

You will probably also face a lot of other issue while building because the project is a little bit complicated to understand (it's not a basic android project : there is a native library, a dependancy on other android library project). So you should have a look here : http://code.google.com/p/csipsimple/wiki/HowToBuild (You'll also notice that your problem should not appear if you follow instructions here because I tell that to import the project you have to create new project > android > from source :) ).
