csipsimple - issue #1524

Sipgate + display own number breaks phone number detection

Posted on Jan 20, 2012 by Quick Bear

Background: I'm based in Germany, but use a US-based Sipgate number (415--*) to receive calls from people in the US

csipsimple works fine on my old Android 2.3.x Nexus One phone, but on my Galaxy Nexus, there are several problems:

(a) on csipsimple's own call log, the incoming numbers are not matched up with my contacts. The numbers are shown without the international prefix (so e.g. 812-xxx-xxxx) while the contacts on my phone are all prefixed with the country code (so that I can call them regardless of which country I happen to be in). No such problem on 2.3.x

(b) with call logs integration, instead of showing the same number from csipsimple's call log (that's already unmatched since the country code is missing), instead my Sipgate number (415-*) appears.

What steps will reproduce the problem? 1. Get sipgate number 2. Add the would-be caller to Android's Contacts app, with country prefix (e.g. +1-...) 3. Get that person to call 4. Observe both Csipsimple's and Android's call logs

What is the expected output? What do you see instead? Expected: Csipsimple and Android both shows the contact's name Actual: Csipsimple shows the unprefixed number, Android shows my own Sipgate number

What version of the product are you using? On what operating system? 0.03-01 r1108

Please provide any additional information below.

Comment #3

Posted on Jul 15, 2012 by Quick Bear

This is fixable (if you're using Sipgate anyway) by changing your Sipgate setting to not display your own phone number: Advanced Settings -> "Hide own number in phone display".

It appears this is disabled as default in recent times, and when Sipgate displays both your number and the caller's number, CSipSimple shows your number instead.

@fafis - does this work for you? In which case, @r3gis, feel free to close the bug. CSipSimple from the Market has been working fine for me since I found that fix, but I forgot to update this report - apologies.
