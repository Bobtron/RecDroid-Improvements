Comment #22

Posted on Jan 11, 2012 by Quick Hippo

Hi Alain, AFAIK, MagicJack blocked couple of month ago the hack that was present in csipsimple (and other tools to use sip on magic jack). In fact they use a non standard and private way to authenticate users. Previously this way was hacked by guys on magicjack forum that told me how to implement the private authentication used by magicjack server. Unfortunately, meanwhile, MagicJack changed this algorithm. And it made useless my modification in csipsimple. That's why it's not anymore included in latest version. As far as I understood, MagicJack business model does not goes well with the fact a sip server could be accessed by clients application different than their application. So they try to do some proprietary stuff on the top of sip which make their service not really a sip server. And so it makes impossible to sip application to connect their sip service.

If there is a new possible hack to connect this proprietary way I'll include this in csipsimple again and re-add a wizard for magicjack.
