csipsimple - issue #2130

CSipSimple seems to crash when ringing to the account it is also registered at.

Posted on Dec 16, 2012 by Massive Rhino

What steps will reproduce the problem? 1. Start CSipSimple; 2. Register, for example, test1@sip2sip.info and test2@ekiga.net; 3. Call to test2@ekiga.net from test1@sip2sip.info

What is the expected output? If support for multiple calls is implemented and/or enabled, sort of "test loopback call" occurs, otherwise some error message like "User busy" is displayed.

What do you see instead? "Calling", then a black screen with "SIP registered" notification and CSipSimple showing previous screen (history of calls, in my case).

What version of the product are you using? On what operating system? "0.04-05 r2041", Android 2.3 on SE Xperia X10

Please provide any additional information below.

Logcat shows various errors mentioning "Dead object exception", "CSipSimple process died", "NullPointerException" and SIGSEGV report with libpjsipjni.so involved (I was reading from bottom up).
