Comment #1

Posted on Aug 29, 2012 by Quick Hippo

Bonjour,

Il est possible qu'une option de contournement pour l'audio soit nécessaire.

Dans un premier temps il faut vérifier que c'est bien un problème sur le routage audio de la tablette et non un problème réseau. Pour se faire, lors d'un appel, utilisé la dernière option de la barre du bas pour les Options du média.

Une popup s'affiche indiquant le niveau de son. Si la barre de son du speaker bouge c'est que le réseau est ok et que c'est bien un problème de routage audio. Dans le cas contraire, il me faudra les logs pour comprendre ce qui se passe au niveau réseau.

Pour résoudre le problème de routage réseau, il existe différentes options (cachées) dans csipsimple qui permette de s'en sortir avec les téléphones/tablettes qui se comportent bizarrement. Pour accéder à ces options il faut passer en mode expert (cf : http://code.google.com/p/csipsimple/wiki/ExpertSettingMode?wl=en#General_settings -- la section sur les options générale).

Puis aller dans media > option de contournement. Ici dans un premier temps je conseillerai de tester avec le "Audio Focus" désactivé (il semble que ça résolve des problèmes sur quelques téléphones). L'implementation audio java au lieu de openSL-ES peut peut-être également aider (bien que moins performante). Les autres options présentes peuvent également être testée. (Quelquefois même des combinaison de ces options sont nécessaires).

Comment #3

Posted on Aug 30, 2012 by Quick Hippo

Pour le mode expert il s'agit du mode expert des options générales (pas des options du compte). Ce sont les captures d'écran de la première section. Pour y accéder : Lorsque tu es sur la vue de numérotation il faut appuyer sur la touche menu du téléphone (c'est une touche physique en général). De là il faut aller dans les Options. Puis à nouveau menu pour passer en mode expert. Puis aller dans la section Media et les options de contournement (c'est la toute dernière en bas).
