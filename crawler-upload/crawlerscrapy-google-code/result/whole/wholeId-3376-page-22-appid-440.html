csipsimple - issue #1439

More android integration options

Posted on Dec 15, 2011 by Quick Camel

I would like to see the following options for Android integration with dialer:

  1. Always use cSipSimple
  2. Use cSipSimple if available (that is, don't ask and use cSipSimple when it is registered, and use the phone if it is not).
  3. Always use phone
  4. Always ask

These may not be the right wording. Right now it seems like it always asks, which is inconvenient when using bluetooth and the phone is locked. I also "disabled" cSipSimple by using the wrong credentials, but the system still tried to make calls over it.

Comment #1

Posted on Dec 17, 2011 by Quick Hippo

Well, in fact all your options are implicit to other settings you may have missed.

Always use csipsimple => simply create a filter/rewriting rule on your account to match "all" number and force call. Notice that this is more powerful than what you propose because you can do that per phone number rules.

Use CSipSimple if available => go in settings > network and choose network you want to allow for outgoing. If a network is not allowed for outgoing and you try to make the call csipsimple will automatically ignore the outgoing call and let the stock gsm call handler manage the call directly. So this option already exists.

Always use phone : same as point one but using filter/rewriting rule on GSM entry in filters (option > filters). Note that again it is much more powerful a simple switch case.

Always ask : I guess that's the default if you have not filters. Even better, if you have several accounts, the way it's currently done is more powerfull cause you have another filter to exclude one account from the list which may allow to do much more things.

About disabling one account, the correct way is not to change credentials, but just to click on the little green button on the left of the account row. It will deactivate the account and you'll not see it anymore. Also you can create a widget on your home screen "SIP Account switcher" to quickly do that.

All these features are already included in csipsimple :D :D . I think that it allows to solve your problem, and may also do better that what you ask. (for example do that based on phone number rules).

Also, another good news, I'm working on a project that may allow to do much more powerful things, a sort of locale/tasker app, that will be opensource and will have plugins for csipsimple. So that you'll be able to enable / disable accounts based on much more things. For example a time range, or the weather, or the wifi hotpspot name are anything crazy you may imagine :).
