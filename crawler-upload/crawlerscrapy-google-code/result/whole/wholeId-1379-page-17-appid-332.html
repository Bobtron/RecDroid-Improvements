android-on-freerunner - issue #2

Wifi doesn't work after suspend/resume

Posted on Sep 19, 2009 by Happy Rabbit

What steps will reproduce the problem? 1. Configure wifi and connect to a access point. 2. Wait for the phone to suspend. 3. Wake the phone--wifi doesn't reconnect.

What is the expected output? What do you see instead? Wifi should scan and reconnect to available access point. The work around is to turn the wifi off then on again, using the Settings app or by turning Airplane mode on then off. Wifi will then connect again.

I've patched the code to simply restart wifi when the Freerunner wakes up. I'm not sure that's the best approach, but it seems to get the job done without any immediately obvious downsides. It does take several seconds for wifi to connect after the phone resumes.

The patch should be in the daily build starting 23-Oct, so please let me know your results.
