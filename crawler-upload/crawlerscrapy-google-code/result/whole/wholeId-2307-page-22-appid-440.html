Comment #3

Posted on Oct 25, 2013 by Quick Hippo

The transport is something that applies to an uri. You can read more here : http://trac.pjsip.org/repos/wiki/Using_SIP_TCP (csipsimple is based on pjsip).

The standard SIP protocol specifies that the default transport for sip is UDP. As consequence any sip settings where not transport is specified will use UDP. If you do not configure the proxy uri, the only thing that can receive the transport uri is the registrar uri (current way pjsip works). Also, if you call somebody's sip uri, with your settings, udp will be used (unless you specify sip:user@domain;transport=tcp). And this is standard. You register over tcp. Well, it's fine....but you call a sip uri. So, sip stack will resolve the uri, sip rfc states that default transport to reach a sip endpoint is udp, udp will be used.

As consequence, it's always a good idea to set up the proxy uri. The proxy uri just means that for all requests you'd like to pass through your sip server (using the transport configure to communicate with it). That's what you want isn't it? (if you don't want the Route header to appear in sip packets, just add ";hide" (without quotes) to your sip proxy).
