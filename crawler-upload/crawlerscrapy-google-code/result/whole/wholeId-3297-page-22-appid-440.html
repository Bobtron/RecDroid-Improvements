Add support for for Announcify via Api / Plugin

Posted on Jan 19, 2012 by Grumpy Horse

Please add support for Announcify in CSipSimple.

Announcify is an app which reads the caller id or other event messages via TTS to the user.

If you haven't tried yet, give it a shoot! Its such a useful way to get notified of who is calling.

Really hoping for this. Thanks very much in advance.

Comment #1

Posted on Jan 19, 2012 by Quick Hippo

MMmhh, the github page of the Announcify project is down : https://github.com/TomTasche/Announcify-API

However, CSipSimple already publish the fact that there is an ongoing call. So it should be part of a plugin to do the work of re-publishing the info to Announcify (or any other third party app).

Android is well designed enough to not have to pollute one app with support of all other existing apps. Since CSipSimple publish the info and Announcify seems to be extensible via plugins (at least that's what's written on their FAQ), it should not be so hard to create a plugin that plug csipsimple to announcify ;).

But not really relevant of csipsimple project. (and maybe not of announcify as well)... Probably should be another project.

I've no time to dive into that, all the more so as I'm preparing an app that will allow to do much more things than Announcify, Tasker and Locale together ;) (and opensource/GPL ;) ) and still working on new features on sip itself such as video and messages ;).

If somebody wants to create the plugin for csipsimple to announcify, he will be welcome :). And I can provide help on the google developer group of csipsimple if needed.
