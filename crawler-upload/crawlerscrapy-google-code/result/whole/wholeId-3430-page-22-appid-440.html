csipsimple - issue #1393

With Anveo : sending but not receiving SIP MESSAGE

Posted on Nov 22, 2011 by Grumpy Rabbit

What steps will reproduce the problem? Sending an SMS through VoIP provider configured to send message to the registered device via SIP MESSAGE.

What is the expected output? What do you see instead? Expect CSipSimple to receive and display the message. However, CSipSimple does not appear to receive the message.

What version of the product are you using? On what operating system? .003-01 r1108 on 2.3.5 (installed to a Galaxy S2).

Please provide any additional information below. Messages can be sent via CSipSimple with no trouble. However, I was able to receive messages through MizuDroid, so it seems like it might be a CSipSimple issue, maybe just in the settings. VoIP provider in this case is Anveo.com.

If you are not already in expert wizard, long press on the wizard row and choose wizard > expert.

It could help if their sip server does not consider you are registered as the correct receiver for this account if you register with @sip.anveo.com instead of @anveo.com. This could explain why on the other sip client you only receive. If anveo allow to receive on @anveo.com and only send using @sip.anveo.com it may explain why csipsimple only send for now and the other only receive.
