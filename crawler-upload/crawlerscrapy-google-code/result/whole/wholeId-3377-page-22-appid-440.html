Added in r1150 + the codec pack addon :

http://nightlies.csipsimple.com/trunk/ (for r1150)

http://nightlies.csipsimple.com/plugins/ (for codec pack addon).

Once codec pack is installed, if csispimple was already started, you have to restart it in order to see it in settings. To do so, simply go in settings and press back (it will restart the stack). Then go back in settings > media > codec, and you'll see new g726 codecs.

Take care to not enable too many codecs if using UDP cause it could produce INVITE with too big size for UDP so that the packets will not be transmitted.

Once the codec pack is installed (http://nightlies.csipsimple.com/plugins/CSipSimpleCodecPack.apk), go in settings > media > codecs and long press on row you want to enable and select enable.

About codec limit it depends on : * the network you are connected on. The size of UDP datagrams varies depending on the operating environment but has a maximum of 65535 bytes. However you can encounter routers that have lower limit. That's about the transmitted data limit. * What you have to transmit. If you use a router, or if there is many sip proxy behind you and the registrar, the app may have to transmit more infos. Also if you enable the sip compact mode (settings > network > use compact mode), you may save significant transmited size. But that's not supported by old sip servers. Also in this mode some known codec are not transmitted in details in the SDP which can make almost size free to add a ~known~ codec (known codec has a fixed PT number here : http://tools.ietf.org/html/rfc3551#section-6).

So I can't tell you exactly the limit since it depends on your configuration and capabilities of your server. If it can use compact mode and that you enabled pcmu/pcma you may save 3 to 4 codec slot.
