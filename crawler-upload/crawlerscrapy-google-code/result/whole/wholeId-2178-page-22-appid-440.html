Hi, Sounds similar to issue 2589.

First thing that might be interesting to ensure is that Bluetooth is not enabled on the phone. Not only that there is no bluetooth device paired, but the BT option totally disabled. Else, under some conditions, the android audio layer might make csipsimple think it should route audio through BT while there is no BT device paired. If does not help, instructions here : http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting might be helpful (even if given for routing through earpiece/speaker, can have an impact for other audio related problems).

Comment #3

Posted on Dec 26, 2013 by Quick Hippo

Mmmh, I don't think it's 4.4.2. I have tested 4.4.2 on nexus 4, 7 and 10 (all 3 with official ROM) and works properly. Maybe 4.4.2 on this device?

BTW, are you using an official ROM (I mean a S4 google edition)? Or is it a cyanogen like ROM? Often Cyanogen ROM takes shortcuts in first implementation with audio driver. I don't want to blame them for that, they do an excellent work and they have constraints due to the fact audio drivers are not always opensourced and sometimes requires extra dev to work from a version to another. (It's same problem for many things related to drivers such as camera/Bluetooth etc). By default csipsimple tries to use the most recent features available on the android version it detects which makes it less robust to not fully implemented audio drivers; that's why that's sometimes necessary to go in audio troubleshooting options and use more legacy options (that are maybe those used by default by other sip phones). So, normally with all options/combination of options available in audio troubleshooting mode it should be always possible to find some working mode (sometimes it reduce quality/performance, but if works, better than nothing ;) ).
