Comment #10

Posted on Aug 26, 2012 by Quick Hippo

Problem solved thanks to help of one user. Optimus again changed their server and now needs to not see Route header while they serve it.

I've pushed it to latest nightly build. If you want to get it working with current stable version here's how to do :

# First create the wizard with the Optimus wizard (if not already done). # Long press the account row and select "Choose wizard", then select the Advanced wizard.

# Edit the account, and in proxy field (the latest one), change it from asbg.sip.optimus.pt to asbg.sip.optimus.pt;hide (add the ;hide at the end)

Thank you. following your tip I realized something that might be useful for other users. The password to be used with the SIP client is not the same as the one used on the Optimus Client site.
