Hi, Usually timeout error is either because the network used has a firewall that prevent SIP to be transmited to remote server, or because the remote server address is incorrect. In fact usually it is an error get because the app tries to send the registration but never get any reply from the sip server.

Can you try to collect some logs? See HowToCollectLogs wiki page. Maybe there is some clue to help you to find out what's going wrong with the network or the configuration to workaround network limitation.

Comment #4

Posted on Nov 21, 2011 by Happy Dog

Here is the answer from Talkswitch.

  1. On the Local Extensions/Fax page, an IP Extension configured as an Other IP Phone.
  2. On the same Local Extensions page, you can manually configure a username and password for registration. Some SIP phone do not differentiate between an Authorization Username, and just a username, so you may want to configure the 'username' on this page as the extension number.
  3. On the IP Configuration page, note the Unit 1 IP address, as this will be the registration server (and just about any other server you might need to configure).
  4. On the Troubleshooting - VoIP page, note the port configured for SIP. Use this when referencing a registration port on the SIP phone.

I have now successfully registered. I changed the username to be the same as the extension, as she advises. It still didn't work until I deleted the account on CSipSimple and recreated it, so I must have had something else wrong.
