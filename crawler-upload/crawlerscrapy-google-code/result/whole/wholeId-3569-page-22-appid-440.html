csipsimple - issue #1245

TLS and ZRTP Capability Unavailable on Motorola Xprt

Posted on Aug 31, 2011 by Helpful Cat

What steps will reproduce the problem? 1. Install a TLS capable CSipSimple APK nightly build on a Motorola Xprt. 2. Open the CSipSimple application 3. Press the Menu button, navigate to Settings 4. Select Network 5. Scroll down and select Secure transport

What is the expected output? What do you see instead? Once I arrive at the Secure transport menu in either Expert or Advanced mode, I expect to see and be able to select security options under the TLS menu, such options for installing certificates as well as SRTP and ZRTP under the Secure media menu. However, all I see is the Secure Media menu in SRTP mode and the only available option is SRTP.

What version of the product are you using? On what operating system? I am using CSipSimple-r1017-tls.apk The Motorola XPRT is running Android 2.2.2

Please provide any additional information below. I have loaded several other nightly builds which include TLS capability onto the XPRT [CSipSimple-r1015-tls.apk and CSipSimple-r998-tls.apk] but the same issue arises. I have tested the same August 31 2011 nightly build on a Motorola Droid Pro running Android 2.2.1 and all of the options are there. Finally, I have tested other TLS-capable nightlies and on a Motorola Atrix running Android 2.3.4 and all TLS and Secure Media options are available.
