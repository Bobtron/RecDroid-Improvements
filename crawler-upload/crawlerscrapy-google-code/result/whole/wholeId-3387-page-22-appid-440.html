Comment #1

Posted on Dec 5, 2011 by Quick Hippo

5060 is optional regarding sip rfc. So it should not matter. If it matters for the sip server, it would mean that the sip server is not correct regarding sip rfc.

I guess by 3cx you mean 3cx for android. The weird point is that 3Cx actually use exactly the same sip stack than csipsimple (but they are closed source project). So there is probably one configuration that should work ;)

So could you : 1- try to collect logs (see HowToCollectLogs wiki page). Maybe there is some clue about why server refuse the register. 2- try to change local binding port (in csipsimple's settings, press menu key, choose expert mode, then in network in udp port change from 0 to 5060 (the default 0 means that local port is random)
