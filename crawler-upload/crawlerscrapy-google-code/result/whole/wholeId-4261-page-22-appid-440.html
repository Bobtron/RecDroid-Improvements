Comment #3

Posted on Dec 29, 2010 by Quick Hippo

Ok so pretty normal :) I think that if you are on a wifi network back button will not close your sip registrations.

In fact, that's pretty normal : on the current network connection csipsimple is configured for outgoing calls only. So if you press back button somewhere in the app to quit the app, it will unregister to save your battery (that's the purpose of these options : saving battery). So if you are in main csipsimple dialer and press back it will unregister, if you are on dialer outgoing chooser and press back it will unregister. However, don't worry, if you dial again from contact, CSipSimple will wake up, register and propose you for what outgoing account you want to choose.

This method allow to start the app only when needed (when you do an outgoing call) and save a lot of battery.
