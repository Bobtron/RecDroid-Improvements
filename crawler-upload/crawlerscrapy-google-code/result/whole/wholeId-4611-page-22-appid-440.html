HTC desire - Keep awake while in call > set default to true

Posted on Sep 21, 2010 by Happy Monkey

Sound coming from the other side became garbled in -35 and still such in -36. I installed -34 and verified that that sound is good in that version (so this is not a connectivity problem or something).

HTC Desire running 2.2, the other side is a "talking clock" app so its quality is consistent across tests.

Comment #3

Posted on Sep 21, 2010 by Quick Hippo

Oh ! I'm just wondering something.... HTC desire is maybe affected by the psp mode too...

So are you in wifi when you make tests and does the screen turns off when your face is near to the screen? If so, it can be due to the fact now on all other devices except the nexus one I activate full proximity sensor instead of keeping screen in dim mode.

So what you can test to check that, is go in settings > UI > and tick "keep awake while in call" option. Then screen will not turn off but wifi will correctly receive rtp packets.
