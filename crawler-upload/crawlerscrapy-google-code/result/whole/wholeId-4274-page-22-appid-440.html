csipsimple - issue #565

Add SOCKS5 proxy support

Posted on Jan 4, 2011 by Grumpy Dog

Add support for using a SOCKS5 proxy.

This will allow the data to be encrypted.

Note that a SOCKS5 proxy can be easily created by setting up a dynamic port forwarding to a ssh server using ConnectBot for example.

Comment #1

Posted on Jan 4, 2011 by Quick Hippo

Just for info, a better way to have encrypted sip is to use SIP+TLS (SIPS). Which is already supported by CSipSimple in special TLS builds.

Sure, but with a SOCKS5 proxy nobody can tell it's SIP, which is what I'm looking for.

Comment #3

Posted on Jan 4, 2011 by Quick Hippo

For info number 2 :) : This enhancement should be done in the pjsip stack (the opensource SIP stack on which we are relying). As soon as the stack will support SOCKS5 we will support it too in CSipSimple and it will also benefit all programs based on pjsip sip stack (the ones on iPhone, on PC, on MAC on PSP etc etc etc). So I don't think that 'we' (I mean CSipSimple developers will take in charge that) but if anybody wants to contribute that to pjsip he will be welcome. ZRTP enhancement was for example done initially to be integrated to CSipSimple (I've still do to the last meter of integration ;) ), but will benefit every apps that rely on pjsip (on android CSipSimple and the closed source 3CX and many others apps on other platforms). I did the SILK codec support that I contributed to pjsip. But usually I don't change the reliable and stable pjsip stack (cause I'm not really competent in this domain and cause it's harder to maintain then).

Regarding pjsip wiki it's planned by pjsip developpers : http://www.pjsip.org/pjnath/docs/html/ . So if anybody wants to help them on that point I'd advise to go on pjsip mailing list ;).

However I keep the point open to allow CSipSimple users to track the issue. But be aware that it is not a enhancement "directly" relevant on CSipSimple but of pjsip.
