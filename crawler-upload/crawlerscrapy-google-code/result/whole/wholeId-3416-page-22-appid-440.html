Comment #6

Posted on Nov 15, 2011 by Quick Hippo

MMmmmm, interesting. Sounds not directly linked ot csipsimple however. CSipSimple just increases the fact there is something that becomes crazy in background.

I think that one key point is that your tablet is GSM capable. Was not the case on the one I tested on 6 month ago and the one that other users I talked with had.

Regarding logs there is a lot of events raised by your system about the wifi and the available hotspot arround you. Normally the OS should do that few times but here it seems to do that a lot of time. Since CSipSimple is registered to this kind of event (in order to keep the registration with sip server up to date if necessary), it is launched a lot of times. In fact, in logs there is few lines written by csipsimple, but a lot by the wifi driver card and android wifi manager. There is also a lot about on wifi access point notification which leads to a lot of sound device openning. But regarding csipsimple logs it look likes what it should look like on normal devices.

However, even if Csipsimple does really few things, I think that the simple fact that there is a raise of a wifi event on android system, and that this event inform csipsimple, pledge the CPU.

The question is, what is the root cause of these events?... I guess it will not only impact CSipSimple but could impact other apps that cares about connection reachability dynamically (it may also drain your battery faster).

Maybe it's a bug on samsung side, would not be the first (there is already a necessary workaround automatically applied for calls audio routing).

Did you experimented this problem with other SIP apps ? (Most of SIP/VoIP apps needs to register this kind of event). Also, it could be interesting to test over a wifi connection. Since the events I see in logs should never be raised while in wifi it may behaves differently. If my first analysis is correct you should try to investigate independantly of csipsimple the root cause of the problem. It's not so annoying if you can't use csipsimple (there is many other cool SIP softphone on android :) : linphone, imsdroid, sipdroid) (my goal is just to make users happy, if that's with another opensource software, it does not really matter for me ;) ). But would be more annoying if it's something about your ROM : would mean your battery drains faster than it should, and that's more annoying for you.
