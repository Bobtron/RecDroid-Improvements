csipsimple - issue #1271

DTMF tones are not generated on Xoom tablet

Posted on Sep 15, 2011 by Helpful Kangaroo

What steps will reproduce the problem? 1. Install latest version of CSipSimple on Xoom (nightly build experiences same issue) 2. Call any toll-free number that requires DTMF (cheapoair is a good one) 3. Press any number on the dial pad

What is the expected output? What do you see instead?

The automated menu should recognize the key being pressed, but does not.

What version of the product are you using? On what operating system?

CSipSimple 0.02-03 r944 Android 3.2

Please provide any additional information below.

When set to use RTP, every key generates tone 0. This is proven by calling the cheapoair help line, because the menu tells you that you're pressing zero. And yes, I was out of the country at the time trying to make a flight change! In-band generation does nothing at all, and won't even generate a tone client side. 3rd party tone generators cannot be used because CSipSimple locks the audio stream.

It's been over six months since I've started using CSipSimple, and this problem has never been addressed. I often have to resort to using SipDroid for calls that require DTMF due to such a simple feature not working.
