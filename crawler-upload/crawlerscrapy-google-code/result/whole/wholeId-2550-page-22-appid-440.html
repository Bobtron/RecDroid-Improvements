csipsimple - issue #2292

Incoming Sip Port

Posted on Mar 19, 2013 by Quick Camel

Since all Providers in France Have used Incoming Port 5060 for own telephony it would be great to have an option to set another incoming sip-port.

Comment #1

Posted on Mar 31, 2013 by Quick Hippo

Je ne suis pas totalement certain de comprendre la demande. Mais * si c'est de changer le port d'écoute local de l'application, une telle option existe déjà (dans les options, activer le mode expert puis choisir le port ici). Par défault c'est 0 ce qui veux dire que l'application choisi un port au hasard.

  • si c'est changer le port du serveur distant, il suffit de changer la configuration du serveur dans la configuration du compte en rajoutant ":xxxx" où xxxx est le port.

Dans les deux cas je ne vois pas le rapport avec les sip provider en France. Je fais tout mes tests en france avec plusieurs sip provider et je peux utiliser en local port 5060 ou un aléatoire et pour le port distant il me suffit de configurer celui du fournisseur SIP (et en effet ils utilisent quasiment tous 5060 comme a peu près tout les fournisseurs SIP du monde car c'est le port à défaut du protocole... comme le port par défault du http est le 80)
