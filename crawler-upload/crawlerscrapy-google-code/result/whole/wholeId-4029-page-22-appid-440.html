csipsimple - issue #781

Failed to create transport Permission Denied (120013) - when stored on SDCard

Posted on Mar 8, 2011 by Helpful Giraffe

  • Steps to reproduce the problem.

    1. System-initiated update of stock, unrooted Nexus One running CSipSimple from FR83G to GRI40 (Gingerbread). Immediately and frequently the errors began.

  • Expected output: no errors

  • Actual output: popup errors on top of whatever app I'm working in: "Failed to create transport Permission Denied (120013)"

Versions: Stock Nexus One, Gingerbread 2.3.3 (GRI40) Kernel 2.6.35.7-59423-g08607d4 CSipSimple 0.01-01

Notes: Immediately after the update I started getting random error messages, message pops up on the screen all over the place, not just when using CSipSimple. Messages stop when the service is killed, and when restarted the messages seem to disappear for a while, but after a soft-reset they come back.

Comment #1

Posted on Mar 9, 2011 by Quick Hippo

Have you changed default local TCP/UDP or RTP port bindings in settings?

Usually when you get this error it's cause actually android os does not allow you to use a port for local binding. With default ports ~should~ not happens.

Besides, if you switched to android 2.3.3, it could be very to reset all your default settings anyway. As android 2.3.3 is not affected anymore by the PSP behavior (in fact it provide a way for developer to lock it), So I'd advise you to reset default settings in csipsimple. (Go in settings then menu and then reset settings). (Don't forget to reactivate STUN if it was activated).

Comment #10

Posted on Sep 19, 2011 by Grumpy Wombat

I found why this happens on boot, Regis, if you hadn't had time to track it down... I'm poasting here so other affected users can avoid the time expenditure of discovering how to fix it.

You can re-create the problem by using the expert settings to set the global UDP port to a number x, where... (x > 0 && x < 1024).

The reason is that the native library is not loaded by the root user, and so, doesn't have the permissions to bind to a low port number. In linux, ports below 1024 require root to establish a listening connection.

In PjsipService.java, the library is initialized and the transports added. When this line happens...

udpTranportId = createTransport(t, prefsWrapper.getUDPTransportPort());

The port is (for most users, I'm guessing) 0, and this works fine. When I register with a SIP server using a non-standard port, my port number is stored in the account data.

If you want to use a SIP server on a low port (IE, 443), append ":443" to your SIP URI in the account settings, and leave the global UDP port set to zero.

Comment #22

Posted on Jun 28, 2012 by Helpful Bird

Additional observations about running from SDCard:

I ran CSipSimple from the Micro SDCard for a long time on a Nexus One, (due to tight memory on that phone).

Lately, most mornings I would find that CSipSimple had simply stopped functioning, and was not running. Even though the phone had been running fine all day long the prior day, on wifi on the charger.

I moved the app back to phone storage, and these mysterious terminations seem to have stopped.

So if it stops or terminates without any warning message or explanation while the phone is sleeping make sure its not on the micro sdcard.
