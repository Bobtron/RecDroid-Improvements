Comment #3

Posted on Sep 3, 2011 by Quick Hippo

About stun errors problem, there is indeed a known issue about that. For some unknown reason depending of configuration it can produce this error, but I still do not understand why it block the registration since stun should be not blocking point (even with a stun server that does not reply). I'm investigating this problem and will keep you in touch. You can try to disable STUN (cause maybe not required in your case).

About audio quality, when quality was poor, was it on wifi or on 4G/3G? Cause in fact, when using over 3g/4g there is a big matter with codec in use. It could be interesting to see which codec is in use while in call (pressing the little arrow under the contact card and the Info option). If it is using g711 (aka pcma/u), a bad quality on mobile networks is "normal". This codec use a big bandwidth (64kB/s) and mobile networks does not support that very well which leads to choppy sound (or no sound at all). If so, could be interesting to force other codecs. I don't know which are supported by voipcheap but if you try to disable all but one not supported by voipcheap, calls will just directly fail, so easy to see if a codec is supported or not ;).
