csipsimple - issue #451

AMR codec

Posted on Dec 6, 2010 by Swift Panda

The AMR codec (nb) is part of the IPP Intel stuff which is already included in pjsip. I know you have to get a license for this codec if you want to use it. So probably it isn't possible to generally enable Intel IPP in csipsimple.

However, would it be possible to let the user compile his own version of csipsimple which includes IPP? It would be the responsibility of the user to purchase licenses for AMR (or G.729).

I am looking for a way to use AMR-nb with csipsimple and I am willing to pay for that codec.

What's the benefit of AMR-nb? Well, it is the best option if you want to make calls over GPRS/UMTS and have good call quality. The backend PBX must support it too, but there are solutions available.

csipsimple with AMR-nb + TLS/SRTP would be a perfect combination for secure calls over mobile networks.

Not possible to build IPP codec simply cose ARM processor are not compatible with IPP codecs. (Try to build, you'll see a lot of compil errors due to the fact IPP use asm for intel procs).

Comment #3

Posted on Oct 10, 2011 by Quick Hippo

Started for NB. Will rely on availability of a private library on the platform (libstagefright introduced in android 2.3). So no issue with licensing since will be delegated to the core platform (and potential speed improvements depending on the way it is implemented on the platform).

However, I need a sip server with this feature to validate my code (for now it works from CSipSimple to CSipSimple but that does not garanty it will works with other sip user agent that support AMR since AMR is pretty complicate and the fact I rely on something dynamic does not make thing easy ;) ).

So once validated, I'll release something to be tested (probably soon). Then maybe it will be possible to backport for older devices by relying on opencore instead (2.2 and lower).
