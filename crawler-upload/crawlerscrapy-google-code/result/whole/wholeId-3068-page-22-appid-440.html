Could this depends on my configuration or something introduced recently (if the case,I don't know since which revision)?

Comment #1

Posted on Jun 12, 2012 by Quick Hippo

Mmm, normally no noticeable changes has been done recently regarding cpu usage. For RAM it has probably grown due to new resources.

The process you see with 1 / 2% is com.csipsimple or com.csipsimple:sipStack? The first one is the user interface while the second is the sip process.

Also what could have impact is the use of STUN. Is it activated in your configuration? Also, did you notice any high battery usage from CSipSimple in the stock setting battery view? CPU % reported by some tools are sometimes not inline with actual battery usage from an app.
