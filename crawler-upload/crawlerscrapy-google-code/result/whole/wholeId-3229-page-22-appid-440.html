Comment #1

Posted on Feb 15, 2012 by Quick Hippo

How it goes if you go in csipsimple settings in : Press hardware menu key, select expert mode Then go in User interface section, and check prevent screen rotation.

The app probably detect that the device has a big screen and activate the tablet mode (which disable the prevent rotation mode by default in version on the android market -- with nightly builds it does not normally).

Comment #5

Posted on Feb 15, 2012 by Quick Hippo

Yes, my approach is to try to keep things as simple as possible for 90% of use case and autoconfigure settings depending on user feedback.

The expert mode is there when there is a problem to try to tweak the app :) So it's normal you didn't see it at first time (it's voluntarily hidden ;) ).

So, thanks for your feedback, now I know that on galaxy Note the autodetection of screen size and auto rotation is probably not as good as it should. So I'll try to take that into account in next version. Maybe nightly builds are already better regarding that because I changed the way rotation of views are managed (if you want to test a nightly build : http://nightlies.csipsimple.com/trunk/ )

CSipSimple is opensource and free as in speech. It's made of user feedback, so thanks you very much for the feedback : it was a very good idea to open an issue about that. It will help to improve the next release. (the feedback from nightly build is also welcome; even if not as stable as the market version it's the current development status and the future of the app, so user feedback is also valuable on these versions).
