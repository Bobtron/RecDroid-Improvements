Comment #1

Posted on Feb 15, 2012 by Quick Hippo

Did you check : 1) CSipSimple is configured as always available (in simple settings mode) 2) your android system is not configured to turn wifi off when screen goes off. This is absolutely not linked to csipsimple, but a global settings you can find in android settings, in network settings in wifi section > advanced option (press menu key when on wifi access point list), and it's the option "wifi when suspended" or something like that.

Normally, if csipsimple is configured to be always available it will take a lock on wifi. If not it will just follow the android settings and not force the wifi to be active. I guess that you are in mode "available on wifi"... which mean activate csipsimple when wifi is on. In this case, it entierly rely on android settings and the app will not try to lock the wifi at all. In this case also, it depends on the setting you decided to set on your android system. I guess that the default changed with android 2.3.4 update of sony and so you think it's a bug in csipsimple while it's just about your conf of both csipsimple and the new default option.

Because, in fact, there is actually a way to configure csipsimple + android so that it behaves the way you don't want it behaves. ;) But there is also 2 ways to make it behaves as you want (either forcing csipsimple to take a wifi lock, or just saying android to not cut wifi when screen goes off).

Comment #5

Posted on Feb 15, 2012 by Quick Hippo

mmm, yes maybe a sony bug indeed.

Just to be sure,

In CSipSimple, go in settings, then press hardware menu key to switch to expert mode (if not already the case). Then in network setting, is the first checkbox 'Lock WiFi' activated? (The second about high perf should be left disabled).

If so and if the android setting to keep wifi locked when screen goes off is also on... well I would say that's a bug on android ROM.

What could confirm that it's a bug on android ROM is that, the setting in android settings is independant from CSipSimple. You could remove csipsimple, if you want wifi to remain on, the android setting should work as well ;) : it's a pure android setting to tell that you want wifi to remain active when screen goes off, and has nothing to do with csipsimple.

In case of "lock wifi", csipsimple try to also take a lock on the wifi as the stock settings should do. But obviously if it does not work for stock settings application there is few chances that it will work for csipsimple ;).

In this case, you should report the bug to sony.
