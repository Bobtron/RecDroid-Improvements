Comment #1

Posted on Feb 8, 2011 by Quick Hippo

For some strange reason the SIP server does not reply ACK to the 200 OK sent by CSipSimple...

However, BYE works flawlessly... (BYE is done by csipsimple after timeout cause other side never ACK its 200 OK messages).

So two questions : * what is your SIP provider * did you try to change your account into an expert wizard account (long press on the account row, choose wizard, and choose expert) then edit it; and then scroll to the bottom and clear the sip proxy field. (It will change the way the SIP client communicate with the rest of the world and if your sip server does not support proxying request it will send direclty its request to the remote contact that may reply).

If last point help, I'd be pleased to add your sip provider as a wizard to avoid the same annoyance for other CSipSimple users.
