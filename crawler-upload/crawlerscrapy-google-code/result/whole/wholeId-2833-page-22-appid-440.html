Comment #1

Posted on Sep 11, 2012 by Quick Hippo

Hi,

What is your sip provider?

In fact in sip you have several names to design same things. 'Caller Id' doesn't really exists in SIP standard. There is username and display name that could be intereted as caller id by the sip provider. By default in CSipSimple I call 'caller id' the 'display name' and username/authentication name the username. Then it depends on sip providers. That's why there is wizard and in each dedicated wizard I give the name as really used by the corresponding sip provider.

Maybe if your sip provider uses username as caller id, you should use advanced wizard to setup the username (and have authentication name different).

Naming is really a matter of sip provider. CSipSimple with expert and advanced wizard allow you to configure everything possible in SIP. So you are absolutely not blocked. The only thing you need to get is what is the field your sip provider considers as caller id. By default csipsimple consider 'Caller id' to be the display name. It's different in some dedicated wizards where I know the sip provider to not user display name as caller id but rather username. It's the case for example of all Betamax clones. If your sip provider is not in the list and considers another field as caller Id, I can add a dedicated wizard for it.
