csipsimple - issue #2307

OTR support

Posted on Apr 2, 2013 by Happy Bear

off-the-record messaging would be great for thic app, please include this. Thanks

Comment #1

Posted on Apr 3, 2013 by Quick Hippo

Well OTR is designed for IM sessions. In SIP there is basically two ways to do IM. One (the currently one used in csipsimple) is SIP SIMPLE protocol. It has a lot of drawbacks and is actually not the way the standard is moving. The other (the one that I wish to see in csipsimple) is the MSRP sessions.

In case of SIMPLE protocol OTR could be added. However : - it's not standard (there is no RFC to describe the protocol and I usually fear to use non standard things even if open). - it would require to make this part a little bit more "pluginizable".

On the other side, if I want to spend time around this topic (and keep in mind I do the project on my free time as a hobby -- so nights, weekends and holidays), it will be to try to add msrp to pjsip (and btw will not only benefit csipsimple android app but any other apps relying on pjsip).

So to sum up previous points : OTR is not in the roadmap. If anyone wants to contribute something it will be welcome. Ideally as a plugin or at least as something that is separated in code from main stuff that implements the RFC standards (and doesn't modify main flows) and can be enabled/disabled by users at choice.
