Hi, the issue was NOT conflict when calling from VOIP TO GSM. But it was that when you had an ongoing SIP call, and get an incoming GSM call (on the same phone) the application had something to do to manage properly the incoming GSM call that comes from the GSM application. This has been fixed normally.
