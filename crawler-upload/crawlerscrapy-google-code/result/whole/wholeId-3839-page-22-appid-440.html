Comment #4

Posted on Jun 16, 2011 by Quick Hippo

Yes, normally the idea is to have that done in each wizard. The goal is to try to cover as much sip providers as possible and if it's a known provider user should not care about the number cause the wizard already know it :).

I think that the fact it is complicated is not a so bad idea :

Having it somewhere else will complicate things for all users while they should not care about that (it should be done in the wizard automatically)

I try to keep the interface and the global feeling in the app very simple. You'll argue that it sounds much more complicated to you to change the voice mail number, but I think that it would be even worse to add 100 options for all users everywhere in the app... that would be more simple for me, but the user will be lost in so many settings. You don't imagine how many settings that could be presented to an user to configure a sip account. Actually, you could have an idea with the expert mode... one settings of expert mode could be absolutely required for one user... should I put that in the basic wizard?? I don't think so.

I think that if you really have the problem : * mistake the first time * use your sip voice mail * have a sip provider without wizard with prefilled voice mail number yet I don't know which % of user it is but I guess less than 1% of users.

Hopefully this 1% of users will search or ask (just like you did, and that's a really good reaction :) ).

And in this case, two good news : 1 - feedback about the fact there is possibly one missing wizard + voice mail number to set automatically and something that I could fix easily so that all other users of the same provider will not have the problem anymore 2 - the user will be able to solve his problem.

Also, that's not so hard to do : you just have to long press on an existing account change it to expert wizard mode (choose wizard option and expert), edit, change VM number, save and if you want then switch back to the previous wizard. Do you change this number each day? I hope not ;)... so about 5 minutes for all the rest of the usage time of the app...
