csipsimple - issue #2021

Can't bring up keypad during a call

Posted on Oct 12, 2012 by Happy Dog

What steps will reproduce the problem? 1. Receive a call (in this case, from Google Voice activation) 2. Answer call 3. Full screen caller ID splash during call, can't find option to bring up number pad, which I need to enter activation number.

What is the expected output? What do you see instead? Upon answering call, expected key pad, or at least option to bring it up. Instead, it is a large (blank) caller ID splash, and options only for speaker, mute, call record and hang up. For some reason, I can minimize the call and access the main CSipSimple app and press the key pad there, but it doesn't register in the current call. Same with stock dialer.

What version of the product are you using? On what operating system? Version 0.04-04r1916 on Android 4.0.3 (Samsung Galaxy sII SC-02C)

Please provide any additional information below. In addition to CSipSimple and the stock dialer, I am also running Viber app. There seems to be automated interaction between Viber and CSipSimple that I don't know how to control.

THANK YOU FOR A USEFUL APP AND ALL YOUR HELP!

Normally the dtmf dialpad is an option that appears on the first black bar. Just bellow the caller picture.

It doesn't appears if the call state is not active. Maybe it's your root problem. From other users feedback callcentric calls doesn't reach active state anymore (see issue 2022 for more details about why). Maybe you are in the same case?

For the dtmf button you can see it on this screenshot : http://www2.r3gis.fr/CSSPublic/screen3.png (it's on the left of the black bar above the hangup button). If you don't see it here, it's probably because call state (the text inside blue bar on the top) is still "ringing" or "incoming".
