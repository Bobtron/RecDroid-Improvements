csipsimple - issue #1606

Lockup on badly-formed registration URI

Posted on Mar 1, 2012 by Helpful Monkey

What steps will reproduce the problem? 1. Create a SIP account using the Expert wizard. 2. Instead of "sip:example.com", enter "sip:.example.com" for the registration URI 3. Attempt to connect with SIP account. (Assuming you have an appropriate network path available for SIP)

What is the expected output? What do you see instead? Expected: The Registration URI setting should be marked in the Red/orange "invalid required setting" color while still in the wizard. Actual: Setting appears correct. Upon connection attempt, CSipSimple locks up. I had to enable Airplane Mode to get in and poke at the settings.

What version of the product are you using? On what operating system? CSipSimple 0.03-01 r1108 Android 2.2.2 Kernel version 2.6.32.9 Build number FRG83G Software version LG-P509-V10s

Please provide any additional information below.

The difference between the valid and invalid registration URIs is a "." prior to the first portion of the FQDN. Because of that ".", the FQDN is invalid, but (I expect) either the DNS resolver being used by CSipSimple doesn't recognize that, or the code which uses the DNS resolver's output isn't recognizing the error'd result from attempting the query.
