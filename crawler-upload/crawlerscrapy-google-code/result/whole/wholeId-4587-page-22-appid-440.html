Comment #8

Posted on Sep 16, 2010 by Quick Hippo

Ok, that confirms my tests with the phone of my collegue (worked properly over wifi - but I didn't test over 3G since our provider doesn't allow it). Most of the time in such case, can be due to the fact something in the SIP configuration doesn't allow to pass through the NAT of your 3G carrier. Have you a stun server configured? In many case it can help with this kind of issues.

To enable stun server : Settings > Media settings > Tick enable stun and fill stun server with a stun server. If your sip provider has a stun server use it, else you can try stun.ekiga.net for example.
