csipsimple - issue #2522

Feature Request: Obtaining NAT type

Posted on Oct 17, 2013 by Happy Lion

What steps will reproduce the problem? 1. 2. 3.

What is the expected output? What do you see instead?

What version of the product are you using? On what device / operating system? r2307

Please provide any additional information below.

Some draconian NATs may cause big trouble for sip calls, especially symmetric NATs. For diagnostic purposes, it's often desirable for the user to know which type of NAT they are in. But AFAIK there is no app to do that in android.

PJSIP already has this functionality, and csipsimple is using it. But csipsimple does not give any relative information, even in Level 4 logs.

Can you just add such log information? Of course it will be much better if the UI can display it.

Comment #1

Posted on Oct 18, 2013 by Quick Hippo

Why not. Should not be too hard. For reference, it was explicitely removed from SDP (as by default pjsip add the X-nat in sdp) because bigger sdp means bigger udp packet and often means to get blocked by restrictive routers. But, in the UI, why not. The info about the calls are the good place to put this (as it's pretty technical info it should not go in the main screen but rather in the 'deep info' popup).
