In fact you don't find that in the code for a simple reason. I do not use the variable name "WIFI_MODE_FULL_HIGH_PERF" simply cause that's not in the public API so it will not build if I use the variable name. As you can read on my comment on android issue, the workaround is very simple, it is to replace the constant by its value :)... So you should have search for "3" in my code ;) lol.

Then, for CM7.... well you have to know that this will works on CM7 only if Cyanogen use a driver that is built to support this high pref mode. I don't know if he has already this code. If not it will just behaves the old way. That's the reason why I'm waiting on my N1 to get the official google update cause I suspect wifi drivers that custom roms has to not have yet this fix. But that's something that will be required for all stock rom in 2.3. Also the good news with 2.3 is that all problems I have with routing on all device will be eliminated cause if the manufacturers wants to support 2.3 and the stock SIP application they will have to follow a strict and correct implementation of android audio API....

So Google with this new feature has added everything a SIP application need. And it will make my life much more easy :).

Also just FYI, the new method for keep alive is directly from the stock SIP application ;) ... well probably not perfectly integrated to pjsip yet as per issue 638... (if you have updates on this issue with r602 I'm interested I think that I'll release this revision on the market).
