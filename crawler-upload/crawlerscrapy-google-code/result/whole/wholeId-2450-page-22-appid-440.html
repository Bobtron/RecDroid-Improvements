Comment #4

Posted on Jun 5, 2013 by Quick Hippo

From some user feedback the troubleshooting option (http://code.google.com/p/csipsimple/wiki/FAQ?wl=en#Audio_routing_troubleshooting) helps to solve the problem. It seems that the "audio tone hack" is necessary with recent sgs cyanogen.

You can't tell the problem is not with the firmware however. The android audio stack is not understood the same way by all manufacturer/ROM maker. Applications must implement workarounds per device. (that's the settings you see in the audio troubleshooting section). Apps has not all same default settings/device detection. For the SGS1 for example, there is some known mandatory workaround to get things working with stock ROM from samsung. Maybe this workaround completely break things when you use a cyanogen mod rom and must be disabled.
