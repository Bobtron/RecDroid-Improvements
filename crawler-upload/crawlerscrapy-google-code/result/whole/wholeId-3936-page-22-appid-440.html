csipsimple - issue #898

Posted on Apr 18, 2011 by Massive Rhino

What is the expected output? What do you see instead? Not sure if CSip should drop the call cleanly or continue on. LOGCAT spits out the below info when the back button is hit a second time.

The call does continue, but if you try and re-enter the UI the program will crash and die.

What version of the product are you using? On what operating system?

Nightly build and SVN. Android 2.2.

Comment #2

Posted on Apr 19, 2011 by Massive Rhino

Additional info:

  1. I don't have a BlueTooth device.
  2. My Network settings are for "incoming only" when the crash happens. When I select "always available", there is no crash.

Seems CSip checks for incoming networks when trying to go to the home screen with the back button. If set in Always Available mode and in call, you can exit and return to CSip without issue.

Looks like CSip needs not only to check if there are Inbound networks, but also whether a call is currently in progress. It seems to want to exit totally, even if a call is still setup in the SIP stack.
