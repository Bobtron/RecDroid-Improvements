As work to make the app more dynamic, we should be able to control as much as we can server side. There isn't really a good/standard way to do this, so this will be a cool opportunity to build our own infra.

My Idea

  • Use GitHub pages to host json config filese
  • The app can ship with the set of configs available at build time, and then download/update them as they change (like how #710 was done)
  • The TBA server can track the last "config update" time via a GitHub webhook (we subscribe to it and automatically update the sitevar)

This way, most features can have "kill switches" to stop things that break or to provide a server-side switch to turn new things on.
