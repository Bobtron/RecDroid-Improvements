Description

When moving items around using the talkback actions the launcher will crash. I did send logs in February or so but so far this has not be fixed

Expected Behavior

What should happen is this, the launcher should not crash if I let's say move an icon from 1 page to another, or even into a folder.

Actual Behavior

The app crashes and the close and send feedback that android brings up shows up.

Steps to Reproduce

  1. Make sure launcher is set to launcheer.
  2. Hit home and find an item you want to move
  3. Bring up the talk back actions menu
  4. Tap move
  5. Tap on the new place you want to move the item to.

This is when the crash happens. The items are moved but the app will still crash. It works afterwords but it should not in theory be doing this.

Environment

Device: moto z2 force
Android version: 8.0

I've forgotten the latest version of the launcher but it is what is on the app store. I'm not a part of the beta program so this is a public release.
