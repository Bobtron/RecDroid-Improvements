@valgit What I meant was to select the Hit/Miss target face and just enter e.g. hit for every arrow that was shot. This allows to count the number of arrows shot and is shown separate from the the other target faces in the statistics. This would at least be a workaround for now.
Your suggestion was to add a completely separate category for "blank bale" training where you can:

  • enter a default arrow number per end (I would leave this option aside and simply reuse the arrow number of the previous end)
  • the input screen consists of a simple counter with +/- buttons to increase/decrease the arrow count of the current end
  • the next button adds the specified number of arrows shot to the end

Correct?
