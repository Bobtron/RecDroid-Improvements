Make sure Doze is enabled and emulator thinks the device is on battery power.

adb shell dumpsys deviceidle enable
adb shell dumpsys battery unplug

NowPlayingFragment will correctly pause. The connection to the server is maintained.

nikclayton added a commit that referenced this issue Mar 4, 2017

Disconnect from the server when entering Doze mode. 
Doze mode suspends network activity, but does not deliver exceptions
or failures to existing connections. When resuming they continue, even
though the underlying network connection may have closed.

Ensure that the network connections are in a known state by explicitly
disconnecting when entering Doze mode.

Fixes #177.

I don't understand how that change would affect a Lollipop device.

Specifically, the code that registers the receiver is also wrapped with a version check in onCreate(),

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
            registerReceiver(deviceIdleModeReceiver, new IntentFilter(
                    PowerManager.ACTION_DEVICE_IDLE_MODE_CHANGED));
        }

so if you didn't modify that as well the receiver won't even be registered on a Lollipop device.
