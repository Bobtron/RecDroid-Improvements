voidstarstar Jan 28, 2018

I can confirm this issue.

Could this be related to #259? Specifically, I'm looking at this post from @mar-v-in: #259 (comment)

I wonder if this bug is due to microG requiring that the spec is strictly met while the Google Voice app does not do this. Google mentions this requirement in their documentation:

The permission name must exactly match this pattern‚Äîotherwise the Android application will not receive the messages.

I decompressed and looked at the AndroidManifest.xml file in the Voice app (full file here). It includes the following permissions:

<permission name="com.google.android.apps.voice.backends.gcm.permission.C2D_MESSAGE" protectionLevel="resourceID 0x2">

<uses-permission name="com.google.android.apps.voice.backends.gcm.permission.C2D_MESSAGE">

The first line in the manifest seems to say that the package should be com.google.android.apps.googlevoice:

<manifest versionCode="resourceID 0x19b7c" versionName="5.4.169585865" package="com.google.android.apps.googlevoice" platformBuildVersionCode="26" platformBuildVersionName="8.0.0">

Is microG requiring that the permission name is com.google.android.apps.googlevoice.permission.C2D_MESSAGE?

If so, perhaps this could be fixed with either of the following:

  1. A temporary hack to patch the Google Voice apk or
  2. A less strict interpretation of the GCM specification in microG

I expected the GCM api to be more forgiving of 3rd party apps, but I assumed official Google apps would at least meet their own specs. I suspect that the spec has become more strict over time, but Google still allowed for the deprecated naming convention for the sake of backwards compatibility. I think option 2 would be the best solution since this appears to affect multiple apps including ones from Google. Also option 2 is probably best since it would help bring the microG implementation closer to the closed source implementation. In the end, we should care about interoperability over adherence to the spec.

Edit: Used another version of the manifest file with better formatting.
