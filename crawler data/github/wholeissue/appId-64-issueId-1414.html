rdoeffinger Mar 19, 2017

Owner

Not sure you really want to know the technical details, but no, it can't be indexing because there is no indexing. The index is stored in the dictionary file, and only loaded in tiny parts and on-demand when needed. All initialization around this is already done by the time something is shown in screen (the index is also needed for scrolling, as the languages a bi-directional an index is needed to know the order in which to show things).
The thing that is loaded in the background and is rather slow is things like how to sort words (e.g. in German, ä is sorted with a, while in Swedish ä is sorted after z) and which how letters can be substituted (e.g. German ä is "equivalent" to ae, German ß can be substituted by ss, Swedish ä and å map to a etc.).
For simple cases like the single-language English dictionary that should be very little data and not take any time, but unfortunately I don't think it is working quite that way and I don't know if it is possible to fix (though I guess an indication that it's still loading should be possible to add at the very least).
