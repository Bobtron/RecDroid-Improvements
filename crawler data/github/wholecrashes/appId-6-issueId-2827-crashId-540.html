https://github.com/timusus/Shuttle/issues/108

Shuttle version:

v1.6.7, Shuttle+ (paid version from Play Store)

Device, OS:

Oneplus 3T, Android 7.1.2 (Lineage OS)

Description of bug:

Creating a new playlist from Folders view hangs the application. Application shows a "Gathering songs..." waiting popup forever while hung.

Steps to reproduce:

  1. Go into the Folders view, select a folder with mp3 files inside and touch the three dots next to it
  2. Select "Add to playlist > New" from the menu and provide a name for the playlist
  3. Press Save
  4. "Gathering songs..." pop up shows and stays forever, app stops reacting to touch.

Expected outcome:

Create a new playlist with songs with selected folder without crashing the app

Observations/Actual Result:

After killing the app via task manager and starting it up again, one can see that an empty playlist with the given name was created before the crash. Trying to add the same folder again to the existing empty playlist works. However, having a crash as a part of regular playlist-creating-via-folder-select workflow isn't the best UX.

Worth noting is also the fact that LineageOS has an app named Privacy Guard for managing app permissions. However, before submitting this bug report, I tested this scenario again with making sure all requested permissions are granted - the bug still happens.
